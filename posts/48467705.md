### npm v12 나오면 설치 스크립트 실행 방식 바뀐다? 
npm v12 나오면 설치 스크립트 자동 실행 안되고, 명시적으로 허용해야 실행됨. 
근데 이미 11.16.0 이상 버전에서는 워닝 메시지 뜨고, 이게 어떤 스크립트가 실행되는지 알려줌. 
_allowScripts_ 옵션이 기본적으로 꺼져 있어서, 설치 스크립트 실행하고 싶으면 explicit 해서 허용해 줘야 함. 
예를 들어, `npm approve-scripts --allow-scripts-pending` 명령어로 허용할 스크립트 목록을 볼 수 있고, 허용하고 싶은 스크립트는 `npm approve-scripts`로 허용하고, 안 허용하고 싶은 스크립트는 `npm deny-scripts`로 거부할 수 있음. 
이 설정들은 `package.json` 파일에 작성되고, 버전 관리에 커밋하면됨. 

이 외에도 git 디펜던시, remote url 디펜던시 사용도 기본적으로 꺼짐. 
이 설정은 npm 11.10.0 이상 버전에서 사용 가능하고, npm 11.15.0 이상 버전에서는 --allow-remote 옵션으로 변경 가능함. 

### 어떻게 대비하면 좋을까? 
npm 11.16.0 이상 버전으로 업그레이드하고, 설치 스크립트 실행하고, 워닝 메시지를 확인하고, 허용할 스크립트 목록을 확인하고, 허용하고 싶은 스크립트는 explicit 허용하고, 안 허용하고 싶은 스크립트는 explicit 거부하면 됨. 
이렇게 해서 npm v12 나오면 설치 스크립트 자동 실행 안되고, 명시적으로 허용해야 실행됨. 
개빡치는 점은, 이미 11.16.0 이상 버전에서는 워닝 메시지가 뜨고, 이게 어떤 스크립트가 실행되는지 알려주고, 명시적으로 허용해야 실행된다는 거임. 
ㅇㅇ, 개 좋은 기능인 것 같음.
