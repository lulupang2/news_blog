### IBM 그라니트 4.1: 8B 모델, 32B 모델을 제낀다 ㅎㄷㄷ
IBM이 공개한 그라니트 4.1은 오픈소스 언어 모델이며, 3가지 버전(3B, 8B, 30B)이 있습니다. 이 모델은 15조 개의 토큰으로 학습되었으며, 파이프라인을 구성하는 데에 대한 집착의 수준이 굉장히 높습니다. 특히 8B 모델은 Dense 아키텍처를 채택하여 MoE 트릭이나 확장된 추론 체인을 사용하지 않으면서도 이전 버전의 32B 모델을凌깁니다. 

#### ArenaHard 벤치마크 결과: 8B 모델이 32B 모델 제치고 1위 ㄷㄷ
- 8B 모델: 69.0
- 32B 모델: 64.7

#### BFCL V3 벤치마크 결과: 8B 모델이 32B 모델 제치고 1위 ㄷㄷ
- 8B 모델: 68.3
- 32B 모델: 64.7

#### GSM8K 벤치마크 결과: 8B 모델이 32B 모델 제치고 1위 ㄷㄷ
- 8B 모델: 92.5
- 32B 모델:不知道 ㅋ

#### 그라니트 4.1 모델은 다음과 같은 특징을 가지고 있습니다:
- Dense 아키텍처를 사용하여 MoE 트릭이나 확장된 추론 체인을 사용하지 않습니다.
- 15조 개의 토큰으로 학습되었습니다.
- 파이프라인을 구성하는 데에 대한 집착의 수준이 굉장히 높습니다.
- 5개의 서로 다른 훈련 단계를 거쳐 학습되었습니다.

#### 훈련 과정:
1. **첫 번째 단계**: 공동 크롤링(common crawl) 59%, 코드 20%, 수학 7% 등으로 구성된 데이터를 사용하여 훈련
2. **두 번째 단계**: 수학 데이터의 비중을 35%로 증가시키고 코드 데이터의 비중을 30%로 증가
3. **세 번째 단계**: 체인 오브 썬드(chain-of-thought) 추론 궤적과 지시 데이터를 함께 추가
4. **네 번째 단계**: 컨텍스트 윈도우를 512K 토큰으로 확장
5. **다섯 번째 단계**:_extended 컨텍스트 윈도우를 사용하여 훈련

이런 훈련 과정을 통해 그라니트 4.1 모델은 32B 모델을凌г며 성능을 발휘합니다.
