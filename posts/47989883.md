> **🔗 원문 보러가기:** [VS Code inserting 'Co-Authored-by Copilot' into commits regardless of usage](https://github.com/microsoft/vscode/pull/310226)

요즘 VS Code 사용하시는 분들은 알고 계실 거지만, 자동으로 'Co-Authored-by Copilot' 이런 커밋 메시지를 넣어줘서 다들 개빡치는 중임 ㅋㅋㅋ. 
   그래서 무슨 일이냐면, VS Code에서 Git 확장 기능을 사용하면 AI가 공동 저작자로 표시해주는 기능이 있는데, 이게 기본적으로 활성화돼서 사용 안해도 자동으로 커밋 메시지에 포함돼버림. 
   이 기능 때문에 다들 환장하는 중인데, '나 왜 커밋 메시지에 Copilot가 포함되어 있지?' 라고 말하는 사람 많음. 
   그래서 이 기능을 끌 수 있는 설정이 있는데, Git 확장 기능의 설정에서 'AI 공동 저작자'를 끌 수 있게 해둠. 
   하지만 이 설정을 기본적으로 끄지 않으면, 사용자들이 AI를 사용 안해도 자동으로 커밋 메시지에 포함돼버리니까 다들 개빡チ네. 
   이 문제는 Git 확장 기능의 설정에서 'AI 공동 저작자'를 기본적으로 끄는 것으로 해결될 거라 생각하는데, 이 문제를 해결하기 위해 Git 확장 기능의 설정을 바꾸거나, AI 공동 저작자 기능을 끄는 등 다양한 해결 방법을 찾아보실 수 있을 거임.
