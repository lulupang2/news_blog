레이디버드 개발 방식 바뀜! 이제 외부 공헌은 없다! ㅋㅋㅋ
안드레아스 클링이란 사람이昨日 이렇게 말했어. 레이디버드 프로젝트가 이제부터 코드 변경이나 pull request를 외부에서 받지 않는단다. 이제는 프로젝트 메인테이너들만 코드를 변경할 수 있게 돼씀. 
레이디버드는 새로운 단계로 넘어가고 있지. 첫 알파 버전을 내보내기 위해 이제부터는 개발 프로세스가 더 견고해야 하고, 보안 모델이 더 분명해야 해. 따라서 코드 변경이나 유지 보수를 담당하는 사람이 더 적어져야 하겠지.
이 결정은 쉽지 않았단다. 지금까지 외부에서 많은 사람들이 공헌해줬어. 이 프로젝트는 오픈소스 프로젝트니까, 외부에서 코드를 보내주고 피드백을 주는 게 중요한 거야. 하지만 이제는 AI 툴들이 코드를 만들기 쉬워져서,.pull request만 보내면 그 사람을 믿을 수 없게 돼씀. 
브라우저는 전체 인터넷의 입력을 사용자의 머신에서 실행하니까, 보안이 중요하지. 하나의 취약점만 있으면 공격자들이 이용할 수 있겠지._external에서 공헌을 받게 되면, 그 코드의 책임도 같이 지야 하니까, 이제부터는 프로젝트 메인테이너들이 코드 변경을 책임지게 돼씀.
레이디버드는 여전히 오픈소스 프로젝트야. 소스 코드는 여전히 공개돼 있고, 외부에서 버그 리포트, 테스트, 디자인 논의, 보안 리포트, 기술 피드백 等을하면 프로젝트에 도움이돼. 
하지만 이제부터는 외부에서 코드를 보내거나 pull request를 하거나 하는 그런 것은 받지 않겠다고. 레이디버드 프로젝트는 이제부터 프로젝트 메인테이너들만 코드를 변경할 수 있게 돼씀. 
이 결정이 레이디버드 프로젝트에 도움이 되길 바래.
