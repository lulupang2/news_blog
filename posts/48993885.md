## npm 릴리즈 쿨다운? 그딴건 보안 취지뿐이야 ㅋ
npm과 같은 패키지 매니저들은 요즘들어 쿨다운에 푹 빠져있음. 업스트림 레지스트리에 7일 혹은 24시간의 시간 제한을 두는 거임. 달콤한 꿀처럼听聽 하지만, 효과는 없다는 거임. 그냥 허구한날 보안극장일 뿐 ㅋ

일단, 사람들은 패키지를 설치하기 전에 커뮤니티에서 검토를 하길 기다림. 하지만 누가 검토할지 UNKNOWN임. 그냥 마음씨 착한 사람이 설치하고, 뭔가 잘못된 걸 찾으면 모두에게 경고를 날릴거라고 생각하겠지? 하지만 현실은 그렇게 호락호락하지 않음. 모두가 쿨다운을 설정하면,早い 사람이라는 사람이 존재하지 않음. 모두가 기다리는데, 기다려도 오지 않는다 이거임. 그냥 고도를 기다린다고 고도는 오지 않는 법 ㅋ

CI/CD에 의존하는건 소용없어. 쓸모도 없고, 늦음. 자동화된 파이프라인이나 CI/CD를 쓰면 최신 패키지를 가져오는데, 악의적인 코드가 있을 때는 두 가지 결과가 나올거임. 악성 코드를 못 찾거나, 성공적으로 악성 코드를 설치하는 거임. PR이 머지되고, 개발자들은 npm install을 그냥 실행할거임. 아직 신뢰할 수 없는 코드인데, 아무도 확인하지 않음. 문제가 터지기 전에는 아무도 모르고, 악성 코드는 그냥 설치됨. 너무晚임.

DYOR이 중요하단 얘기야. Vendored 코드를 랜덤한 윈도우 실행파일로 생각해라. 정적 분석 툴을 가져오면 good, SAST도 가져오면 good, 샌드박스를 가져오면 good, HIPS도 가져오면 good. LLM 지원_audit을 Vendored 코드에서 작동시켜라. PR이 머지되거나 개발자가 설치하기 전에 작동시켜야지. 그냥 기다리지 말고, 적극적으로 스캔을 시작해라. 보안극장은 이제 그만 ㅋ
