### Gigatoken登場
Gigatoken은 언어 모델 토큰화 속도가 HuggingFace의 토큰화 속도보다 1000배 이상 빠르다고 함ㅋㅋㅋ 
직접 사용해 본 결과 확실히 엄청나게 빠르길래 자세한 내용을 알아봄 

### Gigatoken이 뭔데?
Gigatoken은 언어 모델을 위한 가장 빠른 토큰화 도구라고 함ㅋㅋㅋ 
wide range of CPU 하드웨어를 지원하고, 거의 모든 일반적인 토큰화器를 지원하는듯 
Benchmark 부분을 보면 토큰화器와 CPU에 따른 처리 속도 세부 내용을 볼 수 있음

### 사용 방법
Gigatoken은自己的 API 또는 HuggingFace Tokenizers 또는 Tiktoken과 호환되는 모드로 사용할 수 있음 
### 호환성 모드 
```python
import gigatoken as gt
hf_tokenizer = ... 
tokenizer = gt.Tokenizer(hf_tokenizer).as_hf()
# tokenizer는 hf_tokenizer와 동일한 컨텍스트에서 사용할 수 있음 
tokens = tokenizer.encode_batch(["이것은 테스트 문자열", "还有 다른 하나는"]) 
```
이 방법은 HuggingFace Tokenizers와의 호환성을 유지하면서도 속도 향상을 제공하지만, Gigatoken API를 사용하는 것보다 약간 느릴 수 있음 

### Gigatoken API 
```python
import gigatoken as gt 
tokenizer = gt.Tokenizer("Qwen/Qwen3-8B") 
file_source = gt.TextFileSource(["owt_train.txt"], separator=b"<|endoftext|>") 
tokens = tokenizer.encode_files(file_source) 
```
Gigatoken API를 사용하면 Rust 구현이 직접 데이터를 읽어 와서 최대한 오버헤드를 줄이고 최대 병렬성을 허용할 수 있음 
但是, Python 데이터 구조를 통해 이 API를 사용하는 경우仍然 Python에서 읽는 오버헤드가 발생할 수 있음 

### 벤치마크 
owt_train.txt (11.9 GB) 문서의 인코딩 처리 속도를 측정해 봄 
- AMD EPYC 9565 72-Core Processor x 2 소켓 (144 코어) 
- Apple M4 Max (16 코어) 
- AMD Ryzen 7 9800X3D 8-Core Processor (16 코어) 

### 벤치마크 결과 
Gigatoken의 처리 속도는 HuggingFace Tokenizers와 Tiktoken에 비해 월등히 빠름을 확인할 수 있음 
예를 들어, AMD EPYC 9565 72-Core Processor에서 GPT-2 모델의 경우 Gigatoken은 24.53 GB/s의 속도로 처리되는 반면, HuggingFace Tokenizers는 24.8 MB/s, Tiktoken은 36.0 MB/s의 속도로 처리됨 

### FAQ 
Q: 특정 CPU와 토큰화器에 대해 너무 최적화한 것 아닌가요? 
A: 그렇지 않습니다. Gigatoken은 다양한 CPU와 토큰화器에 대해 광범위하게 최적화되었습니다. 
Q: 내 토큰화器가 지원되는지 어떻게 확인할 수 있나요? 
A: 다음 명령어를 사용하여_validate하고 처리 속도를 측정할 수 있습니다. 
```bash
uvx --with tokenizers gigatoken bench 'openai-community/gpt2' owt_train.txt \
--validate --doc-separator "<|endoftext|>"
```
