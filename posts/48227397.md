# 免费BSD로의 이사
안녕하십니까. 오늘은 10년간 우분투 16.04에서 뛰던 블로그를 免费BSD로 이사했는데요. 이과정에서 어떤 부분들이 변경되었는지 설명하도록 하겠습니다.

## 문제의 시작
우분투 16.04는 LTS 버전이지만 이미 지원이 중단된 상태임. 따라서 apt 패키지 저장소에서 업데이트를 받을 수 없게 되는데요. 이는 보안상 매우 취약한 상황이죠.bots가 노드를 찾으려고 할 때 취약한 노드를 찾을 수도 있고 말도 안되는 코드를 집어 넣을 수도 있죠. 다행히 뭐 큰일은 없었지만 우분투 16.04를 사용하던 블로그가突然 이상한 코드들이 나타나는 적도 있었음. 

## 새로운 선택
저는 이미 헌츠너의 VPS를 하나 구매했었는데요. 헌츠너는 유럽에서 굉장히 유명한 IT 회사고 독일에 거대한 데이터 센터가 있음. 저는 가장 싼 3.56유로짜리 서버를 구입했는데요. 이는 원래 사용하던것보다 램이 2배이고 cpu가 2배이고 저장공간은 조금 적지만 트래픽은 10배인 서버였음. 그리고 요금은 반의 반에 불과함.

## 설정
설명을 시작하기 전에 제가 사용하는 서버의 설정을 조금 설명하겠습니다. 
- Digital Ocean 
  - 램 2GB
  - cpu 1개
  - 저장공간 50GB
  - 월 트래픽 2TB
  - 우분투 16.04 x64
  - 뉴욕에 데이터 센터가 있음.
  - 13달러/월 

- 헌츠너
  - 램 4GB
  - cpu 2개
  - 저장공간 40GB
  - 월 트래픽 20TB
  - 5.96유로/월

## 새로운 선택의 이유
저는 免费BSD에 관심이 있었는데요. 우분투와 다른 점이 많습니다.まず는 Jail이라는 개념이있는데 이는 Docker와 비슷한 개념이지만 더 강력하고 안정적인 시스템이라고 생각합니다. 그리고 filesystem 또한 ZFS를 사용하고 있는데 이는 Btrfs와 비슷한 filesystem인데 더안정적이고 다양한 기능을 제공합니다. 
- Jail
  - 가상화/컨테이너화
  - 25년전부터 있었습니다. 
  - Docker 이전에 있었음.
  - Docker는 프로그램을 패키징하는데 적합.
  - Jail은 mini-VM.
  - 같은 커널을 사용하지만 완전히 분리된 환경.
- ZFS 
  - filesystem
  - 안정적이고 강력한 시스템
  - 스냅샷기능을 가짐.
  - 데이터 무결성 유지.

## 설정하기
저는 한 Jail당 사이트를 구성했습니다. Caddy라는 웹서버를 사용했고 각 Jail에서 사이트를 구축했습니다. 그리고 각 Jail에서는 필요한 프로그램을 설치했습니다. 
- Jail
  - 각 사이트마다 하나씩.
  - 각 Jail에는 hugo와 같은 필요한 프로그램이 설치되어 있음.
  - 각 Jail에서 사이트 구축.
- Caddy
  - 웹서버
  - 각 사이트를 구축한 Jail에 접근できる 웹서버

## 설정하는 과정
저는 다음의 과정으로 서버를 설정했습니다. 
1. Jail 생성
2. 필요한 프로그램 설치
3. 사이트 구축
4. Caddy 설정
5. Jail을 Caddy에 연결

## 결론
저는 免费BSD를 사용하고 있습니다. 이는 우분투와는 많이 다른 시스템이지만 더 안정적이고 강력한 시스템이라고 생각합니다. 그리고 Jail과 ZFS는 매우 유용한 기능이라고 생각합니다. หาก 여러분이 새로운 서버를 구축한다면 免费BSD를 고려해 보시기를 바랍니다.
