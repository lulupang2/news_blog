### 개요
리습과 스키마를 사용하는 이유를 설명하는 기사임. 

프로그래밍언어는 아름답고, 수학적으로 완벽한 이상적인 프로그램을 만들기 위한 도구지만, 실제로는 그냥 코드를 작성해서 쓸모있는 프로그램을 만들고 싶을 때가 많다고 함. 

저자는 하스켈을 사랑하지만, 하스켈의 순수 함수형 프로그래밍이 간단한 작업을 하기 어렵게 만든다는 것을Feel한 적이 많다고 함. 

하스켈의 순수 함수형 프로그래밍은 프로그래머에게 높은 수준의 추상화를 제공하지만, 이를 이해하기 어려우며, 간단한 작업을 하기에도 많은 노력이 필요하다고 함. 

### 리습과 스키마
리습과 스키마는 하스켈보다 간단하고, 실제 문제를 해결하기 쉬운 언어라고 함. 

저자는 리습과 스키마의 간단함이 장점으로 작용하며, 실제로 많은 프로젝트에서 사용할 수 있다고 함. 

리습과 스키마는 하스켈의 추상화와 복잡성 대신에, 실제 문제를 해결하는 데에 필요한 단순함과 유연성을 제공하기 때문이란 걸 알 수 있음. 

### REPL
REPL(read-eval-print loop)은 프로그래머가 코드를 작성하고, 실행하고, 결과를 확인할 수 있는 환경을 제공하는 도구임. 

리습과 스키마는优秀한 REPL을 제공하며, 이를 통해 프로그래머가 코드를 작성하고, 테스트하고, 디버깅할 수 있음. 

REPL을 사용하면, 프로그래머는 코드를 작성하고, 실행하고, 결과를 확인할 수 있기 때문에, 개발 과정을 빠르고, 효율적으로 진행할 수 있음. 

### 결론
리습과 스키마를 사용하는 이유는, 하스켈보다 간단하고, 실제 문제를 해결하기 쉬운 언어라는 점임. 

리습과 스키마는 하스ヶ의 추상화와 복잡성 대신에, 실제 문제를 해결하는 데에 필요한 단순함과 유연성을 제공하며, 이를 통해 프로그래머가 빠르고, 효율적으로 개발할 수 있게 해줌. 

그러므로, 프로그래머는 리습과 스키마를 사용하여, 실제 문제를 해결하는 데에 필요한 도구를 제공받을 수 있음.
