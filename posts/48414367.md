# pg_durable: PostgreSQL에서 쿼리 오류 나도 계속 돌아가는 기능 오픈소스公開 ㅋㅋㅋㅋ


포스트그레sql에서 배경 작업을 실행할 때 오류 나거나 껏다 켜도 다시 처음부터 시작해야 하는 문제점을 해결하는 pg_durable 기능을 마이크로소프트에서 오픈소스로 공개했다는 소식 ㅋㅋㅋㅋ


이 기능은 쿼리를 그래프 형태로 저장하고, 각 단계마다 체크포인트를 설정하여 오류 발생 시 바로 이전 단계부터 다시 시작할 수 있게 해줌. 따라서 쿼리 작업을 중간에 껐더라도, 이전까지의 작업 결과를 다시 만들 필요 없이 바로 이전 단계부터 다시 시작할 수 있음 ㅋ


이 기능은 PostgreSQL에서만 작동하므로, 별도의 서비스 인프라가 필요 없다는 장점이 있음. 즉, PostgreSQL만 있으면 이 기능을 사용할 수 있다는 거임. 사용법은 간단해서, df.start() 함수를 호출해서 쿼리를 시작하면 됨. 그리고 쿼리 실행 결과를 확인하려면 df.instances 테이블에 조회하면 됨.


이 기능의 용례는 데이터 수집, 배치 작업, 스케줄링 등 다양한 곳에서 활용될 수 있음. 또한, PostgreSQL에서만 사용할 수 있기 때문에, 데이터베이스 관리자나 개발자들에게는 큰 도움이 될 것임.


마이크로소프트는 이 기능을 Azure HorizonDB에서 이미 사용 중이라고 하며, PostgreSQL 사용자들이 이 기능을 쉽게 사용할 수 있도록 도와줄 것임.


이 기능은 PostgreSQL 사용자들이 배경 작업을 더 효율적으로 처리할 수 있도록 도와줄 것이며, 데이터베이스 quản理와 유지 보수에 큰 도움이 될 것임.
