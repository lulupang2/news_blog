### 비트ウォ든 CLI 뚫림! 
_SOCKET 리서치 팀에서 발견한 바로는 비트ウォ든 CLI 2026.4.0 버전이 Checkmarx 공급망 공격의 피해를 입은 것으로 나타남. 비트ウォ든은 1000만명 이상의 사용자와 5만개 이상의 기업에서 쓰는 오픈소스 패스워드 매니저 중 하나야. 그리고 기업용으로는 탑3 안에 들 정도로 많이 쓰는 프로그램임.

### 비트ウォ든 CLI 뚫린 거 왜 문제가 되는데? 
_affected된 pakcage 버전은 @bitwarden/cli 2026.4.0이고, 악성 코드는 bw1.js 파일에 들어있음. 이 공격은 GitHub Actions CI/CD 파이프라인에 있는 취약한 부분을 노린거지. 다른@Repository에서도 같은 방식으로 공격 당한 거랑 비슷한데, 같은 악성코드 사용해 공격한 거라고 보면 됨.

### 비트ウォ든 CLI 공격 세부 사항은? 
_* 공격은 비트ウォ든 CLI 빌드에影响을 미침
* 공격은 Checkmarx 캠페인에서見た 것과 동일한 GitHub Actions 공급망 벡터를 사용
* 이건まだ進行中인 조사라서 더 자세한 찾아본 내용은 나중에 따로 publish 할 거임

### 이거 왜 이렇게 심각한 거야? 
_비트ウォ든 CLI 쓰면 사용자 정보 등이 노출될 가능성이 높아. 사용자들은 CI 로그를 확인하고, 공격에 노출된 비밀키 등을 돌려놔야 할 거 같음.

### 공격 코드 세부 사항은? 
_* 악성 코드는 bw1.js 파일에 들어있고, same C2 endpoint를 사용해 공격 당한 다른 코드랑 비슷한 점이 많음
* 공격 코드는 GitHub Actions Runner에서 메모리를 스캔해 사용자 정보를 빼가는 코드도 포함하고 있고, npm 패키지도 조작해 사용자 정보를 빼감
* 공격 코드는 여러 가지 방법으로 사용자 정보를 빼감. AWS, Azure, GCP같은 클라우드 서비스 비밀키도 포함해서 빼가.

### 이 상황 어떻게対処해야 하냐고요? 
_*まず는 afectされた pakcage를 삭제하고, 비밀키 등을 돌려놔야 함
* 사용자들은 CI 로그를 확인하고, 공격에 노출된 비밀키 등을 돌려놔야 할 거 같음. 그리고 npm 패키지도 확인해서 이상한 것 있으면 ngay 삭제해야 함.
* 사용자들은 GitHub Actions도 확인해서 이상한 워크플로우 있으면 바로 삭제해야 하고, AWS, Azure, GCP같은 클라우드 서비스도 확인해서 이상한 활동 있으면 바로 신고해야 함.
