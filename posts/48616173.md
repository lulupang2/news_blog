### 윈도우 UI 진화사: 연관되지 않은 파일 클릭하기
   윈도우에서 파일을 열면 보통 익숙한 익스플로러나 노트패드 이런 거로 열리지 않음? 
   그런데 .exe나 .txt가 아니면 어떤 파일을 클릭해도 안 열림 ㅋㅋㅋ
   예를 들어 윈도우 386/2.11 버전에서는 그냥 'ㄴㄴ'이라고 뜨고 말음 
   'ABC.OMG 는 실행할 수 없는 파일이야' 이거 보단 더 심플한 메시지Unavailable ㅋㅋㅋ
   이 버전부터는already '파일 유형 $foo를 프로그램 $bar로 열어봐' 이런 개념이 있었지만 
   이걸 설정할 수 있는 UI는 없었음... 없던가? 
   WIN.INI 파일에서 파일 연관을 설정할 수 있었지만 그게 별 도움이 안 됨 
   윈도우 3.1 버전에서는 상황이 조금 나아짐 
   'ㄴㄴ'는 여전하지만 파일 유형 연관과 설정 방법에 대한 설명이 추가됨 
   이렇게 기본적인 대화상자가 열리게 되는데 
   상단의 텍스트 박스에 파일 확장자를 입력하고 하단의 목록에서 설치된 프로그램을 선택할 수 있음 
   목록에 없으면 'Durchsuchen...' 버튼을 클릭해서 임의의 프로그램을 선택할 수 있음 
   윈도우 NT 3.1 버전은 윈도우 3.1과 거의 같았고 윈도우 for Workgroups 3.11도 비슷했음 
   윈도우 95 버전에서는 대화상자에서 원하는 프로그램을 직접 선택할 수 있게 되었는데 
   목록에 없으면 'Andere...' 버튼을 클릭해서 프로그램을 선택할 수 있음 
   이 행동은 윈도우 98, 윈도우 ME, 윈도우 2000 버전에서도 같았음 
   윈도우 XP 버전에서는 인터넷 서비스를 사용하도록 유도했는데 
   이건 마이크로소프트가 운영하는 서비스였던 것 같음 
   정확히 기억나지 않지만 어떤 프로그램을 목록에 표시했는지, 어떻게 데이터베이스를 쿼리했는지, 프로그램을 설치하도록 제공했는지 기억나지 않음 
   이 서비스는 2006년에 종료되었고, 쓸모가 없던 것 같음 
   윈도우 10 버전에서는 대화상자가 더 평면적으로 바뀌었는데 
   어떤 항목이 클릭 가능하고 어떤 항목이 클릭 불가능한지 구분하기 어려움 
   'Im Store nach einer App suchen' 이건 버튼임... 
   'Weitere Apps'도 클릭 가능하고_installed 프로그램 목록이 열림 
   이 목록을 스크롤해도 다른 항목이 나타나고 
   'Andere App auf diesem PC suchen' 버튼을 클릭해서 임의의 프로그램을 선택할 수 있음 
   이게 윈도우에서 연관되지 않은 파일을 클릭했을 때 어떻게 변해왔는지 요약했음 
   이제 여름을 즐기세요... 개 더워
