# 러스트에서 지그로 리웨이트는 이렇게進行 중임ㅋㅋ

지그 컴파일러 제작 팀에서 1년 반 동안러스트 30만 줄 코드를 전부 지그로 바꾸고 있다고함 ㅇㅇ

recently 원래 컴파일러와 기능이 같아졌다고 발표했음 ㄹㅇ 

근데 [Bun 프로젝트](링크)에서도 지그에서 러스트로 바꾸는 경험담을 공유했었는데 ㅋ 지금이 반성해 볼 좋은 시점인 것 같음 

### 기능 동등점 달성 

이 멤프stones는 새로운 컴파일러로 [Rocci Bird](링크) 게임을 업데이트할 수 있어 ㅋㅋㅋㅋ 이 게임은 1000줄도 안되는 로코드로 구성되어 있고, [itch.io](링크)로도 플레이 가능하고 웹어셈블리도 지원함 ㅋ

### 핫 코드 로딩 & 크로스 컴파일 

새 컴파일러로 핫 코드 로딩도 되고, 크로스 컴파일도 가능해짐 ㄹㅇ 

예를 들면 [예시 링크](링크)처럼 roc server.roc를 실행하고 코드를 바꾸면 자동으로 반영됨 ㅋ 

roc build server.roc 명령어면 최적화된 바이너리도 생성해줌 

### 패턴 매칭 & 스트링 인터폴레이션

새 컴파일러에서 패턴 매칭과 스트링 인터폴레이션도 지원해줌 ㅋㅋ 

HTTP 요청 로직 예시도 같이 공유해줌 [예시 코드](링크) ㅋ 

### 리웨이트 이유 

그리고 리웨이트 이유에 대해서도 설명해줌 ㅋㅋ 

러스트에서 지그로 바꾸기로 한 이유와, 리웨이트를 결정하게 된 과정을 공유함 

그리고 컴파일러 리웨이트의 어려움과, 지그를 선택한 이유를 설명함 ㅋㅋㅋㅋ 

이해가 가지 않으면 [링크](링크)로 가서 좀 더 자세히 看看해 볼거긔!
