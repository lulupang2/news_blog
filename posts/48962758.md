하드코어 인디웹은 기존의 인디웹 원리를 완전히 수용하는 거임. 핵심은 콘텐츠 컨트롤과 웹사이트 이식성임.

### Hardcore 인디웹 방법론
하드코아 인디웹은 90년대 웹 발행 방식을 그대로 따른다니까? 
이런 방식은 매우 단순한데, 
1. 너의 하드 드라이브에서 웹 컨텐츠 작성
2. 브라우저에서 미리보기
3. 업로드하고 반복 

그럼 몇 가지 필요한 도구는 무엇일까? 
- 텍스트 편집기 
- 파일전송 툴 
- 웹 호스팅 

또한, 너는 어떤 특별한 소프트웨어나 프로그래밍 환경이 필요 없다니까? 그냥 간단하게 HTML만 알면 됨. HTML은 웹의 언어니까 ㅋㅋㅋㅋ 
그리고 NearlyFreeSpeech.net에서는 1일 10원으로 정적 웹사이트를 호스팅할 수 있단다. 
그럼 이제 시작해보자! 
まず는 텍스트 편집기를 찾아야는데, 이미 갖고 있다면 좋고, 없으면 그냥 찾아보면 됨. 
파일 전송툴도 마찬가지고, FileZilla 같은 걸 써도 됨. 
그다음은 웹 호스팅인데, NearlyFreeSpeech.net를 추천하는데, 다른 걸 쓰고 싶으면 자유임. 

### 시작하기 
이제编辑기, 파일 전송툴, 웹 호스팅까지 다 갖추었으면, 준비완료임. 
어떤 종류의 웹사이트를運営할지 생각해봐. 
보통 블로그는 이런 구성 요소들로 구성됨:
- 랜딩 페이지
- 포스트/Άρθρο
- 아카이브 페이지 
- 피드 

이런 걸 관리하기 위해 특별한 블로깅 서비스나 소프트웨어가 필요한 줄 알았지만, 사실은 직접 관리할 수 있다니깐? 
그럼 시작해 보자! 
랜딩 페이지는 그냥 index.html 파일을만들면되고, 네가 원하는 거 넣으면 됨.

아카이브 페이지는 포스트들을 모아두면 되고, 
피드는 rss 피드를 관리하면 됨. 
 rss 피드는 걍 xml 포맷으로 작성하면 되는데, 걍 복사 붙여넣기하면서 수정하면 됨.
