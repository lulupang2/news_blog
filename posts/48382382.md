## CPU 성능 개 븅신 되는 경우
요즘 cpu는 성능 개 븅신 됨. 원인은 메모리 접근 하나도 못 틀어서 그런 거임. 
자바 같은 언어들은 클래스 크기가 개 븅신 되게 커서 메모리 접근 하나도 못 틀어. 
그래서 cpu는 성능 개 븅신 되는 거임.

### 캐시 라인 크기
 캐시 라인 크기는 64바이트임. 즉, cpu가 메모리에서 데이터를 읽을 때는 64바이트를 한꺼번에 읽는 거임. 
그런데 자바 같은 언어들은 클래스 크기가 개 븅신 되게 커서 메모리 접근 하나도 못 틀어. 
그래서 cpu는 캐시 라인을 못 맞춤. 
결과는 cpu 성능 개 븅신 되는 거임.

### 구조체와 캐시 라인
 구조체를 정의할 때는 캐시 라인을 고려해야 함. 
 구조체 크기가 64바이트가 넘으면 캐시 라인이 여러 개 필요함. 
그리고 구조체에 있는 변수가 많으면 캐시 라인이 여러 개 필요함. 
그래서 cpu 성능 개 븅신 되는 거임.

### 데이터 접근 패턴
 데이터 접근 패턴은 cpu 성능에 영향을 줌. 
순차적으로 데이터에 접근하면 cpu는 캐시 라인을 미리 읽어옴. 
그러면 cpu 성능 개 븅신 안 돌아감. 
하지만 랜덤하게 데이터에 접근하면 cpu 성능 개 븅신 돌아감. 
 cpu는 캐시 라인을 미리 읽을 수 없으니까. 
 그래서 cpu 성능 개 븅신 되는 거임.
