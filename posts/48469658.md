### 맥북에서 리눅스 환경으로~
맥북 유저라면 맥북에서 리눅스 환경을 쓸 수 있다니 꿀잼인 거 알지? 컨테이너 머신이란 건 리눅스 환경을 맥북에서 쉽게 만드는 기술임. 
컨테이너 머신은 빠르고 가볍고 영속적인 리눅스 환경을 제공함. OCI 이미지로 만들고 공유할 수 있어 ㅋㅋ. 
홈 디렉토리 공유 같은 호스트 통합 기능으로 터미널에서 어디서든 리눅스 환경에 빠르게 접근할 수 있음. 

### 컨테이너 머신 왜 사용할까?
컨테이너는 보통 애플리케이션을 모델로 하지만 컨테이너 머신은 리눅스 환경을 모델로 함. 이미지의 init 시스템을 실행해서 롱런닝 서비스나 프로세스 суп어나이저 아래 애플리케이션을 테스트할 수 있음. 
또 컨테이너 머신은 자동으로 사용자 이름과 홈 디렉토리를 리눅스 환경에 매핑해 줌. 리포지토리와 닷파일이 양쪽 플랫폼에서 사용할 수 있음. 
맥북 에디터나 IDE를 사용해서 리눅스 환경 안에서 애플리케이션을 빌드하고 실행할 수 있음. 맥북 에디터로 수정하고 리눅스 환경에서 빌드하고 실행하면 되지 않겠어? ㅋㅋ

컨테이너 머신은 리눅스 서비스를 테스트하는 데 진짜 리눅스 서비스를 제공함. 포스트그레스ql 같은 데이터베이스나 스택이 필요한 서비스를 시스템 서비스로 실행할 수 있음. 
 컨테이너 머신은 하나의 환경에서 여러 개의 타겟 디스트로를 테스트할 수 있음. 알파인, 우분투, 데비안 같은 다양한 디스트로에서 애플리케이션을 테스트할 수 있음.

### 컨테이너 머신 써보기
컨테이너 머신은 빠르게 시작할 수 있음. 알파인 리눅스 환경을 쓰는 컨테이너 머신을 만들고 나면 whoami 명령어로 호스트 사용자 이름을 확인하고 pwd 명령어로 홈 디렉토리를 확인할 수 있음. 
컨테이너 머신은 기본적으로 stopped 상태로 시작하는데 컨테이너 머신 런으로 실행할 수 있음. 컨테이너 머신을 실행하면 자동으로 시작함. 

컨테이너 머신에서interactive 쉘이나 단일 명령어를 실행할 수 있음. 명령어 없이 컨테이너 머신 런하면 인터랙티브 쉘을 열 수 있음. 
컨테이너 머신을 중지하고 삭제할 수 있음. 컨테이너 머신을 삭제하면 영속적 저장소도 삭제됨. 

### 컨테이너 머신 설정하기
컨테이너 머신 설정은 컨테이너 머신 세트 명령어로 가능함. cpus, 메모리, 홈 마운트 등을 변경할 수 있음. 
컨테이너 머신 설정 변경은 다음에 컨테이너 머신을 실행할 때 반영됨. 

컨테이너 머신 이미지 빌드하기
컨테이너 머신 이미지를自分で 빌드할 수 있음. 리눅스 이미지를 만들고 컨테이너 머신으로 사용할 수 있음. 
 컨테이너 머신 이미지는 /sbin/init이 포함된 아무 리눅스 이미지でも 가능함. 
 예를 들어 우분투 24.04 이미지를 컨테이너 머신 이미지를 만들 수 있음. 
 컨테이너 머신 이미지를 빌드하고 컨테이너 머신으로 만들 수 있음. 

개인이 만든 컨테이너 머신 이미지를 사용하려면 사용자 설정 스크립트를 /etc/machine/create-user.sh에 추가하면됨.
