# AI 모델의 tool 호출 구조가 엉망이라는 것
      Anthropic의 모델은 tool 호출시 정해진 schema를 따르지 않는다는 문제가 있어. 이것은 Opus 4.8나 Sonnet 5 같은 최신 모델에서 더 많이 발생하고, 옛날 모델에서는 상대적으로 적게 발생함.
      
      ## tool 호출 구조는 텍스트 기반으로 작동해
      모델은 입력받은 텍스트와 시스템 프롬프트를 바탕으로 tool 호출을 결정해. tool 호출의 인자값은 json 형태로 전달되는데, Anthropic 모델은 이 부분에서 문제가 생김. 호출 구조를 잘못 만들거나, 임의의 필드를 생성하고, 기존 필드에 이상한 값을 할당할 때가 많아.
      
      ### 문제의 원인은 트레이닝 방식에 있어
      Anthropic의 모델이 클로드 코드(harness)와 함께 트레이닝된다는 점이 문제야. 클로드 코드는 호출 구조에 대한 가_guide_line을 제공하고, 이를 토대로 모델이 호출을 학습해. 하지만 이런 방식은 모델이 특정한 호출 구조에만 너무 적응하게 해, 다른 구조를 처리하기 힘들게 만든다고 볼 수 있어.
      
      #### 이것은 안토픽 모델에만 해당하는 문제가 아니야
      다른 모델에서도 유사한 문제가 발생할 수 있다는 점을 잊지 말아야 해. 오픈 소스인 gpt-oss 모델의 경우 harmony response format을 사용해, 호출 구조가 정해져 있어. 이런 방식은 모델이 호출 구조를 더 잘 따를 수 있게 해줘.
      
      ##### 해결책은 없나?
      Anthropic과 같은 모델은 호출 구조에 대한 стр릭트한 validation을 통해 이런 문제를 해결할 수 있을 거야. 또한, 모델의 트레이닝 방식도 개선되어야 해. 호출 구조에 대한 다양한 경우의 수를 고려해, 모델이 더 유연하게 호출을 처리할 수 있게 만들어야 한다고 bản인.
      
      더 중요한 건, 개발자들이 이런 문제를 예상하고, 호출 구조를 잘設計해야 한다는 거야. 호출 구조가 복잡하거나, 예상치 못한 경우의 수가 많이 발생할 수록, 모델이 호출 구조를 잘 지키지 못할 가능성이 커지니까.
