# 머리말
러스트를 쓰면 코드가 더 안전해진다는 장점이 있는데, 실제로 그렇게 될까? 이번에 우분투에서 [CVE](https://en.wikipedia.org/wiki/Common_Vulnerabilities_and_Exposures) 목록을 공개하면서, 여러가지 버그들이 드러났음. 
이 중에서 borrow checker가 못잡는 버그들을 분석해 봄.
### 1. 파일시스템에서 발생하는 TOCTOU 버그
#### TOCTOU란?
TOCTOU는 Time Of Check To Time Of Use의 약자로, 파일시스템에서 발생하는 대표적인 버그 중 하나임. 
일단 설명하기 위해서 예제 코드를 하나 보자:
```rust
use std::fs;

fn install(from: &str, to: &str) -> std::io::Result<()> {
    // 1. 파일을 삭제
    fs::remove_file(to)?;

    // 2. 파일을 생성
    let mut dest = fs::File::create(to)?;

    // ...
}
```
여기서 발생할 수 있는 문제는, `remove_file`와 `create` 사이에 경로를 바꾸는 치트가 발생할 수 있음. 
예를 들어서, 
```rust
// to가 /etc/shadow 일 때,
fs::remove_file(to)?;
// 이 때, 공격자가 to를 심볼릭 링크로 바꾸는 치트를 친다면?
let mut dest = fs::File::create(to)?;
// 이 경우, dest에는 심볼릭 링크가 저장되게 되고, 이후의 동작은 예상치 못한 동작을 하게 됨.
```
이를 해결하는 방법은, `OpenOptions`를 이용해서 파일을 생성하는 것임:
```rust
use std::fs::OpenOptions;

fn install(from: &str, to: &str) -> std::io::Result<()> {
    fs::remove_file(to)?;

    let mut dest = OpenOptions::new()
        .write(true)
        .create_new(true)
        .open(to)?;

    // ...
}
```
#### 해결 방안
* 파일 디스크립터를 사용해서, TOCTOU 버그를 예방한다. 
* `create_new`를 이용해서, 파일을 생성할 때 경로가 변경되지 않았는지 확인한다.

### 2. 파일권한을 설정할 때 발생하는 버그
예를 들어서, 디렉토리를 생성하면서 권한을 설정하는 경우:
```rust
use std::fs;

fn create_dir(path: &str) -> std::io::Result<()> {
    fs::create_dir(path)?;
    fs::set_permissions(path, 0o700)?;
    Ok(())
}
```
이 때, 공격자가 디렉토리를 생성한 후에, 권한을 바꾸기 전에 접근하여 권한을 얻어버릴 수 있음.
이를 해결하는 방법은, `OpenOptions`를 이용해서 디렉토리를 생성하고,すぐ에 권한을 설정하는 것임:
```rust
use std::fs::OpenOptions;

fn create_dir(path: &str) -> std::io::Result<()> {
    fs::create_dir_all(path)?;
    let mut dir = OpenOptions::new()
        .write(true)
        .create(true)
        .mode(0o700)
        .open(path)?;

    Ok(())
}
```
### 3. 바이트와 문자열을 다룰 때 발생하는 버그
예를 들어서, `comm` 명령어를 구현할 때, 파일의 내용을 읽어서 문자열로 변환하는 경우:
```rust
use std::io;

fn comm(file: &str) -> std::io::Result<()> {
    let mut ra = String::new();
    io::stdin().read_to_string(&mut ra)?;

    // ...
}
```
이 때, 공격자가 비 UTF-8 문자를 포함하는 파일을 전달할 수 있음.
이를 해결하는 방법은, 비 UTF-8 문자를 포함하는 문자열을 처리할 수 있도록, `OsStr`을 사용하는 것임:
```rust
use std::ffi::OsStr;

fn comm(file: &str) -> std::io::Result<()> {
    let mut ra = Vec::new();
    io::stdin().read_to_end(&mut ra)?;

    // ...
}
```
### 결론
* 러스트의 borrow checker는 코드의안전성을 향상하는데 도움을 주지만, 모든 버그를 잡을 수는 없다. 
* 개발자들은 코드를 작성할 때, TOCTOU 버그, 파일권한 설정 버그, 바이트와 문자열 관련 버그 등에 대해서 주의해야 한다.
