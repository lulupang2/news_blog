> **🔗 원문 보러가기:** [wolfSSL releases a new product; wolfCOSE a zero alloc C embbedded COSE stack](https://github.com/wolfSSL/wolfCOSE)


wolfCOSE란게WolfSSL에서 새로 발표한 제품임. 
狼SSL이라고 하면 보안 분야에서 꽤 유명한 기업이긴 한데,狼COSE는 CBOR랑 COSE를 embodded sistem에 쓸 수 있게 만든 라이브러리임.
 wolfCOSE는 zero allocation이라서 메모리를 많이 먹지 않음.狼SSL의 크립토 백앤드를 사용해서 FIPS 140-3도 지원하고, PQC랑 DO-178랑 MISRA C도 지원해줌.
wolfCOSE의 주요 기능은 CBOR랑 COSE를 embodded sistem에 쓸 수 있게 해줌. 
CBOR랑 COSE는 각각 RFC 8949랑 RFC 9052/9053로 정의된 프로토콜임. 
wolfCOSE는 CBOR랑 COSE를 embodded sistem에 쓸 수 있게 해줌. 
wolfCOSE는 wolfSSL의 크립토 백앤드를 사용해서 FIPS 140-3도 지원하고, PQC랑 DO-178랑 MISRA C도 지원해줌.狼SSL에서 만든 거라서 보안이 든든하고, 쓸만한 라이브러리인거 같음.
wolfCOSE의 주요 특징은 아래와 같음:
- 0.alloc이라서 메모리効率 개좋음 
- Tiny footprint: 7.5 KB .text minimal build (Sign1+ECC), 25.6 KB full (40 algorithms), zero .data/.bss 
-狼SSL의 크립토 백앤드를 사용해서 FIPS 140-3도 지원하고, PQC랑 DO-178랑 MISRA C도 지원해줌.
wolfCOSE의 주요 기능은 아래와 같음:
- Complete RFC 9052 message set: all six COSE message types, including multi-signer COSE_Sign and multi-recipient COSE_Encrypt / COSE_Mac
- Path to FIPS 140-3 via wolfCrypt FIPS Certificate #4718 (sole crypto dependency)
- Supported Algorithms: Signing: ES256, ES384, ES512, EdDSA (Ed25519/Ed448), PS256/384/512, ML-DSA-44/65/87, Encryption: AES-GCM (128/192/256), ChaCha20-Poly1305, AES-CCM variants, MAC: HMAC-SHA256/384/512, AES-MAC, Key Distribution: Direct, AES Key Wrap, ECDH-ES+HKDF 
狼SSL에서 만든 거라서wolfSSL 버전 5.8.0-stable 이상이면 쓸 수 있음.wolfCOSE를 쓰려면 wolfSSL을 먼저 설치해야 함. 
wolfCOSE를 설치하려면 아래와 같은 명령어를 써야 함:
```bash
# minimal build (ECC + AES-GCM)
./autogen.sh
./configure --enable-ecc --enable-aesgcm 
            --enable-sha384 --enable-sha512 --enable-keygen
make && sudo make install
sudo ldconfig

# full build (all algorithms)
./autogen.sh
./configure --enable-ecc --enable-ed25519 --enable-ed448 
            --enable-curve25519 --enable-aesgcm --enable-aesccm 
            --enable-sha384 --enable-sha512 --enable-keygen 
            --enable-rsapss --enable-chacha --enable-poly1305 
            --enable-dilithium --enable-hkdf --enable-aeskeywrap
make && sudo make install
sudo ldconfig
```
