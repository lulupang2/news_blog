### LLM 코드 생성기랑 작업할 때 뇌가 멍해진다는 건 알지?
최근에 난 코드를 작성할 때 코드 생성기랑 작업할 때 내가 코드를 완전히 통제하지 못하는 느낌을 받는다는 걸 깨달았어. 
난 코드를 작성하고 나면 코드를 완성한 外적인 모든 징조를 갖추고 있지만, 사람 손으로 코드를 작성할 때 일어나던 내부적 Процесс들은 전혀 경험하지 못하거든. 
간단히 설명하면, 인간의 뇌에는 여러 가지 유형의 기억력이 있는데, 바로 단기기억, 작업기억, 장기기憶이란 게 있음. 
단기기억은 일시적으로 정보를 수집하고 QUICK하게 처리하는 게 특징이고, 장기기억은 전에 배웠던 것들을 저장해 두는 DB 같은 거라 보면 됨. 
그리고 작업기억은 단기기억과 장기기억에서 정보를 가져와서 ترکی합해서 처리함으로써 정보를 이해하고 해결책을 찾아내는 역할을 함. 
코드를 볼 때(보통은 남이 짠 코드를 볼 때), 이 모든 프로세스가 동시에 일어나면서 내가 코드를 이해하도록 돕고 있지. 
코드 생성기는 기본적으로 내가 실력을 유지하는 걸 방해하는 거임. 특히 코드 생성기랑 작업할 때는 UX가 슬롯머신 같아서, 나는 그냥 레버를 당기고 해결책을 얻어. 
어떻게든 코드 생성기를 사용하면서도 내가 가진 실력을 잃지 않으려면 노력해야 하거든. 
오즈라는 사람이 제안한 방법 중 하나는, 코드 일부를 직접 다시 작성하는 거였음. 
그 방법에 영감을 받아, 난 코드 생성기를 더 의도적으로 사용하기 시작했어. 
지금까지의 결과는 이러함:
*   난 코드의 초기 구현을 직접 작성해서 코드 생성기에 검토를 요청하고, 코맨트를 하나하나 살펴보면서 직접 수정하니까 코드를 더 잘 이해하게 된 거 같음.
*   코드의 특정 부분을 이해하지 못할 때, 코드 생성기에 관련 문서와 PR을 요청해서 찾아보는 것도 유용했음.
*   코드 생성기에 두 가지 접근 방식을 제시해서 어떤 걸 사용할지 선택하게 하는 것도 좋은 방법임.
*   코드 생성기로 코드를 짜고, 다른 사람과 토론하는 것도 좋은 방법이야.
*   코드 생성기를 사용하기 전에 20분 동안 문제를 해결해 보는 것도 좋을 것 같음.
*   책이나 논문을 읽는 거도 도움이 될 거야.
*   기본적인 데이터 구조를 다시 구현하는 것도 도움이 되지.
이 모든 방법들은 코드 생성기로 코드를 짤 때의 속도优势을 제거하긴 하지만, 내가 코드 생성기를 사용할 때 더 좋은 코드를 짤 수 있도록 해줌. 
코드 생성기보다 내가 더 힘들어야 해.
