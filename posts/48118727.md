# 소프트웨어 개발은 이제 다르다
에디맥스 같은 소프트웨어가 주류가 된다는 건 어떤 의미일까? 
그저 텍스트 편집기 만드는 사람들이 에디맥스를 사용한다는 뜻이 아니라고 본다.
그것보다는 이제는 사람들이 원하는 툴을 직접 만들 수 있게 됐다는 것을 의미한다는 거야.
이미 텍스트 편집기 같은 단순한 도구부터 시작해 이제는 복잡한 소프트웨어도 직접 만들 수 있는 시대가 도래한 것 같음.
 
 # 마크다운 뷰어, 정말 쓸모없는 도구일까?
난 마크다운 뷰어가 필요하다고 생각해.
그럼 왜 마크다운 뷰어가 필요한 거야?
그건 우리는 이미 많은 마크다운 문서를 읽고 있으니까요.
소프트웨어 개발에서는 마크다운이 표준이 된 지 이미 오래고,
이제 에이아이 같은 기술이 나오면서 터미널 툴들도 많아졌지만,
읽는 경험은 여전히 불편해.
터미널에서 마크다운 문서를 읽는 건 너무 불편하고,
그래서 난 마크다운 뷰어를 만들기로 했다고.
 
 # 에디맥스 사용자처럼, 직접 만들기
에디맥스 사용자는 원하는 기능이 있으면 직접 만들지,
이미 있는 걸 사용하는 게 낫다고 생각하지 않아.
난 마크다운 뷰어를 만들기 위해 몇 시간을 보냈어.
그런데 코드를 직접 작성한 시간은 30분도 안 된다고 해.
그 이유는 이제는 에이이 도움을 받아서 내가 원하는 툴을 직접 만들 수 있으니까.
난 마크다운 뷰어를 만들었고, 그것은 내 삶의 질을 높여줬어.
이제 나는 마크다운 문서를 읽을 때 더 편안하게 읽을 수 있어.
이건 에디맥스 사용자들이 하는 것처럼 직접 원하는 툴을 만들면 되니까.
 
 # 에디맥스 같은 소프트웨어가 온다
이제는 에디맥스 같은 소프트웨어가 더 많이 나올 거야.
그 이유는 이제는 사람들이 원하는 툴을 직접 만들 수 있으니까.
에디맥스 사용자는 원하는 기능이 있으면 직접 만들지,
이미 있는 걸 사용하는 게 낫다고 생각해.
이제는 그 같은 정신으로 더 많은 소프트웨어가 만들어질 거라고 본다.
 
 # 결론
이제는 사람들 자신이 원하는 툴을 직접 만들 수 있는 시대가 왔다고 생각해.
이미 텍스트 편집기 정도는 직접 만들 수 있고,
곧すれば 더 복잡한 소프트웨어도 직접 만들 수 있지 않을까?
난 그 때가 오기를หว망해.
