### ESP32로 스마트 선풍기 만들기
   요즘 유럽 북반구에는 더위가 심한가 봐. 그래서 우리 이번에는 ESP32와 Iroh를 이용해서 스마트 선풍기를 만들어 볼 거야. 이전의 ESP32 예제들은 대부분 에코 프로토콜을 보여줬었지. 하지만 ESP32는 데이터를 에코하는 거 이상의 용도로 쓰일 수 있지. 그래서 이번에는 ESP32로 온도 센서를 읽고 선풍기를 제어하는 예제를 만들볼게.

   이 예제에서 중요한 점은 클라우드コンポーネント가 없다는 거야. 그냥 웹어셈블리를 지원하는 브라우저에서 아무데서나 접근할 수 있는 작은 웹사이트를 만들 거지.

   ESP32-WROVER devkit을 사용할 건데, 이건 4 MiB의 PSRAM을 가지고 있어서 Iroh의 네트워킹 기능을 모두 사용할 수 있어. 그래서 릴레이 연결도 있고, 어디서든지 리모트로 컨트롤할 수 있지.

   M5StickC-Plus2도 사용할 수 있지만, GPIO 핀을 적응해야 할 거야. ESP32-S3와 같은 다른 devkit을 사용해도 작은 설정 조정으로 exemples를 실행할 수 있어.

   PSRAM이 없는 ESP32도 사용할 수 있지만, 릴레이 연결을 비활성화하고 QUIC 버퍼를 조정해야 해. 메모리가 부족하지 않도록 말이야.

   ### 기본 설정
   첫째, Iroh-ESP32-examples에서 에코 예제를 복사할 거야. ESP32 바이너리를 위해 server-esp32-psram을 사용하고, 클라이언트 측은 그냥 client를 사용할 터. 데스크탑 PC에서 동작하고, 바닐라 같을 거야. 이것은 스마트 선풍기 예제니까, server-esp32-psram를 server-smart-fan으로 이름을 바꿔주고, client를 smart-fan-cli로 바꿀게.

   ### 초기 상태
   Flash를 처음할 때, cargo run 명령어를 사용해서 ESP32에 바이너리를 전송할 거야. ESP32를 USB-C 케이블로 연결하고, cargo run --release 명령어를 사용하면 되는 거지.

   ESP32에 전송하는 동안, WIFI_CONFIG 환경 변수를 설정해서 ESP32가 와이파이에 연결할 수 있도록 해줘야 해. 예를 들어 WIFI_CONFIG=myap:mypass 이런 식으로 설정하면 돼.

   ### 시도해 보기
   endpoint를 시작하고, 클라이언트 바이너리를 사용해서 접근해 볼 수 있어. 

   ###セン서 추가
   이제 센서를 추가해서 온도를 측정해서 선풍기를 제어할 수 있도록 만들어 볼게. 
   DHT22 온도와 습도 센서를 사용해서 Wiring을 해줘야 해. +3.3V와 GND, 데이터 선이 필요한데, 데이터 선은 ESP32의 GPIO 26에 연결하면 됩니다. 

   ### 프로토콜 추가
   이제 센서를 읽는 프로토콜을 추가해서 Iroh를 통해 접근할 수 있도록 만들어 볼 거야. 
   irpc를 사용해서 프로토콜을 정의할 거고, 이것을 사용하면 이후에 프로토콜을 확장하기도 더 쉬울 거야.
