# 프로로그 知識베이스, Kb 라고 하자 ㅋㅋㅋㅋ
 Kb 프로로그 知識베이스는 로컬 우선, 하이퍼 관계형 知識베이스임. 컨텐츠 주소 지정 저장소(CAS)를 쓰고, 프로로그와 SQLite, C로 빌드된 거임. 마스터스 논문 프로토 타입으로 만들었고, 
### 핵심 개념
* 하이퍼 관계형 그래프: 知識은 statement(Subject, Predicate, Object, Properties)로 저장되고, Subject와 Object는 문장 자체가 될 수도 있어. 
* 컨텐츠 주소 지정 저장소: 파일들이 스테이징 되고, SHA-256 해싱 및 원자적으로 커밋됨. 중복 제거 자동임.
* 타임 트래블: 업데이트로 새로운 버전의 문장이 만들어지고, 이전 버전은 버려짐.
* 프로로그 런타임: Trealla Prolog가 주 runtime임. SQLite와 Raylib은 C 共有 라이브러리 쓰고, FFI로 로드함.
### 사용법
1. `git clone`으로 프로젝트 클론하고, `make`로 빌드하기
2. `tpl`로 런타임에서 コマンド 입력하기. 예를 들면 `tpl -l main.pl -- pl load concept(mathematics)`
3. 데이터 베이스에 새로운 문장 추가하기. 예를 들면 `tpl -l main.pl -- pl assert "statement(concept(mathematics), foundation_of, concept(logic), [])"`
4. 버전 히스토리 보기. 예를 들면 `tpl -l main.pl -- pl history concept(mathematics)`
5. 파일을 CAS에 저장하기. 예를 들면 `tpl -l main.pl -- cas add document.pdf`
6. CAS 오브젝트 리스트 보기. 예를 들면 `tpl -l main.pl -- cas list`
7. GUI 띄우기. 예를 들면 `tpl -l main.pl -- gui`
8. 데이터 베이스 일관성 체크하기. 예를 들면 `tpl -l main.pl -- pl verify`
9. 가비지 컬렉션 하기. 예를 들면 `tpl -l main.pl -- pl gc`
### 아키텍처
프로로그 런타임과 SQLite, C로 빌드된 라이브러리들이 조합된 아키텍처임.
