### 딥러닝 모델 최적화하는 방법 (feat. 첫 번째 원리)

딥러닝 모델 최적화할 때 제일 중요한 건 compute-bound, memory-bound, overhead-bound 이렇게 세 가지 종류로 나누는 거 임. 
요약하자면 **코딩할 때 GPU에 부하가 걸리는 시간**을 줄여야 한다는 거임 ㅋ. 
그리고 대부분의 GPU는 **매티스乘法** 같은건 엄청 잘 돌아가는데 다른 연산은 거의 못하는 거임. 
예를 들어 보통 딥러닝 모델에서 layer norm이나 activation function 같은건 거의 **매티스乘法**에 비해 rounding error 수준인 거임. 

근데 **매티스乘法**은 **/operator fusion/** 이런 기술 많이 쓰는데 **operator fusion**이 뭔지 모르겠으면 이 그림 보시면 됩니다. 
![operator fusion](https://user-images.githubusercontent.com/12093829/173231671-8d0e0c9f-93c9-46c2-9b4a-0ac4e0aa4aae.png)

즉, **operator fusion**은 여러 연산을 한번에 합쳐서 메모리 접근을 최소화 하는 거임. 
예를 들어 보통 x.cos().cos() 이런건 4번의 global memory 접근이 필요했지만 **operator fusion**쓰면 2번으로 줄어드는 거임 ㅋ. 
그리고 NVFuser나 XLA 이런 컴파일러는 자동으로 **operator fusion**을 해줌. 
하지만 사람이 직접 **operator fusion**을 하거나 커스텀 CUDA 커널을 만들면 더 많은 성능 개선을 볼 수 있음. 
그리고 **operator fusion**을 하게되면 activation function이나 rematerialization/activation checkpointing 같은 부분에서도 성능 개선을 볼 수 있음. 
개 요약하면 딥러닝 최적화할 때는 **compute-bound**, **memory-bound**, **overhead-bound** 이렇게 세 가지를 나눠서 볼거래요 ㅋ. 
이거 하나만 기억하면 딥러닝 최적화 거의 다 된 거임 ㄹㅇ
