### 러스트로 메모리節約하기 ㅋ
러스트 언어로 개발을 하다 보면 메모리 사용량에 대해 신경을 쓰게 됩니다. 특히 러스트는 메모리 관리에 있어서 다른 언어와는 달리独特한 특징을 가지고 있기에, 러스트 개발자라면 메모리 사용량을 줄여주는 방법에 대해 잘 익혀두면 유용합니다.

러스트의 경우, 구조체를 사용하여 데이터를 표현하게 됩니다. 이러한 구조체는 메모리에서 어떻게 표현되는지에 대해서는 많은 개발자가 생각하지 않기도하는데, 실제로 구조체가 메모리에서 어떻게 표현되는지에 대해 잘 알면 메모리 사용량을줄일 수 있습니다.

#### 구조체의 메모리 표현
러스트에서 구조체의 경우, 다음과 같이 정의할 수 있습니다.
```rust
#[derive(Debug, Clone, Deserialize, Serialize)]
pub struct SmithyShape {
    pub shape_type: SmithyShapeType,
    pub operations: Vec<SmithyReference>,
    pub members: FxHashMap<String, SmithyReference>,
    pub key: Option<SmithyReference>,
    pub value: Option<SmithyReference>,
    pub member: Option<SmithyReference>,
    pub input: Option<SmithyReference>,
    pub output: Option<SmithyReference>,
    pub traits: SmithyTraits,
}
```
이러한 구조체가 메모리에서 어떻게 표현되는지에 대해서는 개발자들이 잘 모를 가능성이 높습니다. 러스트에서 구조체는 다음과 같이 메모리에서 표현됩니다.

- 각 필드가 연속적으로 메모리에 할당됩니다.
- 옵션 타입의 필드는 `Option`이라는 열거형 타입으로 표현됩니다.

이러한 특징을 이용하여 메모리 사용량을 줄여보도록 하겠습니다.

#### 메모리 사용량 줄이기
다음과 같이 구조체를 정의하였습니다.
```rust
#[derive(Debug, Clone, Deserialize, Serialize)]
pub struct SmithyReference {
    pub target: ShortShapeId,
    pub traits: SmithyTraits,
}
```
이 구조체를 다음과 같이 변경하여 메모리 사용량을 줄여보겠습니다.
```rust
#[derive(Debug, Clone, Deserialize, Serialize)]
pub struct SmithyReference {
    pub target: ShortShapeId,
    pub traits: Option<Box<SmithyTraits>>,
}
```
이렇게 구조체를 변경함으로써, `SmithyTraits`가 필요한 경우에만 메모리가 할당되므로 메모리 사용량을 줄일 수 있습니다.

### 마무리
러스트에서 메모리 사용량을 줄이기 위해서는 구조체의 메모리 표현에 대해 이해하고, 이를 이용하여 메모리 사용량을 줄여주는 방법을 익혀두어야합니다. 이러한 방법을 이용하면 메모리 사용량을 효율적으로 줄일 수 있습니다.
