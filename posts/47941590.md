포지조(Forgejo) 보안 구멍 발견 기념 포스팅 임니다. 
근데 페도라(Fedora)가 파구레(Pagure)에서 포지조로 갈아탔길래 포지조 보안 점검좀 해봄. 
그리고 결과물은 ㄹㅇ 심각하길래ㅋㅋㅋ. 
요약하면 SSRF는 여러 곳에서 발견되고, CSP/트러스티드 타입스 설정도 안되어 있고 템플릿 관련 문제도 있고 암호화도 부실하고 인증 관련해서도 문제가 많다는 거임. 
그리고 정보 유출도 여기저기 다 있고, 여러 가지 문제점이 있는데ㅋㅋㅋ. 
이런 문제점들을 하나하나鏇合わせ해서 풀 블론 리모트 코드 실행(RCE)을 가능하게 할 수도 있고, 비밀 정보 유출도 가능하고, 영구 계정 접근도 가능하더라. 
그리고 OAuth2 관련 권한 상승도 가능하고 여러 가지 문제점들이 있음. 
이런 문제점들을 발견하는 데 오랜 시간이 걸리지도 않았음. 
저녁에 조금만 시간 내서 확인해 본 결과 이런 문제점들이 있었는데, 다행인지 불행인지ㅋㅋㅋ RCE는 오픈 등록과 특정 설정이 필요한데 포지조의 기본 설정하고 다르니까 판매 가치가 별로 없는 거임. 
이런 문제점들을 포지조에게 알려줘도 되겠지만, 그냥 코드 베이스가 구리니까 또 다른 문제점들을 하나둘 발견할 수 있을지도 모르겠음. 
이런 문제점들을 직접 고쳐서 풀 리퀘스트 отправ해도 되지만, 그냥 포기한 거임. 
이번 문제점들은 그냥 카로트 디스클로저(Carrot Disclosure)라는 걸 쓰기로 했음. 
카로트 디스클로저는 벤더에게 문제점들을 해결하도록 유도하는 방법임. 
_critical한 문제점의 증명コード를公開해서 소프트웨어가_exploitable한 것을 보여줌. 
그럼 벤더는 전체적인 코드_audit을 하는데, 최대 문제점들을 해결하도록努力할 수 있고, 아니면 사용자들이 불쾌해져서 소프트웨어 쓰지 않도록 할 수 있음. 
카로트 디스클로저를 쓰는 사람은 버그 บニー(Bugs Bunnies)라고 불림.
