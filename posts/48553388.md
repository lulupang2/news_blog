### 덕DB는 무엇인가?
덕DB는 프로세스 안에서 돌아가는 분석용 SQL 데이터베이스야. 분석용이라고 해서 행을 몇백만개씩 스캔해서 필터링하고 집계하고 조인하는 그런 쿼리에 최적화되어 있음. 프로세스안에서 돌아간다는 건 서버가 없단 소리겠지?덕DB에 연결해도 되고 안해도 되고 그냥 라이브러리로 가져다가 함수 호출하듯이 쓸 수록 있잖아. 
### 덕DB가这么 빠른 이유
첫째로, 덕DB는 서버가 없어. ODBC, JDBC 이런 클라이언트 프로토콜이 없다는 뜻이야. 그러면 클라이언트는 결과 값을 받을 때 직렬화되고, 네트워크를 통해 전송되고, 다시 역직렬화되는 이런 과정을 모두 생략할 수 있어. 덕DB는 그냥 프로세스 안에서 돌아가니까, 이런저런 메모리 복사도 안해도 되고, 쓰레드 사이의 동기화도 필요없고, 직렬화/역직렬화도 필요없고, TCP/IP 통신도 필요없이 바로 결과를 반환할 수 있어. 
### 쿼리 실행 과정 
덕DB 쿼리 실행 단계는 일반적인 SQL 데이터베이스와 거의 동일해. SQL을 파싱하고 바인딩하고 플랜을 수립하고 최적화하는 단계를 모두 거치고, 마지막으로 물리적 플랜을 생성하는 단계까지 같잖아.
* **Parsing** : SQL을 파싱해서 AST(Abstract Syntax Tree)로 변환하는 단계. 
* **Binding** : AST의 각 노드를 실제 테이블이나 컬럼과 연결하는 단계. 이때 타입 체크도 같이 해give. 
* **Optimization** : 실제 쿼리 플랜을 생성하기 전에 여러 최적화 단계를 거치게 돼. 
   + **Filter Pushdown** : WHERE 조건을 가능한 한 앞으로 밀어내는 최적화. 덕DB에서는 이 과정을 생략할 수도 있어.
