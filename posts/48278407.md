# 이글 3.1의 등장
이글 시리즈, 즉 이글 1, 이글 2, 이글 3은 연구와 실제 시스템両方에서 가장 많이 채택되고 실무에 적용되는 추측 解碼 알고리즘家族 중 하나가 됐지. 이제 이글 팀, vLLM 팀, 토치스펙 팀은共同으로 이글 3.1을 소개하겠어. 이글 3.1은 추측 解碼의 강건성, 효율성, deployability에서 큰 발전을 이루었음.

## 이글 3.1의创新
추측 解碼는 제어된 환경에서 잘 작동하는데, 다양한 채팅 템플릿이나 긴 컨텍스트 입력, 시스템 프롬프트 등에서는 성능이 떨어지더라고. 이글 팀은 이러한 취약성을 주목해서 '주의漂移'라 불리는 현象을 발견했어. 즉, 추측 깊이가 깊어질수록, drafter가 싱크 토큰에서 자신의 생성 토큰으로 주의를 끌어당기게 돼. 두 가지根本的な 문제를 찾아냈어. 첫째, 융합된 입력 표현이 더 높은 隱藏層의 상태가 drafter 입력을支配해서 불균형해지게 돼. 둘째, 隱藏状態의 크기가 추측 단계에 따라서 불규칙적으로 증가하는 것이죠.

이 두 가지 효과가 합쳐져서 drafter가 더 깊은 추측 깊이로 갈수록 안정성이 떨어지게 돼. 그래서 이글 3.1에서는 두 가지 중요한 아키텍처改善을 도입했어. 하나는 목표 隱藏狀態 후에 FC 정규화를 추가하는 거, 그리고 그 다음에는 포스트-정규화 隱藏状態를 다음 解碼 단계에 넣는 거야. 이렇게 되면, 메서드가 단순히 추가레이어를 붙이는 것이 아니라,_recursive하게 drafter를 호출하는 효과를 가졌어. इसस해서 배포 시나리오에서 강건성이 크게 개선됐음.

이글 3.1은 이글 3와 비교해서, 다음과 같은 장점을 가졌어.
*   학습 시간에서 추론 시간으로의 일반화가 더 좋아짐
*   긴 컨텍스트에 대한 강건성이 높아졌어
*   채팅 템플릿 và 시스템 프롬프트의 변異성에 karşı 더 강해졌고
*   다양한 환경에서 더 안정적인 수락 길이를 가진대.

특히 긴 컨텍스트 작업량에서는, 이글 3.1이 이글 3보다 최대 2배 더 긴 수락 길이를 가졌대.

## 이글 3.1의 토치스펙 연계
토치스펙은 الآن 이글 3.1뿐만 아니라, 미래의 추측 解碼 알고리즘들을 효율적으로 훈련할 수 있도록 지원을 제공하겠어. 훈련 오버헤드를 낮추고 실험 워크플로우를 간소화함으로써, 토치스펙은 차세대 추측 解碼 연구 및 배포를 가속화하는 데 도움이 될 것 같아. 토치스펙과 vLLM을 바탕으로, 김기 2.6용 이글 3.1 ドラフト 모델도 훈련해서 오픈소스로 공개했어. 모델은 토치스펙 훈련과 vLLM 서빙 지원을 사용해서 실제 배포 모델에서 이글 3.1을 어떻게 적용할 수 있는지 보여줌.

## 이글 3.1의 vLLM 연계
이글 3.1은 vLLM에서 기존 이글 3 구현의 Конфіги베이션 기반 확장으로 통합되었어. 통합에는 다음과 같은 것이 포함되어 있어.
*   FC 정규화 지원
*   포스트-정규화 隱藏状態フィード백
*   목표 隱藏狀態에 대한 하드코딩된 가정을 제거함

동시에, 기존 이글 3 체크포인트와의 하위 호환성은 완전하게 유지되고 있어. 그래서 이글 3.1 ドラ프트 모델을 기존 이글 3 코드 경로에 직접 플러그인할 수 있어. 예를 들어, 다음과 같아.
```bash
vllm serve nvidia/Kimi-K2.6-NVFP4 \
  --trust-remote-code \
  --tensor-parallel-size 4 \
  --tool-call-parser kimi_k2 \
  --enable-auto-tool-choice \
  --reasoning-parser kimi_k2 \
  --attention-backend tokenspeed_mla \
  --speculative-config '{"model":"lightseekorg/kimi-k2.6-eagle3.1-mla","method":"eagle3","num_speculative_tokens":3}' \
  --language-model-only
```
이렇게 하면 프로덕션 vLLM 서빙에서 ドラ프트 모델 업그레이드가 간단해져.

이 기능은 이미 vLLM의 메인 브랜치에 머지되었고, 나이틀리 릴리즈와 即將發表의 v0.22.0 릴리즈에서도 사용할 수 있을 거야.

한 예로, 김기 2.6 이글 3.1 ドラフト 모델을 김기-K2.6-NVFP4에서 vLLM(TP=4, GB200, 비-디쥬각)과 함께 SPEED-Bench コーディング 데이터세트에서 벤치마크 했어. 이글 3.1은 1개의 동시성에서 사용자당 출력 처리량이 2.03배 더 높았고, 동시성이 커져도 속도 향상은 의미가 있었어(동시성 4에서 1.71배, 동시성 16에서 1.66배).

## 개방형 협력
이글 팀, vLLM 팀, 토치스펙 팀의 협력은 開放ソース 생태계에서 알고리즘 연구, 시스템 최적화, 훈련 인프라스트럭처 간의 협력을 보여주는 強力な 예시야. 이글 팀은 추측 解碼 알고리즘을 계속 발전시키고, vLLM은 이를 대規模 프로덕션 추론 시스템에 적용하는 데 도움을 주고, 토치스펙은 미래의 추측 解碼 알고리즘을 효율적으로 훈련하고 실험할 수 있도록 지원해. 함께해서, 추측 解碼의 전체적인 기준을 높이고, 더廣い LLM 생태계에서 토큰 효율성을 개선하기를 바라고 있어.
