# shadcn/UI 이제 기본값이 Base UI로 바뀜

shadcn/ui가 출시된지 3년이 지났어. 그 때랑 지금은 많이 달라졌지. 그 때는 Radix가 최고였는데, 이제는 Base UI가 나타났거든. Base UI는 Radix의 제작진이 다시 만든 거라고 봐도 됨. 그래서 똑같은 팀이 다시 한번 더 시도를 하는 거지.

_base UI는稳定_한 거 같아. 이제 1.6.0 버전으로 업뎃됐고, 주간 다운로드 수도 600만을 넘어섰어. 새로운 컴포넌트도 계속 추가되고 있어.我們도 Base UI를 사용하고 있고, 새로운 프로젝트도 Base UI를 사용하기로 했어. 

# 이제 새로운 프로젝트는 기본적으로 Base UI를 사용하게 될 거야

새로운 프로젝트创建하면 기본적으로 Base UI를 사용하게 되고, 문서도 Base UI를 디폴트로 사용하게 될 거야. Radix.docs는 한 클릭만에 접근할 수 있게끔 해놨음.

# Radix는 아직도 지원하고 있어

Radix는 아직도 지원하고 있고, 새로운 컴포넌트도 계속 업뎃할 거야. 근데 이제 새로운 프로젝트는 Base UI를 사용하는 게 더 좋을 거야. 

# 마이그레이션 하라는 건 절대 아님

마이그레이션을 해야한다는 건 절대 아니야. Radix를 사용하다가 마이그레이션을 해야한다고 생각하면 큰 오산이야. 그냥 그대로 사용하면 돼. 근데 마이그레이션을 하고 싶다면, 우리가 만든 스킬을 사용하면돼. 

# 마이그레이션 스킬

pnpm dlx shadcn/ui add migrate 

이 스킬을 사용하면, 마이그레이션을 조금씩 진행할 수 있어. 한 컴포넌트씩 마이그레이션을 할 수 있고, 프로젝트가 계속 작동하는 상태에서 마이그레이션을 할 수 있어. 

# 새로운 컴포넌트

새로운 컴포넌트도 출시했어. MessageScroller, Message, Bubble, Attachment, Marker가 출시됐고, 이 컴포넌트들은 채팅 인터페이스를 만들 때 사용할 수 있어. 

# 마치며

새로운 컴포넌트와 마이그레이션 스킬의 출시로, shadcn/ui가 더 강력해졌어. 이제 새로운 프로젝트는 Base UI를 사용하고, 마이그레이션을 원하는 사람은 마이그레이션 스킬을 사용하면 돼.
