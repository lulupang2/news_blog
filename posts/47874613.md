# 코드 리뷰의 기술

요즘 코드 리뷰하면서 느낀건데, 코드 리뷰하다보면 코드를 까는 경우가 많음. 그거 하나하나를 다 코드 리뷰에서 수정하라고 하면 지옥이고, 그냥 넘어가면 코드 품질이 떨어지니까. 그럴 때마다 코드 리뷰에 댓글을 달고 승인해버리는건 어때? 

# 왜 댓글을 달아? 

첫 번째는 코드 리뷰를 할때마다 뭔가를 배운다는 느낌을 받을 수 있음. 코드 리뷰를 할때마다 코드의 품질이 높아지고 있고, 더 좋은 코드를 작성할 수 있는 방법이 있다는 생각을 해. 

둘째는 코드 리뷰를 하다보면 내가 뭘 놓쳤는지, 내가 뭘 하지 않은지 발견하게 됨. 그게 바로 코드 리뷰의 기술임.

# 왜 승인할까?

그냥 코드 리뷰를 하다가 뭔가 문제가 생겼다고 생각되면 승인을 안해버리면 코드가 머지되지 않음. 코드가 머지되지 않으면 프로젝트가 끝나지 않음. 그러면 프로젝트를 끝내야겠다고 생각하는 사람들끼리 코드 리뷰를 하다가 서로 다툼. 

# 프로세스 포인츠

첫 번째는 내가 코드 리뷰를 할때마다 코드의 품질이提高되는지 확인하는 것. 

둘째는 코드 리뷰를 할 때마다 코드가 머지되는지 확인하는 것.

# 비단絡 댓글

비단絡 댓글을 달아도 코드가 머지되는건 마찬가지임. 비단絡 댓글을 달면 코드의 품질이提高되고, 코드 리뷰를 하다보면 코드의 품질이 높아짐.

# 코드 리뷰의 기술

코드 리뷰를 할 때마다 코드의 품질을 높이고, 코드 리뷰를 하다보면 코드의 품질이提高된다는 느낌을 받을 수 있음. 그게 바로 코드 리뷰의 기술임.
