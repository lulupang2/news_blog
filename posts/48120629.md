### 디지털 주권과 유럽 클라우드
유럽 클라우드보다 좋을 수 있는 클라우드는 없다고 생각해. 디지털 주권에 대해 깊이 생각해 보면 데이터가 어디에 저장되어 있는지 알고, 갑자기 정책이나 권한이 변해도 디지털 인프라에 액세스를 잃지 않을 수 있는 것임. 
유럽 클라우드는 데이터 주권과 개인 정보 보호를 강화하고, 더 나은 보안과 안정성을 제공하는 데 도움을 줌. 따라서 디지털 인프라를 유럽으로 이사하기로 결정했음.

#### 분석
구글 애널리틱스 -> 마토모(Matomo)로 변경
- 이유: 구글 애널리틱스는 무료지만 사용자 데이터를 수집하고 광고 목적으로 사용해. 마토모는 자체 호스팅으로 데이터가 자신의 서버에 저장되고, 더 많은 통제권을 가지게 되니까 좋음. 

#### 이메일
구글 워크스페이스 -> 프로톤 메일(Proton Mail)로 변경
- 이유: 프로톤 메일은 스위스에 기반하고 있고, 개인 정보 보호에 강한 철학을 가지고 있음. 끝부터 끝까지 암호화된 메일 클라이언트가 제공되고, 캘린더가 잘 작동하는 데다가, 데이터가 유럽에 저장되는 거임. 
- 단점: 프로톤 메일의 필터 시스템은 제한적이라, 이메일 본문의 특정 단어 또는 키워드를 필터링할 수 없음. 

#### 비밀번호 관리
1패스워드 -> 프로톤 패스(Proton Pass)로 변경
- 이유: 프로톤 패스는 끝부터 끝까지 암호화된 비밀번호 관리자이고, 프로톤 메일과 유사한 철학을 가짐. 인터페이스가 간단하고, 브라우저 확장이 잘 작동하는 데다가, 비밀번호, 이메일, 캘린더를 하나의 암호화된 공간으로 묶을 수 있음.

#### 컴퓨트
디지털 오션 -> 스케일웨이(Scaleway)로 변경
- 이유: 스케일웨이는 유럽 기반의 클라우드 플랫폼이고, 깨끗한 인터페이스와 간단한 사용법이 특징임. 서버를 설정하고, 네트워크를 구성하는 게 쉬움. 또한, 이산화탄소 배출량을 고려하여 데이터 센터를 선택할 수 있기도 함.

#### 오브젝트 스토리지
아마존 웹 서비스 -> 스케일웨이로 변경
- 이유: 스케일웨이의 오브젝트 스토리지는 S3 호환되고, 기존 코드와 설정을 거의 변경하지 않아도 쉽게 이전 가능함.

#### 오프사이트 백업
백블레이즈 -> OVH클라우드로 변경
- 이유: OVH는 유럽 최대 클라우드 제공사로, 안정적이고 가격 경쟁力的 서비스를 제공함. 오브젝트 스토리지를 백업 목적으로 사용할 수 있고, 라이프사이클 규칙을 설정하면 오래된 백업을 저렴한冷 저장로 이동시킬 수 있음.

#### 트랜잭셔널 이메일
센드그리드 -> 렛터민트(Lettermint)로 변경
- důvodu: 렛터민트는 유럽 기반의 트랜잭셔널 이메일 서비스이고, 깨끗한 인터페이스와 간단한 사용법을 가짐. 메일 전송은 안정적이고, API는 깔끔함. 

#### 에러 트래킹
센트리 -> 자체 호스팅 버그싱크(Bugsink)로 변경
- 이유: 버그싱크는 자체 호스팅 에러 트래キング 툴이고, 센트리의 SDK를 사용해 거의 без손실적으로 이전 가능함. 다만, 기본적인 에러 로깅만 지원하고, 성능 모니터링이나 세션이랑 등의 고급 기능은 제공하지 않음.

#### AI API
오픈AI -> 미스트랄(Mistral)로 변경
- 이유: 미스트랄은 파리에 기반한 유럽 AI 제공사이자, 오픈 소스 모델을 제공하고, 깨끗한 API를 가지고 있음. 간단한 모델을 사용하는 경우에 적합하고, 돈이 유럽 기반의 회사로 가는 걸 의미Existential.

#### CDN(제외)
클라우드플레어 -> 제외
- 이유: 클라우드플레어는 미국 기반의 CDN이고, 사용자의 데이터가 이미 공개된 데이터이기 때문에 디지털 주권에 영향을 받지 않음. 
- 대안: 배니 CDN은 유럽 기반의 CDN으로, 좋은 평가를 받고 있지만 클라우드플레어의 기능 세트에 근접하지는 않음.

#### 결제(제외)
스트라이프 -> 아직 이전하지 않음
- 이유: 스트라이프는 아직 유럽 기반의 결제 서비스로 이전하지 않았음. 몰리(Mollie)는 네덜란드 기반의 결제 제공자로, 良い 선택일 수 있음.

#### AI 코드 어시스턴스(제외)
오픈AI 클라우드 -> 클로드 코드(Claude Code)
- 이유: 클로드 코드는 좋은 선택이었음. 미스트랄 비브(Mistral Vibe)는 아직 미흡해서 클라우드 코드를 선택했음. 어시스턴스 기능이 뛰어나고, 개발자들이 좋아하는 코드 편집기와 통합이 잘됨.
