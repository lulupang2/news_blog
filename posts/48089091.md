### 로컬에서 AI 모델 돌리기 🚀
   로컬에서 AI 모델 돌린다는 게 요즈음 핫하다고 하더라고. 그래서 나도 한 번 돌려봤음. 모델 선택하고, 설정값 건들여보고, 에러 고쳐보고... 정말 머리아프지만 재밌음 ㅋㅋㅋ. 
   ### 모델 선택하기 🤔
   모델은 Qwen 3.6 Q3, GPT-OSS 20B, Devstral Small 24B 등 여러 가지 중에 골라야 한다고 함. 그중에서 Gemma 4B만 괜찮았는데, tool use할 때 버벅거림. Qwen 3.5-9B (4b quant)가 가장 나은듯. 
   ### 설정값 조절하기 🔩
   temperature, top_p, top_k, min_p, presence_penalty, repetition_penalty 등 여러 가지 설정값 조절해야 하고, 생각 모드 켜고 끌 수도 있음. 설정값을 잘 조절해야만 제대로 돌아감. 
   ###思考 모드 켜기 🔓
   이렇게 생각 모드 켜고, Prompt Template에 `{%- set enable_thinking = true %}` 추가해야 생각 모드 활성화됨. 
   ### Pi, OpenCode 설정하기 📚
   Pi랑 OpenCode 두 가지 설정 방법이 있는데, 둘 다 설정값 조절하고, 에러 고치고 하면됨. 
   예를 들어, Qwen 모델로 Elixir linter credo 최신 버전으로 업그레이드하고싶은데, warning 나왔을 때 로컬 모델이 어떻게 도와주는지 보자. 
   Qwen: Credo Warnings There are 4 warnings in your test files. The issue is using length/1 to check if lists are non-empty. Credo recommends comparing against an empty list instead.
   => 이래가지고 warning 고치고, 코드 업그레이드해주면됨. 
   ### 마무리 😎
   로컬에서 AI 모델 돌리기 정말 재밌고, 유용한 기술임. 다만, 아직까지는 여러 가지 설정값 조절하고, 에러 고치고 해야 하니까 좀 귀찮을 수 있지만, 이런 기술이 점점 발전할 거라고 믿어. 
   그리고, 로컬에서 돌리니까 인터넷 연결 안 필요하고, 환경 비용도 줄어들고, 데이터센터 부담도 덜해지고,재밌는 거임. 꼭 해보라고 추천함. 개꿀
