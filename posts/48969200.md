### Lean과 형식 검증의 시작
Lean이란 함수형 프로그래밍 언어고, 정리 chứng明器라고도 불려요. 수학자들이 자기들끼리 증명을 쓰고 검토할 때Lean을 이용하면 검토하기 훨씬 수월해짐ㅋㅋㅋ

####Lean의 특징
- Lean은 순수 함수형 언어로, 프로그램에 사이드 이펙트가 없고 함수가 일급시민임.
- Lean을 이용하면 함수를 조각조각 나눠서 검증이 가능해지고, 협업하기에도 수월해짐.
- Lean에는 코드를 자동으로 완성시켜주는 기능도 있댔어

####One-Time Pad 암호화 프로토콜이란?
- One-Time Pad는 간단하게 말하면 암호화할 văn bản과 같은 길이의 키를 xor하면 됨
- 암호화된 문자열을 다시 같은 키와 xor하면 원래 문자열로 돌아옴

### BitString와 XOR를 정의해볼게요
- BitString를 정의하기 위해 Vector를 사용할 거에요
- Vector (ZMod 2) L를 사용하면 길이가 L인 비트열을 만들 수 있어요

####XOR 함수 정의
-Lean에서 함수를 정의할 때는 def를 사용해요.
- 함수를 정의할 때는 Implicit parameter를 사용할 수 있어요

```lean
def xor {L: ℕ} (x y: BitString L): (BitString L) := 
   Vector.zipWith (fun a b => a + b) x y 
```
