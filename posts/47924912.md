### SIMD 쿼드 알고리듬 등장
요즘에 대형 배열에서 값을 검색해야 하는 일이 종종 있잖아? 가장 간단한 알고리즘은 배열의 첫 번째 원소부터 последовательно 검색하는 방법인데, 이 것을 **선형 검색**이라고 해. C++의 `std::find` 함수가 대표적인 선형 검색 알고리듬임.

하지만 이런 알고리즘은 배열이 커질수록 검색 시간도 급격히 증가하기 때문에, **이진 검색**이라는 보다 효율적인 알고리듬을 많이 쓰게 되지. 이진 검색은 정렬된 배열에서 특정 값을 검색할 때 사용되는 알고리즘으로, 배열의 중간 값을 기준으로 검색 범위를 반복적으로 나누어 값을 찾는 방식이야. C++의 `std::binary_search` 함수가 대표적인 이진 검색 알고리듬임.

그런데, 요즘 CPU들은 **SIMD(Single Instruction, Multiple Data)** 명령어를 지원하고 있으니, 한번에 여러 값을 비교할 수 있어. 그래서 이진 검색보다 더 빠른 알고리듬을 만들 수 있을 거라는 생각이 들지 않아?

### SIMD 쿼드 알고리듬
SIMD 쿼드 알고리듬은 16비트 정수 배열에서 특정 값을 검색하기 위한 알고리듬으로, 이진 검색과는 달리 4개의 부분 배열로 나누어 검색을 진행해. 먼저, 배열을 16개의 원소로 구성된 블록으로 나눠. 각 블록의 마지막 원소를 기준으로 검색을 진행해.这样, 검색 범위를 4분의 1로 줄일 수 있어.

검색 범위를 좁혔으면, SIMD 명령어를 사용하여 16개의 원소를 한번에 비교해서 값을 찾을 수 있어. 배열의 길이가 16의 배수가 아니면, 남은 원소에 대해서는 선형 검색을 진행해.

### 성능 비교
SIMD 쿼드 알고리듬의 성능을比较하기 위해, 선형 검색과 이진 검색을 사용해서 같은 작업을 수행한 결과를 비교해 봤어. Apple M4와 IntelEmerald Rapids 프로세서에서 각각의 알고리듬을 수행한 결과를 비교한 것이지.

결과는 확실히 SIMD 쿼드 알고리듬이 최고였어. especialmente 큰 배열에서 검색을 수행할 때, SIMD 쿼드 알고리듬이 훨씬 빠르더라고. 결론은, SIMD 쿼드 알고리듬을 사용하면 이진 검색보다 더快速하게 배열에서 값을 검색할 수 있따는 거지.
