### RGB 값 Normalize하기

RGB 값을 Normalize할 때, 255랑 256 중에 뭐가 더 좋은 건가요? 
이 문제는 사실 이미지 프로세싱할 때 자주 등장하는 문제중 하나입니다. 
그렇다면 두 방법의 차이를 비교해 볼까요?

#### 1. 255로 나누기

이 방법은 가장 기본적인 방법입니다. 
RGB 값을 255로 나누면, 0 ~ 255의 범위를 0 ~ 1의 범위로 Normalize할 수 있습니다. 
예를 들어, RGB 값이 (128, 128, 128)이라면, Normalize后的 값은 (0.5, 0.5, 0.5)입니다.

```python
pixels = img / 255.0
result = process(pixels)
output = np.trunc(result * 255 + 0.5)
```

#### 2. 256로 나누기

이 방법은 255로 나누는 방법과 비슷하지만, 0.5를 더해줍니다. 
이 방법도 0 ~ 255의 범위를 0 ~ 1의 범위로 Normalize할 수 있습니다. 
예를 들어, RGB 값이 (128, 128, 128)이라면, Normalize后的 값은 (0.5, 0.5, 0.5)입니다.

```python
pixels = (img + 0.5) / 256.0
result = process(pixels)
output = np.trunc(result * 256)
```

#### 두 방법의 차이

두 방법의 차이는 0 ~ 255의 범위를 0 ~ 1의 범위로 Normalize하는 방법의 차이입니다. 
255로 나누는 방법은 0.5를 더해주지 않기 때문에, 0 ~ 255의 범위를 0 ~ 1의 범위로 Normalize하는 데에 더 자연스럽습니다. 
하지만, 256로 나누는 방법은 0.5를 더해주기 때문에, 0 ~ 255의 범위를 0 ~ 1의 범위로 Normalize하는 데에 더 편리합니다.

### 결론

두 방법의 차이는 0 ~ 255의 범위를 0 ~ 1의 범위로 Normalize하는 방법의 차이입니다. 
255로 나누는 방법은 0.5를 더해주지 않기 때문에, 0 ~ 255의 범위를 0 ~ 1의 범위로 Normalize하는 데에 더 자연스럽습니다. 
하지만, 256로 나누는 방법은 0.5를 더해주기 때문에, 0 ~ 255의 범위를 0 ~ 1의 범위로 Normalize하는 데에 더 편리합니다. 
따라서, 두 방법 중에 더 좋은 방법은 없습니다.
