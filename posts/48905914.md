### 소개
C++26에新增된 리플렉션 기능으로 더 효율적인 타입 이레이저를 구현하는 것이 가능한가? 기존에는 100줄이 넘는 코드를 手動으로 작성하거나 보일러 플레이트가 많이 필요한 라이브러리들을 사용해야 했지만, 이제는 C++26 리플렉션으로 더 쉽고 편하게 구현할 수 있게 된 것임.

### 기본 예제
```cpp
#include <rjk/duck.hpp>

struct [[=rjk::trait]] Container {
  auto size() const -> std::size_t;
  auto empty() const -> bool;
  auto clear() -> void;
};

rjk::duck<Container> c{std::vector<int>{1, 2, 3}};
c.size(); // 3
c = std::string{"hello"}; // swap underlying type at runtime
c.size(); // 5
c = std::map<int, int>{{1, 2}, {3, 4}};
c.empty(); // false
c.clear();
c.empty(); // true
```
이 예제를 보면, 우리는 인터페이스를 한 번 선언하고, 이미 작성했던 정의들이 나머지 작업을 모두 처리해 줌. 라이브러리는 단 하나의 헤더 파일로 제공되며, 소유권과 비소유권 semantic, 연산자, 인터페이스 composition, 기존 인터페이스 어댑터, 제3자 타입 확장 메소드 등을 제공함.

### C++26 리플렉션 기본
C++26 리플렉션은 `^^` 연산자를 사용하여 어떠한 것도 리플렉션할 수 있게 해줌. 예를 들어, `^^MyType`는 `MyType`에 대한 리플렉션을 반환함.
```cpp
constexpr inline struct{} trait{};
return std::ranges::any_of(annotations_of(^^MyType), [](std::meta::info annotation) {
  return type_of(annotation) == type_of(^^trait);
});
```
여기서 `annotations_of` 함수는 타입에 대한 어노테이션을 반환하며, `type_of` 함수는 어노테이션의 타입을 반환함.

### vtable 생성
duck의 핵심은 vtable을 생성하는 것임. vtable은 함수 포인터의 배열로, 인터페이스의 각 함수에 대한 함수 포인터를 저장함.
```cpp
template <is_trait... Traits>
struct vtable_generator {
  struct vtable;
  // ...
  consteval {
    std::vector<std::meta::info> members{ /* typeid, copy, move, destroy... */ };
    constexpr static std::array<std::meta::info, sizeof...(Traits)> traits{^^Traits...};
    template for (constexpr auto trait : traits) {
      // ...
    }
    define_aggregate(^^vtable, members);
  }
};
```
여기서 `define_aggregate` 함수는 vtable을 생성하며, `members` 배열에는 각 함수의 정보가 저장됨.

### 함수 호출
vtable을 통해 함수를 호출하는 것은 비교적 단순함.
```cpp
template <typename T, typename Invoker, typename Ret, typename... Args>
struct vtable_fn_maker {
  constexpr static auto erased_call(void* self, Args... args) -> Ret {
    auto* typed = static_cast<T*>(self);
    return std::invoke(Invoker{}, *typed, std::forward<Args>(args)...);
  }
};
```
여기서 `erased_call` 함수는 실제 함수 호출을 담당하며, `Invoker`는 함수 포인터를 invoke하는 데 사용됨.

### overload resolution
overload resolution은 C++의 핵심 기능 중 하나임. duck에서는 overload resolution을 다음과 같이 구현함.
```cpp
template <typename... Callables>
struct overload_set : Callables... {
  using Callables::operator()...;
};
```
여기서 `overload_set` 구조체는 여러 callable 객체를 inherit하며, 각 callable 객체의 `operator()`를 사용할 수 있게 함. 이로써, 함수를 호출할 때, 적절한 함수를 자동으로 선택할 수 있게 됨.

### 결론
C++26 리플렉션을 사용하면 더 효율적이고 편한 타입 이레이저를 구현할 수 있음. duck은 C++26 리플렉션을 사용하여 타입 이레이저를 구현한 라이브러리이며, 기존의 라이브러리들보다 더 쉽고 편하게 사용할 수 있게 해줌. 따라서, 더 빠르고 효율적인 프로그램을 작성할 수 있음.
