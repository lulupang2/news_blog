# UV는 개꿀인데 패키지 관리 UX는 개망임
UV의 아스트랄이라는 게 파이썬 세계를 휩쓸 정도로 좋는데, 그 이유는 존나 빠르고 파이썬 버전 관리가 개쉽고 여러 툴이 하나로 합쳐진 거임. 존나 간단하게 시작할 수 있음.

## 문제는?
하지만 존나 초기 세팅은簡單인데 유지 보수 단계에서 문제가 생김. 예를 들어서 패키지 업데이트 체크 같은 거. uv에서 이것저것 찾아보는데 개빡친다는 거임.

### pnpm vs uv
pnpm은 $ pnpm outdated라고 치면 깔끔하게 업데이트 필요 목록 나옴. uv는 $ uv tree --outdated --depth 1라고 존나ยาว게 쳐야함.输出도 존나 길어서 찾아야함. poetry也是 비슷한 문제임.

### UV는 버전 관리 개 븅신
UV의 경우 그냥 $ uv add pydantic 하면 pyproject.toml에 `dependencies = [ "pydantic>=2.13.4", ]` 추가됨. 개 븅신같은 부분은 상한선이 없다는 거임. pnpm이나 poetry는 이런 문제가 없음.

## 개선점
- $ uv outdated 같은 명령어 필요함.
- 업데이트 명령어 개선 필요함.
- 버전 관리 개선 필요함.

UV는 존나 빠르고 좋지만 유지 보수 단계에서는 개판이라는 거임. 개선이 필요함.
