# FUTO Swipe: 혁신적인 스와이프 타이핑 모델 
FUTO Swipe는 최신 오픈 소스 스와이프 타이핑 모델이야. 안드로이드 키보드 앱인 FUTO Keyboard에서 사용할 수 있고, 모델을 다운받아 직접 빌드해도 됨. 얘네는 서버 쪽 데모라서 웹페이지 용량을 작게 유지하기 위해 이렇게 해놓음. 실제 사용할 때는 기기에서 실행되니까 레이턴시도 낮고 좋음. FUTO Keyboard v0.1.29를 다운받거나 아래 자세한 정보를 확인해봐.
 
## 기존 스와이프 타이핑 문제점 
좋은 모바일 스와이프 타이핑은 항상 개인 정보 유출이 될 수 있는 키보드 앱이나 라이선스 없는 사적 라이브러리后에 빠져있었지. FUTO Swipe는 이런 문제를 해결하기 위한 오픈 소스 모델과 알고리즘이야. 얘네는 FUTO Keyboard를 위해 만들어졌지만, 더 넓은 커뮤니티에서도 사용하게 해줬음. 얘네가 오랜 시간 투자해서 만든 거라서 결론은 끝 пользователя에게 출처를 표시해주면됨. 라이선스를 읽어봐.
 
## 스와이프 데이터셋 공개 
 얘네는 100만 개의 스와이프 데이터셋을 MIT 라이선스로 공개했어. [HuggingFace](https://huggingface.co/)에서 확인해봐. 데이터셋을 검증하기 위해 기여해줘.
 
2024년 8월에 swipe.futo.org 도메인에서 데이터 수집 작업을 시작했어. 사용자가 자발적으로 мобाइल폰으로 웹페이지에 가서 데이터에 대한 정보를 제공 받고, 위키피디아에서 가져온 문장을 단어별로 스와이프 하라고 했음. 이렇게 하면 100만 개가 넘는 스와이프 데이터가 만들어졌고,质量가 낮은 작은 데이터셋을 걸러냄. 2025년 3월에 MIT 라이선스로 100만 개의 스와이프 데이터셋을 공개했어. 
이 데이터를 이용해서 모델을 훈련시키고, 다른 스와이프 타이핑 시스템을 평가했음.
 
## FUTO Swipe 구조 
3가지 모델이 있어. 
- 인코더 모델: 레이아웃과 언어에 구애받지 않고, 일반적인 스와이프 타이핑 예측을 함. 하지만 정확도는 최고가 아니야. 
- 문맥 언어 모델: 언어 하나당 작은 언어 모델을 훈련시켜서 예측의 품질을 높여줌. 이전 단어들에 앞서서 말도 안되는 단어들을 걸러내줘. 텍스트 데이터만으로도 학습이 가능해.
- 디코더 모델: 레이아웃과 언어에 özel한 모델로, 레이아웃의 특이성을 학습해서最高의 정확도를 달성해. 하지만 레이아웃과 언어 별로 학습해야 하니까, 지금은 영어 키보드 레이아웃만 제공하고 있 음. 3개의 모델과 빔 너비 300으로 설정하면 테스트 세트에서 탑4 실패 비율은 약 4%밖에 안 돼. 말도 안되는 단어들은 무시하면 오류율은 1% 밑이야. 벤치마크에 따라서 실제 사용할 때는 달라질 수 있지만, 얘네는 빅 테크 회사 키보드와 비교할만 한 성능이라고 믿어. 
인코더 모델은 635,140 개의 파라미터고, 디코더는 304,155개의 추가 파라미터야. 가장 큰 모델은 문맥 언어 모델로 1,500,000개의 파라미터인데, 거의 대부분은 임베딩이야. 이렇게 보면 총 2,494,767개의 파라미터가 있고, 1,364,271개의 활성 파라미터가 있어. 모델의 크기가 작아서, 저사양 기기에서도毫秒 단위로 실행할 수 있고, 환경적 비용도 매우 thấp해서 워크스테이션 GPU하나로도 충분했 음.
 
## 스와이프 라이브러리 
스와이프에서 단어 예측까지는 모델 예측만으로는 충분하지 않아. 사전 제한 레이턴시 빔 검색을 수행해서 여러 단어를 점수화하고 가장 가능성 높은 후보를 찾는게 필요해. 얘네는 스와이프 라이브러리를 제공해서 c++로 작성했는데, 간단하게 스와이프 경로에서 단어 예측까지 수행할 수 있 음. 
 VR이나 랩탑 트랙패드에서 스와이프 타이핑을 해보고 싶으면 [데모 링크](링크) 참고해. 
FUTO Swipe 모델은 FUTO 모델 라이선스이고, 추론 라이브러리는 GPL이야. 얘네는 훈련과 구조에 대한 논문을 준비중이라서, 나중에 확인해보라고 함. [HuggingFace 모델 라이브러리](https://huggingface.co/)도 참고해.
