# 소개
   최근에 사운드 블라스터 카타나 v2x라는 스피커를 사봤음. 사실 그냥 리눅스에서 스피커와 통신할 수 있는 툴을 하나 만들려고 했는데, 생각지도 못한 곳에서 취약점을 하나 발견함.
   # 취약점 원리
   카타나 V2X는 USB로 연결되는 사운드바고, 크리에이티브에서 스피커 설정을 바꿀 수 있는 앱을 제공함. 이때 사용하는 통신 프로토콜을 CTP라고 부름. 
   CTP 프로토콜 자체는 별로 복잡하지 않는데, 그냥 커스텀 프로토콜임. 
   CTP 프로토콜을 사용하려면 일단 인증을 해야하는데, 인증 키는 스피커에서 고정되어 있음. 
   # 펌웨어 분석
   펌웨어는 총 3가지 parte로 나눌 수 있음. FBOOT, FMAIN, CHK2. 
   FBOOT는 부트로더인데, 스피커를 복구모드에서 부팅하면 이 부분이 실행됨. 
   FMAIN은 일반적인 부트에서 실행되는 메인 펌웨어 부분임. 
   CHK2는 전체 펌웨어 컨테이너에附된 SHA-256 체크섬임. 
   # 공격 방법
   공격은 블루투스를 통해行함. 블루투스 프로토콜은 CTP 프로토콜과 통신 가능함. 
   그래서 블루투스를 잡은 후에 CTP 프로토콜을 사용해서 펌웨어를 업데이트할 수 있음. 
   # 결과
   결과적으로 스피커를 해킹해서 아무거나 할 수 있음. 근데 이건 사실 크리에이티브 측에서 보안을 제대로 안했기 때문임.
