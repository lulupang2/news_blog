### 모질라의 Prompt API 반대에 대한 입장
모질라가 크롬의 Prompt API에 빡친 것 같음. 근데 이게 왜 문제임? 
크롬에서_prompt API라는 것을 만들었는데, 이게 뭐냐면prompt가 들어올 때마다 알림창 같은 거 띄워주는 기능임. 
근데 모질라는 이 기능을 안 좋은 방향으로 가고 있다고 보고 반대하는 중임. 
[이슈 페이지](https://github.com/webmachinelearning/prompt-api/blob/main/README.md)에도 반대 이유를 설명하고 있음. 근데 설명을 읽어보니까 너무 어렵게 말해서 무슨 말인지 잘 모르겠음 ㄹㅇ.
### 모질라의 반대 이유
모질라 측은 Prompt API를 사용하는 것에 대해 크롬만의 안 좋은 Influence를 가할 수 있다고 봄. 
개돼지 같은 크롬만의 표준을 만드는 건 안 된다고 주장하고 있음. 
이 외에도 WebKit도 크롬의 Prompt API에 대해 비슷한 반대 의견을 냈음. 
근데 블링크에서 이 기능에 대한 Intent-to-prototype를发布했었는데, 이 부분이 모질라가 빡친 주因인 것 같음. 
개빡치는 점은 Prompt API가 실제로 사용되기 시작하면, 웹 표준을 지키는 건 어려워질 거라는 점임. 
이미 크롬의 점유율이 너무 높아서, 표준을 크롬이 정하는 상황이 발생할 가능성이 높다고 봄. 
### 결론
모질라는 Prompt API에 반대하는 중이고, 이에 대한 의견을 수렴하고 있음. 
웹 개발자들 사이에서 Prompt API의 필요성과 표준화의 필요성에 대한 논의가 많이 이루어지겠다고 봄. 
그럼 여기까지였음. Prompt API에 대한 더 많은 정보는 [이슈 페이지](https://github.com/webmachinelearning/prompt-api/blob/main/README.md)에서 확인할 수 있음.
