# 포기그라운드(Postgres) 서바이벌 가이드

안녕하세요 하체트의 알렉산더 벨랑게입니다. 최근 반년 동안 저는 포기그라운드의 두년간의 전투를 담은 문서를 작성했습니다. 저는 이 문서가 다른 사람들에게 도움이 될 것 같아서_feedback과 함께 공유하기로 결정했습니다.

포기그라운드는 공식 문서가 너무 방대합니다. 그래서 상황이 안좋아 졌을 때 문서를 참고하기 어려울 때가 많습니다. 문서의 시작점은 Sql의 기본적인 개념, 행, 열, 테이블, 인덱스 등입니다.

## 기본적인 사항: 읽기, 쓰기, 스키마

스키마는 변경하기 어려우므로 신중하게 설계해야 합니다. 저의 스키마 설계 규칙은 다음과 같습니다.
 * 아이덴티티 열 또는 빌트인 UUID를 사용합니다.
 * 타임스탬프를 사용합니다.
 * 프라이머리 키를 사용합니다.
 * 외래 키를 사용합니다.

읽기 질의에 대해서는 다음을 참고하세요. 포기그라운드는 기본적으로 인덱스를 사용하여 데이터를 찾습니다. 인덱스를 사용하면 데이터를 빠르게 찾을 수 있습니다.但是, 인덱스를 사용하지 않으면 데이터를 순차적으로 검색해야 하므로 느립니다.

쓰기 질의에 대해서는 다음을 참고하세요. 트랜잭션을 短時間으로 유지하세요. 트랜잭션 중에 외부 서비스를 호출하지 마세요. 잠금을 최소화하세요.

## 중급적인 사항: 쿼리 플래너, 벌크 업데이트, 오토바キュ룸

쿼리 플래너는 내부 구현으로, 사용자에게 노출되지 않습니다. 그러나 쿼리 플래너의 동작을 이해하면 효율적인 쿼리를 작성하는 데 도움이 됩니다. 쿼리 플래너는 쿼리를 분석하여 내부 연산으로 변환합니다. 쿼리 플래너는 인덱스를 사용하여 데이터를 찾을 수 있습니다.

오토바キュ룸은 기본 설정으로 설정할 수 있습니다. 오토바キュ룸은 자동으로 데이터베이스를 청소하여 효율성을 유지합니다.
