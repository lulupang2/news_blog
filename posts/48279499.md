### CPU랑 GPU랑 플로팅 포인트 처리가 다르다고? ㅋㅋㅋ
   플로팅 포인트 숫자에 대해 알아보고 있었는데, CPU랑 GPU랑 다르게 작동하는 경우가 있었음. 
   이게 무슨 소리냐면, 플로팅 포인트는 부동소수점이라는 형식으로 숫자를 처리하는 거야. 
   근데, 이 숫자가 매우 작으면 denormalized 숫자라고 하는데, 이 경우에 CPU와 GPU가 다르게 처리함.
   ### denormalized 숫자는 뭘까?
   denormalized 숫자는 매우 작은 숫자로, 일반적으로 1.18 * 10^-38 미만의 숫자를 의미해. 
   이 숫자는 특별한 형식으로 저장되고, CPU와 GPU가 다르게 처리해. 
   CPU는 이 숫자를 preserve하는데, 반면에 GPU는 flush하거나 preserve할 수 있어. 
   ### 예를 들어 보자
   -1.175493930432748e-38이라는 숫자를 floor 함수로 처리하면, CPU는 -1.0을 반환하지만, 
   GPU는 -0.0을 반환할 수도 있어. 
   이는 플로팅 포인트를 처리하는 방식의 차이 때문이야.
   ### 그러면 이걸 어떻게 해결하느냐?
   해결 방법은 simple 하게, custom 함수를 만들어서 denormalized 숫자를 처리하는 거야.
   예를 들어, DeterministicFloor 함수와 DeterministicCeil 함수를 사용할 수 있어.
   ```hlsl
float DeterministicFloor(float x) { 
   if ((asuint(x) & 0xFF800000u) == 0x80000000 && // sign = 1, exponent = 0 
       (asuint(x) & 0x007FFFFFu) != 0) // mantissa != 0 
   { 
       return -1.0f; 
   } 
   return floor(x); 
} 

float DeterministicCeil(float x) { 
   if ((asuint(x) & 0xFF800000u) == 0 && // sign = 0, exponent = 0 
       (asuint(x) & 0x007FFFFFu) != 0) // mantissa != 0 
   { 
       return 1.0f; 
   } 
   return ceil(x); 
}
   ```
   이 함수들은 denormalized 숫자를 올바르게 처리하도록 해줘. 
   그러니까, CPU랑 GPU랑 플로팅 포인트를 다르게 처리하는 경우가 있다는 걸 알아두면,
