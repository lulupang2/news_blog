### asm.js는 이제 歴史吧
어제부터 파이어폭스 148버전에선 asm.js최적화가 기본적으로 꺼져있음. 그냥 끈거랑 별반 다를바がない긴 한데, asm.js쓸때랑 비슷하게 코드가 작동하는건 맞지. 
### 왜 이렇게 한거냐?
 asm.js는 원래 네이티브 속도로 코드가 작동하게 해주는거였는데, 웹어셈블리가 이제 더 성능 좋고 효율적인 기술이니까 폐지하기로 한 거임. 
웹어셈블리 쪽으로 이사가면 더 빠르고 더 작은 바이너리가 생성됨. 개꿀임.
### asm.js의 역사
asm.js는 파이어폭스 22에서 처음 나온 기술이었고, 유니티나 언리얼같은 프로젝트들이 웹으로 코드를移植할 수 있었음. 
 asm.js는 웹 어셈블리의 시초가 된 기술이기도 하니까, asm.js없었으면 웹 어셈블리도 없었을 가능성이 높음. 
웹 어셈블리가 나오기 전에는 asm.js가 네이티브 속도로 코드를 작동시키는 유일한 방법이었음.
### 왜 이제 폐지하는 거야?
 웹 어셈블리가 더 성능 좋은 기술이고, asm.js는 이제 더 이상 필요없으니깐废지시키는 거야.
웹 어셈블리로 코드를 재컴파일하면 더 빠르고 더 효율적이라는 뜻임.
웹 어셈블리 쪽으로 가면 更好的 미래가 기다린다고 생각하니까, asm.jsを使っている 개발자들은 지금부터웹 어셈블리로 코드를 옮겨놔.
