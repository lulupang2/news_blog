### FPS 카운터 구현 방법 정리
요즘 게임 개발자들 사이에서 FPS 카운터 구현 방법을 어떻게 해야 좋을지 많이 논란이 있음. 그래서 오늘은 이 vấn đề에 대해서 정리해 볼께.
#### Wrong한 방법
일단 첫번째 방법은 latest frame 기반의 방법임. 이건 너무 간단해서 코드로 설명하기도 귀찮지만 예를들면
```c
float fps = 0; 
Time prev = Time::current(); 
while (true) { 
    // 게임 로직 처리 
    Time curr = Time::current(); 
    fps = Duration::seconds(1) / (curr - prev); 
    prev = curr; 
}
```
이 방법은 최신 프레임의 처리 시간을 基準으로 하所以 FPS를 계산하는건데, 이렇게하면 일시적으로 빠른 프레임이나 느린 프레임이 발생해도 즉시 반영이 되버림. 
다음 방법은 특정 수의 latest frames 기반의 방법임. 예를들면
```c
const int windowFrames = 5; 
float fps = 0; 
Queue<Duration> processingTimes; 
Time prev = Time::current(); 
while (true) { 
    // 게임 로직 처리 
    Time curr = Time::current(); 
    if (processingTimes.size() == windowFrames) 
        processingTimes.pop(); 
    processingTimes.push(curr - prev); 
    fps = Duration::seconds(1) / averageVal(processingTimes); 
    prev = curr; 
}
```
이 방법은 특정 수의 프레임을基準으로 하所以 FPS를 계산하는건데, 이 방법은 window size를 조절하면서 다양한 결과를 얻을 수 있긴 한데, 각 프레임의 처리 시간이 서로 다르면 그래프가 이상하게 나와서 사용하기 어려울 수 있음.
#### OK한 방법
그 다음은 1초간격으로 리셋하는 방법임. 예를들면
```c
float fps = 0; 
int frames = 0; 
Time prev = Time::current(); 
while (true) { 
    // 게임 로직 처리 
    Time curr = Time::current(); 
    if (prev + Duration::seconds(1) < curr) { 
        fps = frames; 
        frames = 0; 
        while (prev + Duration::seconds(1) < curr) { 
            prev += Duration::seconds(1); 
        } 
    } 
    ++frames; 
}
```
이 방법은 1초마다リ셋하는 방식인데, 이건 간단하면서도 제대로된 방법임.
#### Right한 방법
그다음은 rolling window 기반의 방법임. 예를들면
```c
const Duration window = Duration::seconds(1); 
float fps = 0; 
Queue<Time> frameTimestamps; 
while (true) { 
    // 게임 로직 처리 
    Time curr = Time::current(); 
    frameTimestamps.push(curr); 
    while (frameTimestamps.next() + window < curr) 
        frameTimestamps.pop(); 
    fps = frameTimestamps.size() * Duration::seconds(1) / window; 
}
```
이 방법은 rolling window 기반으로 FPS를 계산하는 방식인데, 이건 제대로된 방법임. 
이 방법은 queue에 latest frame timestamp를 계속 넣으면서, queue에 있는 가장 오래된 프레임이 현재 시간에서 window 사이즈보다 더 오래된 경우 queue에서 제거하게 되는데, 그렇게하면 queue에 있는 프레임의 수로 FPS를計算할 수 있음.
이 방법은 rolling window의 사이즈를 조절하면서 다양한 결과를 얻을 수 있고, 게임의 성능을 보다 자세하게 분석할 수 있음.
그리고 이 방법은 queue에 frame timestamp를 넣을 때, queue의 사이즈를 제한할 수 있음. queue의 사이즈를 제한하면 memory를節約할 수 있고, 게임의 퍼포먼스를 개선할 수 있음.
#### 결론
FPS 카운터 구현 방법은 다양한 방법이 있지만, rolling window 기반의 방법이 가장 제대로된 방법임. rolling window의 사이즈를 조절하면서 다양한 결과를 얻을 수 있고, 게임의 성능을 보다 자세하게 분석할 수 있음. 그리고 queue에 frame timestamp를 넣을 때, queue의 사이즈를 제한하면 memory를節約할 수 있고, 게임의 퍼포먼스를 개선할 수 있음. 
이 방법을 사용하면, 게임의 성능을 개선하고, 사용자 experiencia를 향상시킬 수 있음.
