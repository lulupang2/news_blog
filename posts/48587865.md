# DGX 스팍에 두 개의 퀜3 모델 올리기

-devashish.me에 가입하고 최신 글을 받아보세요!- 

오늘은 DGX 스팍에 두 개의 퀜3 모델을 올리기 위해 필요한 수학을 계산하고, gpu_memory_utilization 트랩에서 벗어나기 위한 방법을 설명하려고 함. 일단 시작해 볼게요.

최근에 제 에이전트 스택은 험레스로 돌아가고, 모델은 DGX 스팍에서 돌아감. 이게 의도적인 분리인 거야. 워크스테이션이 반응이 잘되고, 스팍이 GPU 작업을负责해주고, 둘 다 HTTP 프록시로 통신하니까.

클로리움으로 에이전트군을 관리하는 순간부터, 험레스의 수가 올라감. 에이전트 수가 많아지고, 호스트도 많아지고, 동시에 트래픽이 올라가면 스팍에 모델이 하나만 있는 상황이 되어버리잖아. 이때부터 모델 서버가 하나만 돌아가는 상황이 되어버리면 서버가 하나만 돌아가는 상황에서 처리할 수 있는양이 아니라서 문제가 생기게됨.

이거 때문에 퀜3 모델을 업그레이드하고, vLLM을 사용해서 gpu_memory_utilization을 제어하는 방법을 찾아봤어. KV 캐시를 관리하는 방법도 찾아봤고, PagedAttention을 사용해서 KV 블록을 재사용하는 방법도 찾아봤어. 이 방법들은 다 해결책이 되었어.

하지만 DGX 스팍에 두 개의 퀜3 모델을 올리기 위해서는 gpu_memory_utilization을 정밀하게 계산해야 해. gpu_memory_utilization은 총 gpu 메모리의 일부라는 거 잊지 마! free 메모리가 아니라 총 메모리야! 그래서 두 개의 모델을 동시에 올리기 위해서는 gpu_memory_utilization을 조절해야 해. gpu_memory_utilization을 조절할 때는 총 메모리에 대한 비율을 조절해야 해.

이제 gpu_memory_utilization을 계산하는 방법을 설명하겠어. 우선 실제 메모리 사용량을 측정해야 해. nvidia-smi 명령어를 사용해서 현재 gpu 메모리 사용량을 측정할 수 있어. 이 값을 사용해서 gpu_memory_utilization을 계산할 수 있어.

gpu_memory_utilization = (총 메모리 - 사용 메모리) / 총 메모리

이 계산을 통해 gpu_memory_utilization을 정밀하게 계산할 수 있어. 이 계산은 매우 중요한 거야. 이 계산을 통해 두 개의 모델을 동시에 올릴 수 있어.

마지막으로, gpu_memory_utilization을 계산하는 방법이 아니고, 실제로 DGX 스팍에 두 개의 퀜3 모델을 올리기 위한 방법을 설명하겠어._liteLLM을 사용해서 두 개의 모델을 동시에 올릴 수 있어._liteLLM은 gpu_memory_utilization을 제어할 수 있기 때문에, 두 개의 모델을 동시에 올릴 수 있어.
