### 디istributed 시스템 테스트 자동화하는 AI 에이전트 쓰는 법
디istributed 시스템 테스트하는건 개빡침. 그래서 AI 에이전트를 사용해서 자동화 해보려고 함 ㅋ. 
AI 에이전트는 디자인과 실행 두가지 스킬을 가지고 있는데, 디자인은 테스트 계획을 만들고 실행은 테스트를 실행하는 역할임. 
이 두 스킬은 테스트 계획과 결과 리포트를產生하는데, 리포트에는 버드 verdict와 SUT/harness/checker/environment 블레임 분류가 포함됨. 
리뷰어가 리포트를 읽고_ship quyết정을 내림. 
스킬은 SKILL.md 파일로 되어있고, 에이전트가 실행함. 계획과 결과 리포트는 아웃풋임. 
계획에서는 클레임에서 시작해서 가설을 생성하고 시나리오를 생성하고, 각 시나리오는 클레임 하나당 하나의 테스트 케이스를 만들고coverage adequacy argument와 confidence statement를 생성함. 
실행에서는 클레임을 검증하고, 결과 리포트를 생성함. 
에이전트는 Claude Code, Codex, Copilot CLI, Cursor, Gemini 등등을 사용할 수 있고, INSTALL.md 파일을 사용해서 설치하고 설정할 수 있음. 
INSTALL.md 파일은 에이전트가 설치와 설정을 도와주는데, 에이전트가 INSTALL.md 파일을 읽고 설정을 자동으로 해줌. 
에이전트를 사용해서 테스트 계획을 만들고 실행하고, 결과 리포트를 확인할 수 있음. 
이 방법으로 디istributed 시스템 테스트를 자동화할 수 있음 ㅋ. 
### 설치 방법
에이전트를 설치하려면 INSTALL.md 파일을 읽고 설정을 따라야함. 
설치 후에 테스트 계획을 만들고 실행하고, 결과 리포트를 확인할 수 있음. 

### 사용 방법
에이전트를 사용해서 테스트 계획을 만들고 실행하는 방법은 두 가지임. 
첫번째는 causal ask 방법이고, 두번째는 USAGE.md 파일에서 제공하는 방법임. 
causal ask 방법은 에이전트에게 테스트 계획을 만들라고 요청하는 방법이고, USAGE.md 파일에서 제공하는 방법은 에이전트에게 테스트 계획을 만들고 실행하는 명령어를 제공하는 방법임. 
### 디distributed 시스템 테스트 자동화의 장점
디distributed 시스템 테스트 자동화의 장점은 테스트 시간을 줄이고, 테스트의 정확도를 높일 수 있음. 
또한, 에이전트를 사용해서 테스트를 자동화하면 humano의 실수를 최소화 할 수 있음. 
그러나, 에이전트를 사용해서 테스트를 자동화하려면 에이전트를 설치하고 설정해야 하고, 에이전트를 사용해서 테스트 계획을 만들고 실행해야 함.
