### 클로드에서 개인 정보 인증 이제 필수야 ㄹㅇ
   클로드에서 이제부터 사용자 인증이 필수임. 왜냐면 강력한 기술을 사용할 때 responsable하게 사용자 정보를 확인해야 하거든. 
   사용자 인증은 남의 사용자 정보를 도용하는 걸 막고, 사용자 규칙을 제대로 지키게 하고 법적 의무를 지키는 데 필요해.
   클로드에서는 일부 사용례에 대해 사용자 인증을 도입하고, 안전하고 법적 의무를 지키는 데 필요한 기능에 접근할 때 사용자 인증 확인을 요청할 수 잇음.
   우리가 사용자 인증 데이터를 사용하는건 오직 사용자 신원만을 확인하는데 사용할 뿐임. 
   **인증 방법**은 persona identities 기술을 사용함. 이 기술을 사용한 이유는 기술의 강력함과 프라이버시 보호, 보안 안전장치가 뛰어나기 때문임.
   ### 인증 절차
   1. 인증 시작하기 전에 정부 발급한 신분증명서(여권, 운전면허증 등)을 준비하고
   2. 카메라가 있는폰이나 컴퓨터를 준비하고
   3. 5분 정도 준비해둠.
   **수락되는 신분증명서 종류**는 원본, 물리적인 정부 발급 신분증명서(여권, 운전면허증, 국가 신분증명서 등)임.
   **거절되는 신분証명서 종류**는 복사본,화면 캡처, 스캔본, 사진 등임.
   ### 사용자 데이터 보호
   우리는 사용자로부터 신분증명서를 요구하는 것이 의미가 있는 요청임을 알고, 이 과정을 사용자 정보를 보호하는 데에 최선을 다하고 임.
  Anthropic 은 사용자 인증 데이터의 데이터 컨트롤러임. 
  Anthropic은 Persona 에게 사용자 인증 데이터를 처리하도록 맡기고, Persona 는 Anthropic 의 지시를 받아 사용자 인증 데이터를 처리하고, Anthropic 은verification 기록에 접근할 수 잇지만, 본인 서버에 저장하지는 않음.
   Persona 는계약에 의해 사용자 데이터를 사용할 수 잇는 범위가 제한되어 잇고, 사용자 데이터는 사용자 인증을 제공하고, 사기 예방 능력을 향상하는데만 사용할 수 잇음.
   Persona 는 산업 표준의 보안 통제를 준수해야 하며, 법적 요구에 따라 데이터를 삭제해야 함.
   모든 데이터는 전송 중과 저장 중에 암호화되어 잇음.
   자세한 내용은 클로드의 [개인 정보 보호 정책](https://www.claude.ai/privacy-policy)을 확인해라.
   ### 사용자 인증 실패시
   사용자 인증이 실패할 수 잇는 이유는 여러가지가 잇음. 사진이 불분명하거나, 문서가 읽을 수 없거나, 기술적인 문제 등이 이유가 될 수 잇음.
   사용자 인증이 실패하면, 다시 시도해봐. 대부분의 실패는 사진을 다시 찍거나, 다른 정부 발급 신분증명서를 사용하면 해결될 수 잇음.
   사용자 인증이 여전히 실패하면, 클로드의 고객 지원에 문의해라.
   ### 사용자 인증과 계정 정지
   클로드에서는 사용자 인증을 통해 사용자 계정을 안전하게 관리함.
   계정 정지는 사용자 인증과 관련이 잇을 수 잇음. 
   계정 정지 이유는 여러가지가 될 수 잇지만, 사용자 규칙을 위반하거나, 사용자 인증을 거부한 경우 등이 이유가 될 수 잇음.
   계정이 잘못된 이유로 정지되었다고 생각하면, 클로드의 고객 지원에 문의해라.
   사용자 인증과 관련된 질문이 있으면, 클로드의 고객 지원에 언제든지 문의할 수 잇음.
