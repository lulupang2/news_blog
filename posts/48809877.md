### GLM 5.2
GLM 5.2를 Z.ai에서 지난 두 주 동안 사용해 봤음. GLM 5.2는 오푸스와 GPT에 진짜 경쟁할 수 있는 오픈 웨이트 모델이라고 생각해. 정말로 좋고, 난 오푸스와 GLM 5.2를 구별하기 어려웠음. 단점은 느려.interactive агент任务에선 문제가 되지만, 비interactive任务에선 전혀 문제가 안돼. 또한 비전 지원이 없어서, 이미지 기반 PDF, 스크린샷, 디자인 파일 등을 읽을 수 없어서真的 아쉬웠음.

 둘째, 웹 검색 기능이 부족해. 거의 모든 에이전트 세션이 웹 검색을 많이 하거든. Z.ai의 MCP가 있지만, خیلی 나쁨. фай어웍스에선 아직 이런 기능이 없다고回答 받았음. 일단 CLI 기반 웹 검색ツール을 사용하라고 말하더라. 이건 정말 약점이라고 생각해. 좋은 웹 검색 API는 엄청난 기회라고 생각함.

### Drop-in Replacement
GLM 5.2를 사용하는 방법은非常 간단해. Z.ai와 Fireworks는 오픈AI와 앤트로픽 호환되는 엔드포인트를 제공해. Claude Code와 Codex에서 설정만 바꿔주면 끝임. Anthropic이 recent에 클라우드 非_interactive에 대한 API 비용을 발표하고撤回했지만, GLM 5.2를 대신 사용할 수 있다고 생각해. 비전과 속도가 느려서 약점이지만, 진짜 오픈AI를 쓰고 있는지 구별하기 어려울 정도임.

### 비용节约

Z.ai의 '코딩 플랜' 구독도 있음. 이것은 오픈AI와 앤트로픽의 플랜을 모방해서 만든 거지만, 사용량 제한이 更宽裕해. 다만, 훈련과 데이터 보관 정책이 매우 느슨해서, 엔터프라이즈에선 사용하기 힘들 거라고 생각해. 만약 오픈AI와 앤트로픽이 가격을 크게引き上げ면, 이것은 信용할 수 있는 선택지가 될 수 있을 거야. 앞으로 GLM 5.2의 가격이 더 낮아질 거라고 생각해.
