### Silk: 리눅스용 협동식 파이버 스케줄러

Silk는 리눅스용 협동식 파이버 스ケ줄러로, 협동식 스케줄링을 지원해줌. io_uring 통합 및 작업窃取에 최적화 된듯. 파이버는 가벼운 栈을 가진 코루틴으로, OS 쓰레드를 블록시키지 않고 대기상태로 전환해서 높은 동시성을 가지면서 낮은 오버헤드를 제공할 수 있음.

#### 문서

* scheduler: [scheduler	loop, context switching, suspension pattern, async IO, sleep cancellation, work-stealing design, and performance benchmarks](https://github.com/ClickHouse/silk/blob/main/docs/scheduler.md)
* sync: [synchronization primitives: FiberFuture, FiberFutex, FiberMutex, FiberSequencer, FiberEvent, FairFiberMutex](https://github.com/ClickHouse/silk/blob/main/docs/sync.md)
* util: [utility library: lock-free data structures, TSC timing, memory pool, CPU topology, logging, assertions](https://github.com/ClickHouse/silk/blob/main/docs/util.md)
* perf: [net-perf and file-perf benchmark results and fio comparison](https://github.com/ClickHouse/silk/blob/main/docs/perf.md)
* coroutines: [stackless coroutines vs stackful fibers: design differences and performance data](https://github.com/ClickHouse/silk/blob/main/docs/coroutines.md)

#### 예제

* src/fibers/tests/: [usage examples: fiber lifecycle, futures, synchronization primitives, async IO](https://github.com/ClickHouse/silk/tree/main/src/fibers/tests)
* src/gdb/fiber.py: [GDB extension; load with source src/gdb/fiber.py, then use fiber-list, fiber-savecontext, fiber-restorecontext, fiber-switchcontext](https://github.com/ClickHouse/silk/blob/main/src/gdb/fiber.py)

#### 빌드 요구 사항

* CMake >= 3.28
* Ninja
* Clang 21
* ccache (optional)
* Boost headers (libboost-dev)
* libelf (libelf-dev) — optional, required only for src/profiler; the profiler is silently skipped if absent.

#### 빌드 명령어

* ./bb [options] [command]: Global options
 + -b, --build: debug, release
 + -s, --sanitizer: thread, address, memory, undefined
 + -v, --verbose: Print every command before running it; also passes --verbose to perf binaries to enable their debug logging
* Commands
 + configure: Configure (or reconfigure) the CMake build directory.
 + fmt: Format all source files with clang-format-21.
 + clean: Remove the entire build/ directory.
 + build: Build the project.
 + test: Build and run tests.
 + bench: Build and run benchmarks.

#### 성능 예제

* file-perf: Async file I/O benchmark using io_uring.
* fio-perf: fio comparison using io_uring engine.
* net-perf: TCP echo benchmark.
* net-perf-asio: TCP echo benchmark using Boost.Asio C++20 coroutines.
* http-perf: HTTP/1.1 GET benchmark.
* s3-perf: S3 object storage benchmark.
