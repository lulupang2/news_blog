### 엘릭서(Elixir)에서 최고의 랜덤 가중치로 노드 관리하기

디씨 인사이드 개발자 갤러리에서 볼 법한 썰 풀겠습니다. 엘릭서에서 분산 노드 관리를 위한 알고리즘 중 하나인 **고정 해싱(Consistent Hashing)**에 대해 살펴보고, 이를 구현하는 **ExHashRing**라이브러리와 **랜덤 가중치 최고값(Rendezvous Hashing)** 알고리즘을 비교해 보겠습니다. 

#### 고정 해싱(Consistent Hashing)

고정 해싱은 분산 노드에서 데이터를 균등하게 분배하는 기술입니다. 엘릭서에서 이는 노드 간의 데이터 분배를 위해 널리 사용됩니다. 노드들이 클러스터를 구성하고, 각 노드가 특정 키를 관리하게 됩니다. 

#### ExHashRing

ExHashRing은 엘릭서용 고정 해싱 라이브러리입니다. 이 라이브러리는 노드 간에 키를 분배하여, 각 노드가 키를 관리할 수 있도록 지원합니다. ExHashRing은 성능이 우수하고 안정적이지만, 노드를 추가 atau 제거할 때 상태를 관리해야 하는 단점이 있습니다.

#### 랜덤 가중치 최고값(Rendezvous Hashing)

랜덤 가중치 최고값 알고리즘은 노드를 추가 또는 제거할 때 노드 간의 상태를 관리할 필요가 없습니다. 이 알고리즘은 키와 노드에 해싱 함수를 적용하여, 키와 노드의 조합에서最高의 가중치를 갖는 노드를 키의 소유자로 정합니다. 이는 함수형 프로그래밍으로 구현이 가능하여, 매우 간단하고 효율적입니다.

```elixir
defmodule HRW do
   def owner(key, nodes) do
     Enum.max_by(nodes, fn node -> :erlang.phash2({key, node}) end)
   end
end
```

#### 성능 비교

ExHashRing과 랜덤 가중치 최고값의 성능을 비교해 보았습니다. 노드 수를 14개로 설정했을 때, 두 알고리즘의 성능은 유사했습니다. 하지만 노드 수를 10,000개로 증가시키면, 랜덤 가중치 최고값 알고리즘은 성능 저하가 발생했습니다. 

#### 개선된 랜덤 가중치 최고값

#### 분산 성능

분산 성능을 비교해 보았습니다. ExHashRing은 노드 수가 증가해도 안정적인 성능을 발휘했지만, 랜덤 가중치 최고값 알고리즘은 노드 수가 증가할수록 성능이 저하되었습니다. 

### 결론

ExHashRing과 랜덤 가중치 최고값 알고리즘은 엘릭서에서 분산 노드 관리를 위한 유용한 도구입니다. 성능과 안정성을 고려하여, 적절한 알고리즘을 선택하여야 합니다.
