### 텔레그램 서버리스: 서버 걱정 말고 코드만 집중해
텔레그램에서 백엔드 코드를 돌리려면 서버를 프로비저닝하고, 컨테이너를 유지하고, 스케일링도 신경 써야했음. 그런데 이제 텔레그램 서버리스가 나오면서 코드만 집중하면 된다는 거임 ㅋ

#### 서버리스란?
서버리스는 서버를 쓰지 않고 코드를 실행할 수 있는 방식임. 텔레그램에서 백엔드 코드를 돌리려면 텔레그램 인프라에서 코드를 직접 돌릴 수 있게 해줌. 그러면 서버를 프로비저닝하거나, 컨테이너를 유지할 필요가 없고, 스케일링도 신경 쓰지 않아도 됨.

#### 어떻게 작동하는 걸까?
 텔레그램 서버리스는 자바스크립트 모듈로 작성된 코드를 받아서 텔레그램의 V8 샌드박스에서 실행해 줌. 그러면 코드가 Bot API와 내장 데이터베이스에 직접 접근할 수 있게 됨.

#### 왜 사용해야 할까?
 텔레그램 서버리스는 백엔드 코드를 간단하게 배포하고, 관리할 수 있게 해줌. 텔레그램에서 제공하는 SDK와 데이터베이스를 사용하면 코드를 더 간단하게 작성할 수 있음.

### 시작하는 법
#### 프로젝트 생성
 텔레그램 서버리스 프로젝트를 생성하려면 `npm create @tgcloud/bot` 명령어를 사용하면 됨. 그러면 텔레그램 서버리스 프로젝트가 생성되고, 필요한 의존성이 설치됨.

#### botfather에서 서버리스 활성화
 텔레그램에서 서버리스를 활성화하려면 botfather에게 가서 서버리스를 활성화해야 함.

#### 로그인
 텔레그램 서버리스에 로그인하려면 클라이언트 비밀번호를 받은 다음, `npx tgcloud login` 명령어로 로그인하면 됨.

### 데모 봇 만들어보기
 텔레그램 서버리스에서 데모 봇을 만들어보려면 `schema.js` 파일에 데이터베이스 테이블을 정의하고, `handlers/message.js` 파일에 메시지 처리 로직을 작성하면 됨. 
```javascript
// schema.js
import { table, integer } from 'sdk/db';

export const counters = table('counters', {
  chatId: integer('chat_id').primaryKey(),
  seen: integer('seen').notNull().default(0),
});
```

```javascript
// handlers/message.js
import { api, db } from 'sdk';
import { counters } from 'schema';

export default async function (message) {
  const chatId = message.chat.id;
  const [row] = await db.insert(counters)
    .values({ chatId, seen: 1 })
    .onConflictDoUpdate({ target: counters.chatId, set: { seen: db.sql`${counters.seen} + 1` } })
    .returning()
    .run();
  await api.sendMessage({ chat_id: chatId, text: `Hell오! 이 봇은 ${row.seen}개의 메시지를 받았어.` });
}
```
#### 배포
 텔레그램 서버리스에 코드를 배포하려면 `npx tgcloud push` 명령어를 사용하면 됨. 그러면 코드가 텔레그램 서버리스에 배포되고, 봇이 작동하기 시작함.

#### 데이터베이스 마이그레이션
 데이터베이스 스키마를 변경하려면 `npx tgcloud migrate` 명령어를 사용하면 됨. 그러면 데이터베이스가 변경되고, 봇이 다시 작동하기 시작함.

### 서버리스 장점
#### 서버 걱정말고 코드만 집중
 텔레그램 서버리스를 사용하면 서버 걱정 말고 코드만 집중할 수 있음. 코드를 작성하고, 배포하고, 관리할 수 있게 해줌.

#### 간단한 배포
 텔레그램 서버리스를 사용하면 코드를 간단하게 배포할 수 있음. `npx tgcloud push` 명령어만 사용하면 코드가 배포됨.

#### 관리 쉽게
 텔레그램 서버리스를 사용하면 코드를 관리하기 쉽게 해줌. `npx tgcloud status` 명령어로 코드 상태를 확인할 수 있고, `npx tgcloud pull` 명령어로コード를 업데이트 할 수 있음.
