### C 언어의 결함

C 언어를 30년 넘게 사용해 왔지만, 이 언어는 쓰레기임을 인정해야 함. 아무리 전문가라고 해도 C 언어를 올바르게 사용하는 것은 불가능함. C++도 마찬가지고, 이 언어는 1985년에 만들어졌으니 соврем적인 언어는 아님.

### 정의되지 않은 행동

C 언어는 메모리 안전 언어가 아니기 때문에 다양한 정의되지 않은 행동(UB)이 존재함. 대표적인 예로 더블 프리(double-free), 프리 이후 사용(use after free), 배열 범위 밖 접근 등이 있음. 하지만 이러한 예들은 아직도 많은 개발자들이犯하는 실수임.

### 컴파일러 최적화에 대한 오해

일부 사람들은 컴파일러 최적화를 끄면 정의되지 않은 행동이 발생하지 않는다고 생각하지만, 이는 오해임. 정의되지 않은 행동은 컴파일러가 코드의 의도를 파악할 수 없음을 의미함.따라서 컴파일러는 UB 코드에서 특별한 경우를 처리하지 않음.

### 정의되지 않은 행동의 예시

- `int foo(const int* p) { return *p; }` 함수에서 포인터가 올바르게 정렬되지 않았을 경우 UB임.
- `void set_it(std::atomic<int>* p) { p->store(123); }`와 `int get_it(std::atomic<int>* p) { return p->load(); }` 함수에서 오브젝트가 올바르게 정렬되지 않았을 경우 UB임.
- 캐스트를 사용하여 `char`를 `int`로 변환할 경우 UB임.
- `float`를 `int`로 변환할 경우 UB임.
- NULL 포인터를 선언할 경우 UB임.
- 가변 인자 함수에서 타입을 일치시키지 않을 경우 UB임.
