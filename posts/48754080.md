### 핵 미사일 감지 C 코드에 NaN Poisoning 공격은 가능할까?
난 정말 핵 미사일 감지하는 건 못 해ボㅅㅊ지만 C언어로 만든 코드에 NaN 값을 넣으면 디텍팅이 안되는 버그를 만들 수 있다고는 생각해 본 적은 없다고 할 수 있찌만 실제로 그런 공격이 가능한 줄은 몰랐어. C언어로 만든 코드에 NaN 값을 집어넣는 공격을 'NaN Poisoning'이라고 하는 건 알지?
### NaN이란?
NaN은 Not a Number라는 의미의 약자로 sqrt(-1.0)이나 0/0과 같은 잘못된 연산을 통해 만들어지는 값이야. NaN값은 다른 수와 연산을 수행하면 항상 NaN이 나오고, 비교 연산은 항상 False가 나타남. 따라서 NaN 값을 잘 활용하면 공격자가 원하는 방식으로 코드를 작동시킬 수 있어.
### NaN Poisoning 공격
NaN Poisoning 공격은 코드에서 NaN 값을 만들어내는 방법을 이용해 버그를 만들어서 디텍팅을 피하는 공격 방식이야. 이 공격은 Floating point 값을 이용해서 NaN 값을 만들어내는데, 예를 들어 sqrt(-1.0)이나 0/0과 같은 연산을 통해 NaN 값을 만들 수 있어. 만들어진 NaN 값은 다른 연산을 수행하면 항상 NaN이 나오기 때문에 코드의 흐름을 공격자가 원하는 대로 만들 수 있어.
### 실례 코드
```c
int match(double* test, double* reference, int bins, double threshold) {
    double* testMinusBaseline = (double*) malloc(bins*sizeof(double));
    double* referenceMinusBaseline = (double*) malloc(bins*sizeof(double));
    for (int i = 0; i < bins; i++) {
        testMinusBaseline[i] = fmax(0.0, test[i]-baseline[i]);
        referenceMinusBaseline[i] = fmax(0.0, reference[i]-baseline[i]);
    }
    // Compute the Pearson correlation of test with reference.
    double correlation = 0.0;
    for (int i = 0; i < bins; i++) 
        correlation += (testMinusBaseline[i]-meanTest) *(referenceMinusBaseline[i]-meanReference);
    correlation /= stddevTest*stddevReference;
    return (correlation < threshold ? 0 : 1);
}
```
위의 코드에서 baseline[] 값을 테스트 값보다 크게 만든다면 correlation이 NaN이 돼. 
NaN Poisoning 공격은 코드의 버그를 이용해 공격을 수행하지만, 실제로 공격을 수행할 수 있을지는 코드의 구조와 공격자가 원하는 공격 방식에 따라 다를 거야. 하지만 NaN 값을 이용한 공격은 코드의 흐름을 변경할 수 있기 때문에 중요한 공격 기법 중 하나야.
### 결론
NaN Poisoning 공격은 코드의 버그를 이용해 공격을 수행할 수 있는 기법이야. 공격자가 NaN 값을 만들 수 있는 코드의 취약점을 이용하면 공격을 수행할 수 있어. 따라서 개발자들은 코드를 작성할 때 NaN 값을 만드는 코드를 주의하고, 입력 값에 대한 검증을 엄격히 해야 해.
