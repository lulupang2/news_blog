### GC 쉐입 텬실링, Go 제네릭스 구현하는 법
Go의 새로운 제네릭스 기능은 GC 쉐입 텬실링이라는 방식을 사용해서 구현한댕. 일단 이게 무슨 말인지 정확하게 알기 전에, 기존의 제네릭스 구현 방식인 **모노모피제이션**과 **타입 이레즈**에 대해서 먼저 설명해 볼게.
#### 모노모피제이션
모노모피제이션은 제네릭스 코드를 구체적인 타입으로 변환하는 방식이야. 예를 들어서, 제네릭스 함수가 있다면, 그 함수를 호출하는 코드에 따라서 실제 타입으로 변환해줘. 예를들면 다음과 같은 Go 코드가 있을 때,
```go
func identity[T any](value T) T {
    return value
}
```
만약에 이 함수를 `int`, `float64`, `*User` 타입으로 호출하면, 컴파일러는 실제로 세 개의 함수를 만들어 낸대. 각각의 함수는 실제 타입으로 변환된 구체적인 코드를 담고 있겠지.
#### 타입 이레즈
타입 이레즈는 제네릭스代码를 한 개의 공유된 버전으로 유지하는 방식이야. 제네릭스 코드에서 타입 파라미터를 제거하고, 대신에 공통된 타입을 사용해. 예를 들어서, Java에서는 제네릭스 타입 파라미터를 `Object`타입으로 대체한다던가.
#### GC 쉐입 텬실링
그리고 Go에서는 GC 쉐입 텬실링이라는 방식을 사용해서 제네릭스를 구현했어.GC 쉐입 텬실링은 타입의 GC 쉐입, 즉 메모리 레이아웃과 가비지 컬렉션 정보를 이용해서 코드를 최적화하는 방식이야. 예를 들어서, 다음과 같은 Go 코드가 있을 때,
```go
type User struct{}
type Order struct{}

func identity[T any](value T) T {
    return value
}

func main() {
    fmt.Println(identity(42))
    fmt.Println(identity(3.14))
    fmt.Println(identity(&User{}))
    fmt.Println(identity(&Order{}))
}
```
컴파일러는 이 코드를 컴파일할 때, 각 타입의 GC 쉐입을 계산해서, 같은 GC 쉐입을 가진 타입끼리 코드를 공유할 수 있게 해줘. 예를 들어서, `*User`와 `*Order`는 같은 GC 쉐입을 가졌기 때문에, 컴파일러는 이 두 타입을 위한 코드를 한 개만 생성해서 공유할 수 있지.
이렇게 하면, 코드의 크기가 줄어들고, 컴파일 속도가 개선될수도 있겠지만, 런타임 오버헤드가 없다는 장점도 있어. 다만, 컴파일러가 코드를 최적화하는 과정에서, 일부 코드가 느려질 수도 있고, 코드가 복잡해질 수도 있대.
### 결론
요약하자면, Go의 제네릭스는 GC 쉐입 텬실링이라는 방식을 사용해서 구현했어. 이 방식은 코드를 최적화하는 새로운 방식이야. 컴파일러가 중간중간 코드를 만들때, GC 쉐입 텬실링을 이용해서 코드를 최적화하는데, 그럼 컴파일 속도는 느려질 수도 있고, 코드도 복잡해질수도 있지만, 런타임 오버헤드는 없어서 성능은 좋을거 같은디? ㅋㅋㅋㅋ
