### XLIDE: 엑셀 없는 VBA WORLD
엑셀 없이 VBA 코드를 쓸 수 있는 도구가 나왔 븅븅. XLIDE라는 이름으로 알려진 이 도구는 VS Code 확장 프로그램으로, VBA 코드를 직접 편집하고, 엑셀 파일에 저장할 수 있게 해줌. 
#### XLIDE 특징
*   엑셀 설치할 필요 없음
*   COM automation 안 쓰고, pyOpenVBA로 직접 VBA 바이너리 형식 읽고 씀
*   win32com도 안 씀
*   윈도우, 맥, 리눅스, 리모트 컨테이너에서 다 작동함
*   깃허브 코파일럿도 지원함
#### 설치하는 법
1.  `git clone https://github.com/WilliamSmithEdward/xlide_vscode.git`
2.  `cd xlide_vscode`
3.  `npm install`하고 `npm run compile`
4.  python 가상 환경 생성하고, `pip install -r python/requirements.txt` 실행
5.  VS Code에서 F5 누르면 확장 programas 개발 환경 호스트가立ち上が고, 감시 컴파일러가 작동함
#### 아키텍처
*   `xlide_vscode/` : VS Code 확장 프로그램 소스 코드
*   `python/` : 파이썬 백엔드 소스 코드
*   `xlide/` : VBA IO 관련 소스 코드
*   `syntaxes/` : VBA 문법 정의
*   `language-configuration/` : VBA 언어 구성
*   `docs/` : 문서
#### 디자인 결정
*   장기 실행 Python 프로세스 : Python 시작 시간을 모든 요청에 걸쳐 분산시키는 거임
*   FileSystemProvider : 가상 파일 시스템을 읽고 쓰는 거임. Ctrl+S를 누르면 writeFile이 호출되어서 커스텀 세이브 명령 안 거치고 바로 저장됨
*   가상 URI 스키마 : 워크북 경로와 모듈 이름을 에디터의 파일 개념에서 분리하는 거임
*   공유 export 로직 : UI와 코파일럿 에이전트 도구를 위한 단일 출처의 진실임
*   COM 안 쓰고 pyOpenVBA : 포터블하고, 直接 VBA 바이너리 형식 읽는 거임. 
*   확인บน 쓰기 : AI 에이전트가 생산 워크북을 조용히 변异시키는 걸 방지하는 거임
*   JSON-RPC 메소드 : 파이썬 브릿지에서 호출할 수 있는 메소드들임
#### JSON-RPC 메소드
*   `listModules` : 모듈 목록 반환
*   `listSubs` : 서브루틴 목록 반환
*   `readModule` : 모듈 내용 반환
*   `writeModule` : 모듈 내용 변경
*   `renameModule` : 모듈 이름 변경
*   `deleteModule` : 모듈 삭제
*   `readCells` : 셀 내용 반환
*   `writeCells` : 셀 내용 변경
