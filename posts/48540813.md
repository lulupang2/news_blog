# 소개 
 디시인사이드나 펨코 같은 커뮤니티에서 개발자들이 많이 쓰는 가성비 모델로도 원하는 결과물을 얻을 수 있는 방법을介绍하려고 함. 이런 것들 다 알면 80~90%의 개발 관련 일들은 가성비 모델로도 충분히 할 수 있다고 함. 

 #(prompt)란 
 (prompt)라는 건 결국 컴퓨터한테 명령을 내리는거고, 이 명령을 효율적으로 잘 내리는 법을 알려줌. 

 ## 1단계: raw intention 
 그냥 내가 원하는 걸 말하는 거야. '리액트에서 스테이트를 업데이트해줘' 같은 느낌 

 ## 2단계: decomposed problem 
 내가 원하는 것의 핵심을 파악하는 거야. '리액트 18에서 사용하는 스테이트 훅이 안되는데, 왜 안되는지 알려줘' 

 ## 3단계: structured prompt 
 그냥 내가 생각한 걸 구조적으로 풀어 쓰는거지. '리액트 18에서 스테이트 업데이트하는 방법 알려줘. 스테이트 훅 사용. 오류メ시지 없음.' 

 # prompt를 잘 쓰는 법 
 1. **context economy** :prompt에서 쓸데없는말을 빼고 핵심만 담는 거지.  
- 예를 들면 리액트 코드 작성할 때, `Stack: React 18 + TypeScript + Vite` 라고 먼저 말해주고, 
- 그 다음부터는 이것에 대한 설명 생략하고 바로 문제 핵심으로 넘어감 
 2. **prompt framing techniques** :prompt의 형식을 잘 정의하는 거야. 디버깅할 때, 생성할 때, 설명할 때 각기 다른 형식으로 정의함. 
 3. **iterative refinement vs. one-shot prompting** :단순한 일은 one-shot으로 하지만, 복잡한 일은 여러 단계로 나눠서 하는 것이 효율적. 

 # 모델 분류 
Budget 모델은 대략 4가지 성능 등급으로 나뉜다고 함. 
- **Tier 1**: GPT-4o, Claude Sonnet, Gemini 1.5 Pro 등이 있음. 
- **Tier 2**: LLaMA, Mistral 등이 있음. 
- **Tier 3**: 더 안 좋아지는 모델들 
- **Tier 4**: 제일 성능이 낮은 모델들
