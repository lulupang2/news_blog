### 윈도우 2000 UI에 대한 감상
   윈도우 3.0부터 2000까지의 UI를 정말 좋아함. 
   여기서 윈도우 2000을 예로 들고 있지만, 이는 QEMU/KVM에서 잘 돌아가고 스크린샷을 쉽게 찍을 수 있기 때문임.
   일부 내용이 너무 사소러 보일 수 있지만, 언급할 가치가 있다고 생각함.
   윈도우 2000을 처음 설치하고 부팅하면 다음과 같은 화면이 나옴:
   - 독일어 버전이야. 이 글의 목적에선 상관없음.
   - 이 다이얼로그에 대해서는 나중에 더 이야기할 것임.
   - 지금으로선 이렇게 보인다는 것만 알면 됨:
     -干擾이 없는 깨끗한 단색 배경.
     - 데스크톱에 라벨이 달린 아이콘들.
     - 스타트 버튼, 쿽 라치 아이콘, 시스템 트레이 아이콘, 시계가 있는 탭바.
     - 선명한 글자 렌더링. 
     - 이 부분은 CRT 모니터에서 보니까 진짜 매끄러웠었음.
   - 스타트 버튼을 누르면 다음과 같은 스타트 메뉴가 나타남:
     - 스타트 버튼이 지금은 눌린 상태로 바뀌었음. 
     - 버튼을 누를 때처럼 눌려있는 걸 알 수 있음.
     - 버튼이라는 개념은 사람들이 평소에 많이 사용하는 거라 익숙함.
     - 작은 화살표가 달린 메뉴 항목은 또 다른 메뉴가 있을 걸 알 수 있음.
     - 마우스를.hover하면 해당 항목에 대한 툴팁이 나타남.
   - 스타트 메뉴의 구조가 트리 구조처럼 나타남.
   - 각 항목의 경로가 명확하게 보여줌.
   - 아이콘들이 색이付き고 서로 다름.
   - 아이콘들의 도움을 받아 원하는 항목을 찾을 수 있음.
   - 윈도우 탐색기를 실행하면:
     - 기본 설정이 좀 아쉬운 거 같음. 
     - 파일 확장자를 숨기는 건 잘못된 개념이라고 생각함.
   - 테이블로 보기 변경:
     -테이블로 되어있는 항목들에 폴더 먼저 보여주고, 파일 보여주고
     - 아이콘들이 파일 유형을 나타내고 일관성이 있음
     - 트리 구조로 되어있는 항목은 빠르고 쉽게 탐색할 수 있음.
   - 원드패드를 열어보면:
     -대부분의 항목이 버튼이나 핸들로 나타나거나, 흰 배경을 가지고 있음.
     -이 색상들은 처음엔 헷갈리지만, 일단 알아두면 기억에 남음. 그리고 일관된 패턴이 mọi 곳에 있음
   - 색상 조화가 좋음.
     - 지나치게 하얀색이나 까만색, 또는 근처에 색상이 없음. 
     - 적절한 대조를 이루고 있음.
   - 워드패드가 윈도우 2000에서 새로 만들어진 프로그램이라면 조금 다르게 보일 것임
   - 데스크탑 설정을 열어보면:
     - 같은 일관된 UI 스타일과 시각적 힌트가 있음.
     - 탭에 3D 리얼 效과가 있어 클릭할 수 있을 것이라고 추측할 수 있음.
   - 모든 스크린샷에서 스크롤바가 항상 보여짐.
     -이것은 사용자로 하여금 해당 영역이 스크롤할 수 있는지 알 수 있음.
   - 전역 프로그램 옵션들이預測可能한 패턴을 따름:
     - 새로운 창이 나타나고 탭이 있음.
     - 탭 안의 항목들이 그룹화되어 있음.
   - 비교 대상은 다음과 같음:
     - 현대의 설정 다이얼로그들은 서로 다른 항목들끼리 어떤 항목들이 연결되어 있는지 구분하기 어려움.
   - 성능에 관해서:
     -少し 느린 감이 있음. 부팅 시간도有点 길어.
     - 그래도 사용에는 문제 없음. 요즘 웹사이트들만큼은 아니지만, 평균적인 웹사이트의 성능과 비슷함.
   - 이 설정은 다음의 사양에서 구동했음:
     - 윈도우 2000 SP4
     - 펜티엄 133
     - 64MB 램
     -하드 디스크
     - 펜티엄2나 펜티엄3을 사용하면 훨씬 더 빠르게 작동함.
   -이 당시 펜티엄 133 프로세서는 이미 나온 지 4-5년이 되었었음.
   - 느린 시스템에서 좋은 성능을 내기 위해서는 몇 가지를 희생해야 함.
   - 윈도우 2000은 프로덕트 액티베이션 이전의 것임.
     -완전히 오프라인에서 돌아감. 
     -여전히 사용 가능함.
   -이러한 사항을 종합해서 본다면:
     - NT 커널이 탑재된 빠른 운영체제임.
     - 명료하고 일관된 UI를 가지고 있음.
     -구매한 제품이기 때문에 구독이 아님.
   -이후로 UI 스타일이 조금씩 변한 걸 볼 수 있음.
     -윈도우 2000의 'Welcome' 다이얼로그에서부터 이러한傾向을 볼 수 있음.
   - comparer를 보면:
     -윈도우 95와의 대조를 보면
     -윈도우 2000의 툴바 아이콘이 버튼처럼 보이지만, 실제로는flat한 스타일임.
     -윈도우 95의 툴바 아이콘이 실제 버튼처럼 보임
   -이러한 시각적 힌트가 점점 사라지는 추세이다 보니
     -현재 사용자들은 어떤 항목이 interactive한지 알기 어렵게 됨
   - 예전에는 사람들은 다음과 같이 말했음:
     --flat한 UI가 좋다고 생각함.
     -하지만, 실제론 사용자로 하여금 interactive한 항목을 찾는 것이 힘든 일임.
   -이런 trend를 멈추길 바라는 마음에 이 글을 남김
   - 구체적인 예를 보면:
     -안드로이드의 스와이프 gesture는 실제生活에서 존재하지 않음.
     -체크박스가 더 의미가 있지 않을까?
     -실제生活에서 버튼이나 끈을 조작하는_experience를 반영하는 것 역시 좋은 예임
