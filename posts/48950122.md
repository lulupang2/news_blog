### SQLite 쓰면서 학습한 것들
   최근에 다자고 라는 사이트를 만드려고하는데요. 다자고는 다이어리 쓰는 걸 좋아하는 사람들이 모여서 각자의 일기들을 공유하는 커뮤니티 사이트입니다. 
   그래서 다자고를 만들면서 Django 사이트를 최근 작업하고 있는데요, SQLite를 데이터베이스로 사용하기로 했습니다. 
   SQLite를 데이터베이스로 사용할 때는 몇 가지 작은 걸 배우고 있습니다. 
   이 사이트가 SQLite를 사용하는 네 번째 사이트인데요, 이전보다 더 어려운 것 같아요. 
   그 이유는 Django의 ORM을 사용하기 때문입니다. 이전에는 Django를 사용하지 않았어요. 
   
   ### ANALYZE의중요성
   첫 번째는 ANALYZE의 중요성을 알게 되었습니다. 
   오늘 FTS5를 사용한全文検索 쿼리를 실행했는데요, 4000개의 행이 있는 테이블에서 5초가 걸렸어요. 
   컴퓨터가 빠르니까 5초는 오래 걸린 것 같아요. 
   그래서 ANALYZE를 실행했더니 쿼리 속도가 0.05초로 빨라졌어요. 
   정확히 어떤 문제가 발생했는지는 아직 모르지만, 쿼리 플랜이 잘못된 것 같아요. 
   ANALYZE는 통계를 생성해서 쿼리 플래너가 더好的선택을 할 수 있게 해줍니다. 
   
   ### 데이터베이스 정리하기
   데이터베이스를 정리하는 것은 어렵습니다. 
   가끔씩 원하지 않는 행들이 데이터베이스에 저장되는데요, 이러한 행들을 삭제하기 위해 명령을 실행합니다. 
   하지만 이러한 명령이 5초 이상 걸리면 다른 워커들이 데이터베이스에 쓰려고 하는데 타임아웃이 발생하고요,_VM이 종료됩니다. 
   그래서 이러한 정리 작업들을 작은 배치로 나누어 выполしています. 
   
   ### 백업 방법
   SQLite 백업을 두 가지 방법으로 진행하고 있습니다. 
   첫 번째는 restic을 사용하는 방법입니다. 
   ```bash
   sqlite3 /data/calendar.db "VACUUM INTO '/tmp/calendar.sqlite'"
   gzip /tmp/calendar.sqlite
   # S3 업로드
   restic -r s3://some-bucket/ unlock
   restic -r s3://some-bucket/ backup /tmp/calendar.sqlite.gz
   restic -r s3://some-bucket/ snapshots
   restic -r s3://some-bucket/ forget -l 1 -H 6 -d 2 -w 2 -m 2 -y 2
   restic -r s3://some-bucket/ prune
   ```
   두 번째는 Litestream을 사용하는 방법입니다.
   ```bash
   litestream replicate -config litestream.yml
   ```
   Litestream으로 백업하면Incremental 백업이 가능합니다. 
   
   ### 결론
   결론은 SQLite를 사용하면서 ANALYZE의 중요성을 알게 되었습니다. 
   또한 백업 방법은 두 가지입니다. 
   ORM 쿼리 성능은 걱정 안 해도 될 것 같아요.
