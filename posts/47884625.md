# 소개
   파일 시스템 보안 관련해서 요즘 많이 고민중임. 코드 하나 추가하거나 호출하는 것만으로도 엄청난 취약점이 생겨버림. 
   # 문제는 어디에서 오는걸까? 
   - 경로 조작하기 쉬움. 쉽게말해 상대 경로../.ssh/id_ed25519 이런식으로. 
   - 심링크(symlink) 때문에 발생하는 TOCTOU(time-of-check to time-of-use) 문제도 있음. 
   - 쉽게 말해 경로 체크 후에 파일을 여는 시점에 경로 이름이 바뀐 경우임. 
   - 이와 관련해서는 [libglnx](https://gitlab.gnome.org/alexl/libglnx) 라이브러리에 Chase 기능이 있는데, 이게 경로를 안전하게 따라가줌. 
   - GLib/Gio API는 이런 경로 문제를 다루지 못하고, 경로만 받기 때문에 보안 상 매우 취약함. 
   - 그래서 GLib의 새로운 API를 만들었다고 하더라. 
   [FUSE](https://github.com/libfuse/libfuse) 기반으로 구현한것으로, fd-chasing 방식을 사용해서 보안 취약점을 줄일 수 있다고함. 
   # 결론
   - 경로를 사용하는것은 매우 위험하다. 
   - 파일 디스크립터를 사용해 경로를 체크하고, 그 디스크립터로 파일을 여는 방식을 사용해야한다. 
   - FUSE 기반으로 파일시스템을 구현하면, fd-chasing 방식으로 경로를 안전하게 따라갈 수 있기 때문에, GLib/Gio의 보안 취약점을 피할 수 있음.
