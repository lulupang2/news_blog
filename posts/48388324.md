### 엘릭서 1.20: 이제 점진적 타입 언어임다!
   이제 엘릭서도 점진적 타입 언어가 됐음. 엘릭서 1.20은 점진적 타입 시스템을 도입했 karak. 이게 무슨 의미인지 알아보자.

   엘릭서팀은 2022년부터 엘릭서에 집합론적 타입 시스템을 도입하려고 했음. 그리고 2023년 6월에는 엘릭서의 타입 시스템 디자인에 대한 논문을 발표했 karak. 이번 엘릭서 1.20은 첫 번째 개발 里석인 타입 추론과 점진적 타입 검사를 모든 엘릭서 프로그램에 대해 انجام함으로써 완수했음. 이것은 엘릭서가 점진적으로 죽은 코드와 검증된 버그를 보고한다는 것을 의미함. 엘릭서가 찾은 버그는 실제 런타임에 실패할 것을 보장하는 타입 위반임.

   엘릭서의 타입 시스템은 다음과 같은 목표를 가지고 있음:
   * 안전함: 타입 시스템이 추론하고 할당하는 타입이 프로그램의 동작과 일치함
   * 점진적임: 엘릭서의 타입 시스템은 동적 타입을 포함하는데, 이것은 변수나 식의 타입을 런타임에 확인할 때 사용됨
   * 개발자 친화적임: 타입은 기본 집합 연산을 사용하여 설명됨. 이것은.union, 교집합, 부정과 같은 연산을 사용하여 타입을 구성할 수 있음.

   ### 엘릭서의 점진적 타입 시스템
   엘릭서의 점진적 타입 시스템은 다른 언어들과 다르게 동적 타입을 사용함. 동적 타입은 다음과 같은 두 가지 중요한 속성을 가지고 있음:
   * 호환성: 엘릭서의 타입 시스템은 동적 타입을 사용하여 타입 위반을 보고하지만, 이것은 실제 런타임에 실패할 것을 보장하지는 않음.
   * 좁혀짐: 엘릭서의 동적 타입은 프로그램에서 사용함에 따라 좁혀질 수 있음. 이것은 엘릭서가 더 정확한 타입 정보를 얻을 수 있도록 해줌.

   엘릭서의 점진적 타입 시스템은 다음과 같은 예제를 통해 이해할 수 있음:
   ```elixir
   def percentage_or_error(value) when is_integer(value) do
     # ...
   end
   ```
   이 예제는 엘릭서의 타입 시스템이 동적 타입을 사용하여 타입 위반을 보고하는 것을 보여줌.

   엘릭서의 점진적 타입 시스템은 또한 Narrowing을 지원하는데, 이것은 동적 타입을 좁혀서 더 정확한 타입 정보를 얻는 것을 의미함. 예를 들어:
   ```elixir
   def add_a_and_b(data) do
     data.a + data.b
   end
   ```
   이 예제는 엘릭서의 타입 시스템이 동적 타입을 좁혀서 더 정확한 타입 정보를 얻는 것을 보여줌.

   ### 앞으로의 계획
   엘릭서팀은 앞으로도 타입 시스템을 개선하고, 새로운 기능을 추가할 계획임. 특히, 엘릭서팀은 사용자 정의 타입 시그니처를 지원하고, 재귀적 타입과 매개변수 타입을 지원할 계획임.

   엘릭서 1.20은 개발자들에게 더 안전하고, 더 빠른 개발을 가능하게 해줌. 엘릭서팀은 앞으로도 엘릭서를 계속 개선하고, 더 많은 기능을 추가할 계획임.
