### 브라우저 구동 원리
   - 브라우저 구동을 위해 필요한 세 가지 요소: 빠른 시작, 격리, 저렴한 비용
   - 파이어 크래커VM을 EC2에 설치해서 브라우저를 1초만에 시작하는 방법에 대해説明함
   - 브라우저에 필요한 요소: 크로미움, 파일시스템, 쿠키, 캐시, 프록시 설정, 다운로드, 로그인 세션 등
   
   ### 파이어 크래커의 특징
   - 파이어 크래커는 가벼운 VM 시스템
   - 일반 VM보다 빠르고 저렴하다
   - EC2에서 사용 가능
   
   ### 브라우저 구동 과정
   1. 사용자가 브라우저 요청
   2. 컨트롤 플레인이 호스트 선택
   3. 호스트가 브라우저 VM을 시작
   4. 브라우저가 크로미움 구동
   5. 브라우저가 준비되면 사용자에게 연결 URL 제공
   
   ### 성능 최적화
   - 메모리 맵핑 최적화: 4KB 페이지에서 2MB 페이지로 변경
   - 페이지 폴트 처리 최적화: 커스텀 핸들러 사용
   - 크로미움 시작 최적화: 虛擬 CPU 핀닝 사용
   - 등등 다양한 최적화로 인해 브라우저 구동 속도 개선 ㅋㅋㅋㅋ
