### JSON-LD란 뭐냐?
JSON-LD, 즉 JSON Linked Data는 웹페이지에 구조화된 데이터를 추가하는 형식이야. 웹 크롤러가 사이트의 의미 구조를 이해하는 데 도움이 되며, 더 풍부한 링크 미리보기 및 검색 순위 개선 등의 이점을 제공할 수 있어.

### JSON-LD 적용 방법
JSON-LD를 페이지에 추가하려면 다음 코드를 `<head>` 섹션에 추가하면됨.
```json
<script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@graph": [
      {
        "@type": "WebSite",
        "@id": "https://hawksley.dev/#website",
        "url": "https://hawksley.dev/",
        "name": "Ethan Hawksley"
      }
    ]
  }
</script>
```

### 주요 속성 정리
- `@type`: 노드의 유형을 지정함. 예: `WebSite`, `WebPage`, `Person` 등
- `@id`: 노드의 고유 식별자를 지정함. 일반적으로 URL의 끝에 해시값을 붙여서 사용함
- `url`: 페이지의 URL을 지정함
- `name`: 页面의 이름을 지정함

### WebSite 노드
웹사이트에 대한 메타데이터를 제공함. 모든 페이지에 포함해야 하지만, 전체 노드만 루트 페이지에 포함하고 다른 페이지에는 축소된 버전만 포함해도 됨.
```json
{
  "@type": "WebSite",
  "@id": "https://hawksley.dev/#website",
  "url": "https://hawksley.dev/",
  "name": "Ethan Hawksley"
}
```

### WebPage 노드
현재 페이지를 설명함. 각 페이지마다 포함되어야 함.
```json
{
  "@type": "WebPage",
  "@id": "https://hawksley.dev/blog/hack-club-campfire/#webpage",
  "url": "https://hawksley.dev/blog/hack-club-campfire/",
  "isPartOf": {
    "@id": "https://hawksley.dev/#website"
  },
  "name": "Winning the Hack Club Campfire Hackathon"
}
```

### Person 노드
개인에 대한 정보를 제공함. 모든 페이지에 포함되어야 함.
```json
{
  "@type": "Person",
  "@id": "https://hawksley.dev/#person",
  "url": "https://hawksley.dev/",
  "name": "Ethan Hawksley",
  "image": {
    "@type": "ImageObject",
    "@id": "https://hawksley.dev/#person-image",
    "url": "https://hawksley.dev/ethan-hawksley.png",
    "caption": "Ethan Hawksley"
  }
}
```
이러한 노드들을 적절히 사용하면 검색 엔진 최적화(SEO)에 도움이 될 수 있음. 더 자세한 정보는 Schema.org를 참고하여.
