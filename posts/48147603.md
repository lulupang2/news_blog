# Radicle 소개
Radicle은 깃헙 같은 코드 호스티ng 플랫폼인데, 깃헙하고 달리 깊은 개념임. 중앙집중식이 아니라는 거임. 즉, 코드 호스티ng 플랫폼의 자유를 느껴라!

## 특징
* 소버린 코드 포지: 중앙집중식 코드 호스티ng 플랫폼이 아니라는 거임. 즉, 코드 호스티ng 플랫폼의 자유를 느껴라!
* 피어 투 피어 코드 협업 스택: 깃헙 같은 중앙집중식 코드 호스티ng 플랫폼이 아니라는 거임.
* 노드 실행 및 협업 흐름 제어: 유저가 노드를 실행하여 협업 흐름을 제어할 수 있는 거임.
* 소프트웨어 개발자들이 자유롭게 협업할 수 있는 환경을 제공하는 거임.

###.Radicle 시작하기
Radicle을 설치하기 위해서는 터미널에서 명령어를 실행하면 됨.
```bash
curl -sSLf https://radicle.dev/install | sh
```
그리고 노드를 실행하기 위해서는 터미널에서 다음 명령어를 입력하면 됨.
```bash
radicle node start
```
그럼 노드가 실행되고, 협업을 시작할 수 있는 거임.

## Radicle 의 구조
Radicle은 노드, 저장소, HTTP 디먼 등으로 구성된 거임.
* Radicle 노드: 코드 협업을 위한 노드임.
* Radicle 저장소: 코드를 저장하는 저장소임.
* HTTP 디먼: 코드 협업을 위한 HTTP 요청을 처리하는 디먼임.

### Radicle 의 장점
* 자유로운 코드 협업: 중앙집중식 코드 호스티ng 플랫폼에 비교해 자유롭게 코드 협업을 할 수 있는 거임.
* 협업 흐름 제어: 유저가 협업 흐름을 제어할 수 있는 거임.
