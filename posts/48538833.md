# Luz 소개 

   Luz는 0개의 외부 라이브러리를 사용해 Scratch부터 만든 C++20 Path Tracer임 ㄹㅇ 

   Luz는 몬테카를로 경로 추적, 전역 조명, BVH 가속, 적응 샘플링, 디노이즈, 대기 산란, 사용자 정의 장면 파일 및 블렌더 익스포터를 지원함 ㅇㄴ 

   ## 기능 

   - 몬테카를로 경로 추적 

   - 전역 조명 

   - 멀티 스레드 CPU 렌더링 

   - 적응 샘플링 

   - 디노이즈 (NFOR 스타일) 

   - 구, 평면, 사각형, 삼각형, 큐브, 볼륨 및 OBJ 메시 지원 

   - 람버트, 금속, 유전체, 발광 및 등방성 재료 지원 

   - 영역, 점, 구 및 방향성 조명 지원 

   - 사용자 정의 `.luz` 장면 파일 

   - `.blend`에서 `.luz`로의 익스포터 

   - CLI 또는 장면 파일을 통해 렌더링 매개변수 완전 사용자 정의 

   - PDF를 사용한 중요 샘플링 

   - BVH 가속 (메시 BVH 포함) 

   - 대기 시뮬레이션 (산란 포함) 

   - dof,抗алиас, 노출, 대조, 톤 매핑, 감마 보정 및 블룸 

   - bmp 및 tiff 출력 

   - 렌더링, 디노이즈, 후처리 및 스코어 분해와 관련된 결정적 ベンチ 마크 하네스 

   ## 시작하기 

   Luz를 사용하기 위해 C++20 컴파일러, 메이크 또는 CMake 3.16+, 파이썬 3(옵션 툴/스크립트 전용)이 필요함 ㅋ 

   ```bash
   make
   ./Luz --file examples/scenes/blender_monkey.luz --samples 50 --resolution 300x300
   ``` 

   기본 출력은 `render.bmp`임 ㅇㄴ 

   장면 파일에서 `outputfilename`을 설정하거나 CLI에서 일반 렌더링 설정을 OVERRIDE할 수 있음 ㄹㅇ 

   ```bash
   make test
   ``` 

   ## 벤치마크 

   Luz에는 렌더링, 디노이즈, 후처리 및 전체 스코어 비교를 위한 결정적 벤치마크가 포함되어 있음 ㅋㅋ 

   ```bash
   make benchmark BENCH_CPUS=1 BENCH_THREADS=1 > before.csv
   make benchmark BENCH_CPUS=1 BENCH_THREADS=1 > after.csv
   make benchmark-compare BEFORE=before.csv AFTER=after.csv
   ``` 

   벤치마크에 대한 자세한 정보는 `docs/benchmarks.md`를 참조하라 ㅋ 

   ## 플랫폼 지원 

   Luz는 macOS, 리눅스, Windows를 지원함 ㅇㄴ 

   macOS 및 리눅스에서 Makefile이 주요 경로임 ㅋ 

   Windows에서는 CMake를 사용해 MSVC 또는 MinGW 기반 Makefile을 사용할 수 있음 ㄹㅇ 

   ```bash
   cmake -S . -B build
   cmake --build build
   ctest --test-dir build
   ``` 

   WSL도 리눅스 빌드 환경으로 지원됨 ㅋ 

   ## 빌드 최적화 

   릴리즈 빌드는 기본적으로 빌드하는 머신에 맞춰 튜닝됨 ㅇㄴ 

   Makefile은 `-O3`, 네이티브 CPU 튜닝 `-march=native`, `링크 시간 최적화 -flto`를 사용함 ㅋ 

   CMake도 같은 릴리즈 意圖를 사용함 ㄹㅇ 

   이러한 기본값은 로컬 렌더링을 빠르게 하지만, `-march=native`로 ビルド된 바이너리는 오래된 또는 다른 CPU에서 실행되지 않을 수 있으며, LTO는 툴체인별 링커 문제를 노출할 수 있음 ㅋ 

   불법 명령어 충돌, 링커 에러 또는 더 포터블한 바이너리가 필요한 경우 агрессив한 옵션을 비활성화하고 깨끗한 오브젝트로 재빌드하세요 ㄹㅇ 

   ```bash
   make clean
   make NATIVE=0 LTO=0
   ``` 

   CMake 빌드의 경우, 최적화 토글을 꺼서 구성하세요 ㅋ 

   ```bash
   cmake -S . -B build -DLUZ_NATIVE_OPTIMIZATIONS=OFF -DLUZ_ENABLE_LTO=OFF
   cmake --build build --clean-first
   ``` 

   ## CLI 사용법 

   ```bash
   ./Luz [options]
   ``` 

   - `--file` : `.luz` 장면 파일 로드 

   - `--resolution` : 렌더링 해상도 Override 

   - `--samples` : 픽셀당 샘플 수 Override 

   - `--adaptive` : 적응 샘플링 활성화 

   - `--no-adaptive` : 적응 샘플링 비활성화 

   - `--adaptive-min-samples` : 적응 샘플링의 최소 샘플 수 

   - `--adaptive-threshold` : 상대적인 적응 노이즈 임계치 

   - `--adaptive-check-interval` : 적응 수렴 확인 간격 

   - `--maxLightBounces` :최대 라이트 바운스 Override 

   - `--threads` : 작업 스레드 수 

   - `--seed` : 랜덤 샘플링 시드 

   - `--gamma` : 감마 보정 토글 

   - `--tonemapping` : 톤 매핑 토글 

   - `--bloom` : 블룸 토글 

   - `--exposure` : 노출 보상 

   - `--contrast` : 대조 муль티플라이어 

   - `--denoise` : 디노이즈 활성화 

   - `--no-denoise` : 디노이즈 비활성화 

   - `--output` : 렌더 출력 경로 Override 

   - `--denoise-output` : 디노이즈드 출력 경로 Override 

   - `--render-times` : 렌더링 시간 작성 

   - `--benchmark` : 내장 벤치마크 장면 실행 

   - `--benchmark-case` : 벤치마크 케이스 

   ## 적응 샘플링 

   `--adaptive`는 `--samples`를 픽셀당최대 샘플 수로扱う것임 ㅋㅋ 

   각 픽셀은 점진적인 픽셀당 샘플 시퀀스를 사용하며 `--adaptive-min-samples` 이상 렌더링한 다음주기적으로輝度 및 RGB 신뢰 구간을 확인함 ㅇㄴ 

   매우 어두운 픽셀은 거의黑色로 수렴될 때까지 보수적인 최소 샘플을 사용함 ㅋ 

   더 낮은 임계치는 자세한 디테일을 유지하지만 더 많은時間이 걸림 ㄹㅇ 

   ## 디노이즈 

   `--denoise`는 Luz의 NFOR 스타일 디노이즈 기능을활성화하며, 별도의 디노이즈된이미지를 작성함 ㅋ 

   디노이저에는 하드 최소 해상도 또는 샘플 수 제한은 없지만, 유용한 색상 및 기능 통계를 추정하기위해 충분한 신호가 필요함 ㅇㄴ 

   픽셀당 하나의 샘플은 주로 스트레스 테스트이며, 픽셀당 분산 추정치가 없으므로 디노이즈된 이미지에는 변경된 내용이 거의ない 것처럼 보이거나 옳지 않은 디테일을 매끄럽게할 수 있음 ㄹㅇ 

   프리뷰를 위해 픽셀당 少なくとも 여러 샘플을 사용하고, 디노이저의 품질을 평가할 때는 대략 16개 이상의 샘플을 픽셀당 사용하기를 권장함 ㅋ 

   매우 낮은 해상도는 평가를误解할 수 있으므로 로컬 필터 윈도우가 이미지의 너무 많은 부분을 덮음 ㄹㅇ 

   ## 장면 파일 

   예시 장면은 `examples/scenes/`에 있음 ㅋ 

   메시 어셋은 `assets/objects/`에 있음 ㅇㄴ 

   장면 파일 형식은 `docs/scene-files.md`에 문서화되어 있음 ㅋ 

   `.luz` 파일에서 오브ジェクト 경로는 현재 작업 디렉토리에 대한 상대 경로로 해결됨 ㅋ 

   `examples/scenes/blender_monkey.luz`는 `../../assets/objects/blender_monkey.obj`를 참조할 수 있으며 리포지토리 루트에서 실행할 수 있음 ㅇㄴ 

   OBJ 메시는 또한 오프셋되어야하고, 장면 물질할당 받을 수 있음 ㅋ 

   ## 블렌더 익스포터 

   블렌더 장면은 블렌더의 파이썬 API를 통해 익스포팅할 수 있음 ㅋ 

   ```bash
   /Applications/Blender.app/Contents/MacOS/Blender -b scene.blend --python tools/blender_export_luz.py -- --output exports/scene.luz
   ./Luz --file exports/scene.luz --threads 8
   ``` 

   익스포터는 `.luz` 파일 및 OBJ 메시를 작성함 ㅇㄴ 

   사용법 및 현재 피дел리티 한계는 `docs/blender-exporter.md`에 문서화되어 있음 ㅋ 

   ## 저장소 레이아웃 

   - `include/luz/` : Public 헤더 

   - `src/core/` : 수학, 기하학, 재료, 이미지, 샘플링 코드 

   - `src/renderer/` : 렌더링 구현 

   - `src/scene/` : 장면 모델 및 장면 헬퍼 

   - `src/io/` : 장면 파일, OBJ, BMP, TIFF 로드/쓰기 

   - `src/cli/` : 명령줄 엔트리 포인트 및 플래그 

   - `examples/scenes/` : 예시 `.luz` 장면 파일 

   - `assets/objects/` : 예시에서 사용되는 OBJ 어셋 

   - `docs/images/` : 압축된 쇼케이스 이미지 

   - `tools/` : 익스포트 및 유틸리티 스크립트 

   - `tests/` : 표준 라이브러리 전용 테스트 프로그램 

   - `docker/` : 벤치마크 컨테이너
