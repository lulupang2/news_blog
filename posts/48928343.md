### SQLITE에서 NUL 캐릭터 때문에 개박살나는 이야기 ㅋㅋㅋㅋ
   SQLITE는 문자열 중간에 NUL 캐릭터를 허용하는데, 이거 때문에 문자열 길이 계산이나 문자열 비교에서 문제가 생길 수 있대요.
   length() 함수는 첫 NUL 캐릭터 전까지의 문자열 길이만 반환하구요. quote() 함수도 첫 NUL 캐릭터 전까지의 문자열만 보여줍니다.
   CLI에서 .dump 커맨드를 쓸 때에도 첫 NUL 캐릭터 이후의 문자열은 생략되는데, 이는 모든 상황에서 첫 NUL 캐릭터 이후의 문자열을 忽略하는 SQLITE의 특성이래요.
   그래서 SQLITE 문서에서는 문자열에 NUL 캐릭터를 쓰지 말라고 경고하고 있습니당.
   
   ### 예상치 못한 행동 ㅋ
   예를 들어 CREATE TABLE t1( a INTEGER PRIMARY KEY, b TEXT ); INSERT INTO t1(a,b) VALUES(1, 'abc'||char(0)||'xyz'); SELECT a, b, length(b) FROM t1; 을 실행하면 1,'abc',3 이라는 결과가 나옵니다. 근데 SELECT * FROM t1 WHERE b='abc'; 을 실행하면 아무 결과도 안 나옵니다. 이유가 뭔지 알겠어요? ㅋㅋㅋㅋ SQLITE는 실제로 t1.b 컬럼에 'abc'||char(0)||'xyz' 문자열을 가지고 있하는데, 이 문자열은 'abc' 문자열과 다르기 때문에 검색 결과가 안 나옵니다. 하지만 CLI에서는 'abc' 문자열만 보여주기 때문에 사용자는 혼란을 느끼게 돼요.
   
   ### NUL 캐릭터 확인 방법 
   문자열을 BLOB으로 캐스팅하면 전체 문자열 길이가 보입니다. 예를 들어 SELECT a, CAST(b AS BLOB) FROM t1; 을 실행하면 1,X'6162630078797a' 결과가 나옵니다. 이 결과에서 4번째 캐릭터는 NUL 캐릭터임을 확인할 수 있습니다. 또는 instr(X,char(0)) 함수를 사용해도 NUL 캐릭터가 있는지 확인할 수 있습니다.
   
   ### NUL 캐릭터 제거 방법 
   UPDATE t1 SET b=substr(b,1,instr(b,char(0))) WHERE instr(b,char(0)); 를 실행하면 NUL 캐릭터 이후의 문자열을 제거할 수 있습니다. 이 방법으로 데이터베이스에서 NUL 캐릭터를 제거할 수 있어요.
