# NGINX 개같이 터지노 ㅋㅋㅋㅋ

NGINX 최근 버그 때문에 개빡치는 점이 하나 생겼어. 익스플로잇이公開되서 이제 아무나 NGINX 서버 장악 가능함. 

NGINX 리모트 코드 실행 버그는 2008년에 생긴 건데, 지금까지 발견 못하다가 최근에 발견된 거임. 이 버그 때문에 Rewrite와 Set 지시문을 사용하는 서버는 공격받을 가능성이 있대. 

이 버그는 다른 세 가지 메모리 오류와 함께 발견되었는데, 이 오류들의 CVE 번호는CVE-2026-42946, CVE-2026-40701, CVE-2026-42934 임. 

NGINX 리모트 코드 실행 버그는 익스플로잇이 공개되었고, 이 버그를 이용해서 서버를 장악하는 방법은 이미 공개되어있음. 그래서 이제는 서버 관리하는 사람들 빡치겠네 ㅋㅋㅋㅋ

# 해결 방법 

解决 방법은 NGINX를 최신 버전으로 업그레이드 하는 거임. NGINX Open Source의 경우 1.31.0 或 1.30.1로 업그레이드하고, NGINX Plus의 경우 R36 P4, R35 P2, R32 P6로 업그레이드하면 됨. 

업그레이드 방법은官网에서 설명하고 있으니,参照해봐.
