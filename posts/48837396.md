# 코딩 평가에서 신호와 노イズ 구분하기
코딩 평가를 할 때 가장 어려운 건, 정말 중요한 코드들만 골라내는 거임.
일단 코드 전체를 다 읽고나면, 이 코드에서 실제로 중요한건 뭐였었는지 기억이 안 나거나, 중요하지도 않은 부분에서 시간을 다 쏟았던 경우가 많음 ㅋㅋㅋ.
이런 문제를 해결하기 위해, 일단 코드를 평가하기 전에 평가하려는 사람과 정확히 어떤 걸 평가하려는지 정의를 내리고, 그 안에서 신호와 노イズ 구분하기를 위해 어떤 기준을 사용할 건지 정해놔야 함.
그리고 코드를 읽을 때마다 '이게 정말 중요한 부분이야?'라고 생각이 들면, 일단 해당 부분에 주석을 달아두거나, 별도로 노트를 작성해두는 것도 좋은 방법임.
그럼 이후에 코드를 다시 읽을 때, 이 부분이 정말 중요한 건지 아닌지 쉽게 구분할 수 있기 때문임. 
개빡치는 점은 이렇게 중요한 코드들만 골라내도, 이 코드가 실제로 어떤 문제를 해결하고, 어떤 값을 생산해내는지를 정확히 파악하기가 힘든 경우가 많다는 거임.
이때는, 일단 코드가 어떤 입력을 받고, 어떤 출력을 내뿜는지, 이런 과정을 중간중간 확인해나가면서, 이 코드가 실제로 어떤 문제를 해결하고 있는지 파악하는 과정이 필요함.
이러한 과정을 통해, 코딩 평가에서 신호와 노イズ 구분하기를 위해 코드를 정확하게 평가할 수 있으면, 더 좋은 코드를 작성하고, 더 좋은 서비스를 개발할 수 있기 때문에, 이 과정이 정말 중요하다고 생각함.
