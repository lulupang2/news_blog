### numa에 대해 알아보장
 numa 란 non-uniform memory access 의 약자 임. cpu와 메모리 사이의 간격이 중요하다는 뜻 이당. 
 numa 전까지는 uma(uniform memory access) 라는 게 있었음. uma는 cpu랑 메모리 사이의 간격이 같다는 뜻 임. 예를들면 cpu 0와 cpu 1 모두 메모리에 같은 시간에 접근 가능 한다는 뜻 이당. 

#### numa의 역사 
 numa는 90년대에 대형 서버 시스템에서 처음 도입되었음. 여러개의 cpu를 한개 시스템에 넣는다고 했을 때, 
 메모리 접근 시간이 cpu 마다 다름. cpu랑 메모리가 가까웠을때는 빨랐음. 

#### numa 의 문제점 
 numa의 가장 큰 문제점은 Remote memory access 때문임. 
 cpu0 가 node 1에 있는 메모리에 접근 하려면, interconnect를 타고 가야함. 근데 이 경로가 오래 걸리므로, remote memory access 는 local memory access 보다 느림. 
 numa Aware 하는 방법은 간단하지 않음. numa를 고려해서 메모리를 할당하거나, 
 cpu affinity 를 조절 해야 하는데, 이 모든게 코드를 짤 때 고려해야 할 문제 임. 

#### numa 에 대한 이해 
 numa를 이해하기 위해서는 시스템 구조와 메모리 접근 패턴을 고려해야 함. 시스템이 어떻게 구성되어있는지, 
 메모리가 어떻게 할당되어 있는지 등을 파악해야 함. 

#### 결론 
 numa는 시스템 성능을 향상시키기 위한 중요한 요소 임. 
 numa에 대한 이해와 적용은 시스템 개발자에게 중요한 일임. 
 numa를 고려해서 시스템을 최적화 하면, 시스템의 성능을 향상 시킬 수 있음.
