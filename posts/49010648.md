# SIMD란 무엇임?
SIMD란 싱글 인스트럭션 멀티플 데이터/single instruction, multiple data의 약자로 cpu가 여러개의 값을 하나의 명령어로 처리할 수 있는 기술임. 
예를들어 4바이트씩 비교하는게 아니라 4, 8, 혹은 그 이상의 바이트를 하나의 명령어로 비교할 수 있는 기술이지.
만약에こう한loop가 있다면
```c
for(byte in bytes){
    //...
}
```
이 loop는 SIMD를 사용해서 속도를 개선할 수 있는데 예를들어 
```c
for(8byte chunk in bytes){
    //...
}
```
이렇게 바꿀 수 있지. 
이 기술이 먹히려면 데이터의 양이 많아야하는데 만약에 데이터가 몇 바이트밖에 없다면 SIMD를 사용하는 것보다 그냥 일반的な loop를 사용하는게 더 빨라질 수 있지. 
SIMD에는 여러가지 종류가 있는데 예를들어 simdutf, simdjson같은 라이브러리들이 있다는데 
이 라이브러리들은 SIMD 기술을极端的に 사용해서 복잡한 알고리즬들을 사용하더라. 
하지만 SIMD를 사용하기 위해서는 이렇게 복잡한 알고리즬들을 사용할 필요가없다는데 
이 포스트에서는 SIMD의 기초부터 시작해서 어떻게 SIMD를 사용할 수 있는지 다루겠다는데 
이 포스트는 zig라는 언어를 사용해서 설명하지만 다른 언어에서도 적용할 수 있는 일반적인 내용이라고 하네 

# SIMD의 일반적인 구조
SIMD를 사용하는 코드는 일반적으로 다음과 같은 구조를 지니지.
1. 상수값을 broadcast하고 벡터의 accumulator를 초기화한다.
2. 입력データ를 하나의 벡터씩 처리한다.
3. 벡터 간의 연산을 수행한다.
4. 벡터의 결과를 축소하거나 저장한다.
5.残った 요소를 스칼라 tail로 처리한다.

# 실제 예제
實際的 예제를 하나보자. 
```c
while (end < cps.len and cps[end] > 0xF) end += 1;
```
이 loop는 한 바이트씩 처리하는데 이걸 SIMD로 최적화할 수 있다는데 
SIMD 버전의 코드는 다음과 같네 
```c
if (simd.lanes(u32)) |lanes| {
    const V = @Vector(lanes, u32);
    const threshold: V = @splat(0xF);
    while (end + lanes <= cps.len) : (end += lanes) {
        const values: V = cps[end..][0..lanes].*;
        const greater_than_threshold = values > threshold;
        if (@reduce(.And, greater_than_threshold)) continue;
        const mask: std.meta.Int(.unsigned, lanes) = @bitCast(greater_than_threshold);
        end += @ctz(~mask);
        break;
    }
}
while (end < cps.len and cps[end] > 0xF) end += 1;
```
이 코드는 어떻게 작동하는지 보자 
## 1. 상수 값을 broadcast하고 벡터의 accumulator를 초기화한다
```c
if (simd.lanes(u32)) |lanes| {
    const V = @Vector(lanes, u32);
    const threshold: V = @splat(0xF);
}
```
simd.lanes(u32)라는 함수는 u32타입의 벡터를 처리할 수 있는 레인의 개수를 반환하는데 
이 예제에서는 arm에선 4, avx2에선 8, avx-512에선 16個의 레인이 반환된다고 하네. 
@Vector(lanes, u32)이라는 함수는 레인의 개수에 맞는 벡터 타입을 생성해주고 
@splat(0xF)라는 함수는 threshold 변수에 0xF의 값을 broadcast시켜주네. 

## 2. 입력 데이터를 하나의 벡터씩 처리한다
```c
while (end + lanes <= cps.len) : (end += lanes) {
    const values: V = cps[end..][0..lanes].*;
}
```
이 loop는 레인의 개수에 맞는 크기로 데이터를 처리하는데 
end변수는 데이터의 끝을 가리키는 변수인데 
 cps[end..][0..lanes]라는 식은 cps 데이터의 end지점부터 lanes개의 데이터를 가져오는거네. 

## 3. 벡터 간의 연산을 수행한다
```c
const greater_than_threshold = values > threshold;
```
이 연산은 values와 threshold를 비교해서 greater_than_threshold라는 변수에 저장하는데 
이 변수는 각 레인에서 threshold보다 큰값이 있는지 없는지를 저장하는 변수네. 

## 4. 벡터의 결과를 축소하거나 저장한다
```c
if (@reduce(.And, greater_than_threshold)) continue;
const mask: std.meta.Int(.unsigned, lanes) = @bitCast(greater_than_threshold);
end += @ctz(~mask);
break;
```
greater_than_threshold변수에는 각 레인에서 threshold보다 큰값이 있는지 없는지를 저장했었는데 
@reduce(.And, greater_than_threshold)라는 함수는 각 레인의 값을 and연산해서 
threshold보다 크지않은値가 하나라도 있다면 false를 반환하는거네. 
만약에 greater_than_threshold의 모든 레인이 true라면 continue문이 실행되어 
다음 loop를 수행하게 되는데 
 그렇지않다면 greater_than_threshold를 비트마스크로 변환하여 
threshold보다 크지 않은 값을 찾는데 
@ctz(~mask)라는 함수는 mask변수의 비트를 뒤집은 다음 첫번째 1비트가 나오기 전까지의 0비트의 수를 반환하는 함수인데 
이 함수를 사용해서 threshold보다 크지 않은 값을 찾은 다음 
end 변수에 해당 인덱스를 더하여 break문을 실행하는거네 

## 5. 잔태요소를 스칼라 tail로 처리한다
```c
while (end < cps.len and cps[end] > 0xF) end += 1;
```
이 loop는残った 요소를 스칼라 tail로 처리하는데 
end 변수가 cps 데이터의 끝을 가리킬때까지 
threshold보다 큰 값을 계속해서 찾는거네. 
이ように 해서 전체적으로 loop를 최적화하는거지 

이러한 최적화가 어떻게 성능에 영향을 미치는지 보자 
해당포스트에서는 avx2를 사용한 intel cpu에서 5배 빠른 성능을 보였다고 하는데 
이처럼 SIMD를 사용하면 성능 최적화를 할 수 있다는 것을 알 수 있지 
이것이 SIMD를 사용하는 것의 장점인데 
그럼 SIMD를 사용하는 것의 단점은 뭘까? 
그건 바로 SIMD를 사용하기 위해서는 데이터가 충분히 크지 않으면 
SIMD를 사용하는 것보다 일반적인 loop를 사용하는 것이 더 빠를 수 있다는 점인데 
그렇다면 SIMD를 언제 사용해야할까? 
그건 바로 데이터가 충분히 클 때SIMD를 사용해야 한다는 점인데 
따라서 SIMD를 사용하기 이전에 데이터의 크기를 확인하는 것이 중요하네. 
이처럼 SIMD는 성능최적화를 할 수 있는 강력한 도구임에 틀림없는데 
SIMD를 사용하기 위해서는 데이터의 크기와 SIMD를 사용하는 것이 정말로 필요한지 확인해야 하네.
