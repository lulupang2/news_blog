# 구글에서 코드 편집기 통일시키는 법
구글 개발자들은 코드 편집기를 정해 사용하라고 시키는데 안 시킨다고 해서 자유롭게 편집기를 골라 사용할 수 있었음. 근데 이게 오히려 문제가 됨. 왜냐하면 각자 편집기를 다르게 사용했거든. 
어느정도 시니어 개발자들은 편집기를 통일시켜야 한다는 질문을 받았는데 답은 '노'였음. 왜냐하면 개발자들은 각자 편집기에 대한 의견이 다 달라서 통일시키기 힘들다는 거였음.
 Jeff Dean이라는 사람이 이 вопрос에 대한 대답을 했는데 '개발자들을 하나의 편집기로 통일시키려고 하는 건 불가능에 가깝다'라고 했음. 
이 얘기는 오랫동안 유지됐는데 구글 개발자 중 한 명이 12년 동안 편집기 관련 툴 개발을 맡았는데 이 때 편집기를 통일시키려고 노력을 했음. 
각자 편집기를 사용하니까 편집기 관련된 일들을 여러 번 반복하게 되서 이 건에 대해서 고민이 되었음. 
예를 들어 빌드 도구나 코드 포맷터 등이 있어야 하는데 이 것을 여러 번 반복해야 했음. 
구글에는 20% 시간이라는 정책이 있는데 이政策은 직원들이 20%의 시간을 자유롭게 사용할 수 있다는 것임. 
이 정책 때문에 편집기 관련된 툴들을 개발하는 사람들이 많았음. 
예를 들어 인텔리제이 편집기와 관련된 툴을 개발하는 팀이 있었음. 
이 팀은 2015년에 만들어졌는데 이 때문에 editar에 대한 지원을 더 잘 할 수 있었음. 
근데 이 편집기들이 구글의 코드를 다루는데 부족한 점이 있었음. 
예를 들어 코드의 크기가 너무 커서 로컬에서 다루기 어려웠음. 
이 문제를 해결하기 위해서 클라우드 기반의 편집기를 개발하기로 함. 
이 편집기를 Cider라고 부르고 2013년?에 개발을 시작했음. 
처음에는 기술 문서 편집을 위해 개발했지만 이후에 개발자들이 사용할 수 있을 정도로 발전함. 
예를 들어 코드自動補完이나 refactor 기능이 있음. 
이 기능들은 구글의 코드 베이스를 다루는데 매우 유용했음. 
이후에 Cider V라는 새로운 버전을 개발했는데 이 버전은 VSCode를 프론트 엔드として 사용했음. 
이 버전은 2020년에 개발을 시작했는데 2021년에 오픈 베타를 시작했음. 
이후에 많은 기능들이 추가됨. 
예를 들어 버전 관리나 코드 리뷰 툴이 있음. 
이후에 Cider V를 사용하는 개발자가 80%까지 증가했음. 
이로 인해 코드 편집기를 통일시키는 것이 가능해졌음. 
이로 인해 개발자들이 더 효율적으로 코드를 관리할 수 있게 됨. 
이후에 AI 기능들이 추가되면서 코드 편집기가更加 똑똑해졌음. 
예를 들어 코드 자동 완성이나 코드 리뷰를 자동으로 하는 기능이 있음. 
이러한 기능들이 추가되면서 코드 편집기를 통일시키는 것이 더욱 महत해졌음. 
이로 인해 개발자들이 코드를 더 효율적으로 관리할 수 있게 됨. 
이러한 점이 있으니까 코드 편집기를 통일시키는 것이 중요하다고 생각함. 
이것이 구글에서 코드 편집기를 통일시키는 법임 ㅋㅋ
