### 데이터 압축의 정석
데이터 압축이 뭔가요?簡單하게說하면 데이터를 줄이는 기술임. 
압축이 가능하게 하려면 모델링과 코딩이 필요함. 모델링은 데이터의 확률 분포를 추정하는데 사용되고, 
코딩은 더 짧은 코드를 더 많이 쓸 수록 데이터를 더 많이 압축할 수 있게 해줌. 
하지만, 데이터를 100% 압축할 수 있는 방법은 없음. 왜냐하면 데이터의 확률 분포를 완벽하게 알 수 없고, 
데이터를 100% 압축하면 원래 데이터를 복원할 수 없게 되기 때문임. 

#### 인포메이션 시어리
데이터 압축의 한계는 ShannonEntropy라고 불리는 것에서 나옴. 
Entropy란 데이터가 얼마나 무작위인지를 측정하는 척도인데, 
데이터가 완전히 무작위라면 Entropy가 최대가 되고, 
데이터가 완전히 규칙적이라면 Entropy가 최소가 됨. 
Entropy는 데이터를 압축할 수 있는 한계를 정의함. 
즉, 데이터의 Entropy가 높을수록 데이터를 압축하기 어려웠다는 뜻임. 

### 코딩의 한계
모델링을 통해 데이터의 확률 분포를 추정하면, 
코딩을 통해 데이터를 압축할 수 있음. 
하지만, 코딩에도 한계가 있음. 
코딩의 한계는 Shannon Coding Theorem에서 나옴. 
Shannon Coding Theorem은 데이터를 압축할 수 있는 한계를 정의함. 
즉, 데이터의 Entropy가 높을수록 데이터를 압축하기 어려웠다는 뜻임. 
Shannon Coding Theorem은 코딩의 한계를 정의함. 
예를 들어, Huffman Coding은 데이터의 확률 분포에 따라 코드를 할당하는 방법임. 
Huffman Coding은 데이터를 효과적으로 압축할 수 있지만, 
데이터의 확률 분포가 복잡할수록 Huffman Coding의 효율성이 낮아짐. 

#### 데이터 압축의 종류
데이터 압축의 종류는 Lossless Compression과 Lossy Compression이 있음. 
Lossless Compression은 데이터를 압축하여 원래 데이터를 완벽하게 복원할 수 있는 방법임. 
Lossy Compression은 데이터를 압축하여 원래 데이터를 완벽하게 복원할 수 없는 방법임. 
Lossless Compression의 예로는 Huffman Coding, LZ77, LZ78 등이 있음. 
Lossy Compression의 예로는 JPEG, MP3 등이 있음. 
Lossless Compression은 데이터를 압축하여 원래 데이터를 완벽하게 복원할 수 있지만, 
데이터를 많이 압축할 수 없는 한계가 있음. 
Lossy Compression은 데이터를 많이 압축할 수 있지만, 
원래 데이터를 완벽하게 복원할 수 없는 한계가 있음. 

### 데이터 압축의 예
데이터 압축의 예로는 이미지压縮, 오디오 압축, 비디오 압축 등이 있음. 
이미지 압축의 예로는 JPEG, PNG 등이 있음. 
오디오 압축의 예로는 MP3, AAC 등이 있음. 
비디오 압축의 예로는 H.264, H.265 등이 있음. 
이미지, 오디오, 비디오 압축은 데이터를 많이 압축할 수 있지만, 
원래 데이터를 완벽하게 복원할 수 없는 한계가 있음. 
따라서, 데이터 압축의 종류를 선택할 때는 데이터의 특성과 압축의 목적을 고려해야 함.
