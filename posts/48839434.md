# 코드면접 문제: 중앙값 구하기
코드면접에서 문제를出는데 대부분 비슷한 유형의 문제들이 나옴. 난 퍼즐 문제는 별로 안좋아함. 그 代わりに 깊이 обсужд할 수 있는 문제를 출제함. 예를 들어 중앙값 구하는 문제가 그런 유형의 문제임.
중앙값 구하는 함수를 작성하는 문제임. 이거는 기본적으로 프로그래밍할 수 있는지 테스트하는 문제임. 배열의 값을 정렬해야하는데 이때 정렬은 함수내에서 해야하는지 호출하는 쪽에서 해야하는지에 대한 논의가 필요함. 
또한 배열의 길이가 짝수인지 홀수인지에 따라 연산이 달라지므로 이문제를 통해 디버깅하는 과정을 관찰할 수 있음. 
统计적으로中央값을 사용하는 이유도 обсужд할 수 있는 문제임. 
그리고 이 문제는 테스트하기 쉬운 문제이므로 후보자가-extra 점수를 얻을 수 있는 기회임.
 python 구현 코드도 있음:
```python
def median(numbers: list[float]) -> float:
    # 빈 리스트가 들어왔을때 어떻게 처리할지에 대한 문제임. 
    # 예외를 발생시키거나 특정값을 반환할지에 대한 논의가 필요함.
    if not numbers:
        raise ValueError("median called with empty list")
    # python은 참조 전달식이므로 sorted()와 numbers.sort()의 차이점에 대해 обсужд할 수 있음.
    numbers = sorted(numbers)
    length = len(numbers)
    mid = length // 2
    # 길이가 짝수인지 홀수인지에 따라 계산이 달라짐.
    if length % 2 == 0:
        return (numbers[mid - 1] + numbers[mid]) / 2.0
    else:
        return numbers[mid]
```
