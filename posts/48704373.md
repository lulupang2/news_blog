# 웨이파인더 라우터: 로컬과 클라우드 모델 사이의 쿼리 판단을 결정해주는 기술
웨이파인더 라우터는 오프라인에서 쿼리를 판단해서 로컬과 클라우드 모델 중에 하나를 선택할 수 있게 해줌. 이 기술은 latency, cost, randomness를 없애고 결정적인 라우팅을 제공해줘.
## 웨이파인더 라우터는 어떻게 작동하는 걸까?
웨이파인더 라우터는 쿼리의 구조와 단어 선택을 분석해서 로컬과 클라우드 모델 중에 하나를 선택할 수 있게 해줌. 예를 들어, 'summarize this' 와 같은 단순한 쿼리는 로컬 모델에서 처리하고, 'fix my typo' 와 같은 복잡한 쿼리는 클라우드 모델에서 처리해줌.
## 다른 라우터와 비교하면?
다른 라우터는 모델 호출을 통해 쿼리를 결정하지만, 웨이파인더 라우터는 오프라인에서 구조와 단어 선택을 분석해서 quyết định해줌. 이는 latency와 cost를 줄여주고, 결정的な 라우팅을 제공해줘.
## 웨이파인더 라우터 사용법
웨이파인더 라우터 사용법은 매우 간단함. 일단, pip install wayfinder-router 로 설치하고, wayfinder-router init 로 설정 파일을 생성하면 됨. 설정 파일에서 로컬과 클라우드 모델을 지정하고, threshold 값을 설정하면 됨.
## 결론
웨이파인더 라우터는 로컬과 클라우드 모델 사이에서 쿼리를 결정지을 수 있는 기술임. 이는 latency와 cost를 줄여주고, 결정적인 라우팅을 제공해줘. 웨이파인더 라우터를 사용하면, 더 효율적인 모델 관리를 할 수 있을 것임.
