**C# 15 유니언 타입 도입**
================================

이제 C# 15에서도 유니언 타입을 사용할 수 있게 되었음. 유니언 타입은 함수형 프로그래밍에서 많이 사용되는 개념으로, 하나의 타입이 여러 가지 다른 타입을 대표할 수 있게 하는 거로 ㄹㅇ 편함.

**유니언 타입은 뭔가요?**
------------------------

유니언 타입은 두 가지 이상의 서로 다른 타입을 하나의 타입으로 합치는 것을 말함. 예를 들어, `Result<T>` 타입은 성공했을 때와 실패했을 때 두 가지 상태를 대표할 수 있으면서도, 성공했을 때에는 `T` 타입의 값을 가졌을 때고 실패했을 때에는 오류 메시지를 가지는 거임. 이런 타입을 유니언 타입이라고 하며, C# 15에서는 `union` 키워드를 사용해서 정의할 수 있게 되어서 정말 좋음.

**C# 15에서 유니언 타입 사용하기**
--------------------------------

C# 15에서 유니언 타입을 사용하려면 `.NET 11 preview 2` 이상의 SDK를 설치해야 하고, `LangVersion`을 `preview`로 설정해야 할 것임. 그리고 `union` 키워드를 사용해서 유니언 타입을 정의할 수 있음.

```csharp
public union SupportedOS(Windows, Linux, MacOS);
```

이렇게 정의된 유니언 타입을 사용하면, 여러 가지 타입을 하나의 타입으로 합치고, 스위치 문을 사용해서 타입에 따라 다른 동작을 수행할 수 있게 되어서 ㄹㅇ 편함.

**유니언 타입의 내부 구현**
-------------------------

유니언 타입은 컴파일러가 자동으로 구현해주지만, 구현된 코드는 다음과 같음.

```csharp
[Union]
public struct SupportedOS : IUnion
{
    public object? Value { get; }

    public SupportedOS(Windows value) => this.Value = (object)value;
    public SupportedOS(Linux value) => this.Value = (object)value;
    public SupportedOS(MacOS value) => this.Value = (object)value;
}
```

这样으로 유니언 타입을 사용하면, 함수형 프로그래밍의 장점을 vyu할 수 있으면서도, C#의 강력함을 살릴 수 있게 되있음. 그래서 C# 15의 유니언 타입을 사용해서 개발하면, 코드를 더 간결하고 읽기 쉽게 만들 수 있을 것임.
