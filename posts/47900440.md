> **🔗 원문 보러가기:** [A web-based RDP client built with Go WebAssembly and grdp](https://github.com/nakagami/grdpwasm)

### 웹 기반 RDP 클라이언트 나왔다!!
   RDP 클라이언트는 Go 웹어셈블리와 grdp로 만들었다는 거임. 
   이제 브라우저에서 바로 윈도우 리모트데스크톱 서버에 연결할 수 있게 되었어. 
   플러그인이 필요한 게 아니라서 너무 좋음 ㅋㅋㅋ.
   브라우저에서 WebSocket을 통해 proxy 서버에 연결하고, 
   proxy 서버가 RDP 서버의 TCP 포트에 연결해 주는 구조임. 
   proxy 서버는 브라우저와 RDP 서버 사이의桥梁처럼 작동하는 거임 ㅇㅇ.
   ### 사용법
   1. Go 1.24 이상을 설치하고, 
   2. RDP 서버를 준비하면 됨. 
   윈도우나 RDP 호환 서버라면 어디나 돼 ㅋㅋㅋ.
   ### 빌드 및 실행
   1. `git clone`으로 소스 코드를 받고, 
   2. `make all`로 빌드하고, 
   3. `make serve`로 실행하면 됨. 
   브라우저에서 `http://localhost:8080`으로 접속하면 끝임 ㄹㅇ.
   ### 키보드와 마우스
   표준 키보드 입력은 모두 RDP 스캔 코드를 통해 전달되고, 
   마우스 이동, 버튼 클릭, 스크롤 휠도 모두 지원된다는 거임. 
  ブラ우저 탭에 포커스가 있어야 키보드 이벤트가 전달된다고 했음. 
  _canvas 영역 안에 클릭하면 키보드 포커스 될 거야 ㅋ.
   ### 오디오
   RDP를 통해 오디오를 스트리밍하고, 
   브라우저의 웹 오디오 API를 통해 재생하게 돼. 
  PCM 44100 Hz, 스테레오, 16비트 리틀엔디언으로 스트리밍된다고 했어. 
   ### 보안 관련 사항
   프록시 서버는 아무런 출처에서 오는 접속을 모두 받아주니까, 
   신뢰할 수 있는 네트워크에서만 실행해야 한다는 거임. 
   브라우저와 프록시 서버 사이의 통신은 WebSocket을 통해 이루어지니까, 
   HTTPS나 WSS를 사용해서 통신해야 한다고 했어. 
   ### 마무리
   요즘에는 웹 기반으로 거의 모든 걸 해결할 수 있게 된 거 같음. 
   이제는 윈도우 리모트데스크톱도 웹에서 쓸 수 있게 된 거임 ㅋㅋㅋ. 
   개발자들은 이런 편의 기능을 많이 개발했으면 좋겠음 ㅋㅋㅋ.
