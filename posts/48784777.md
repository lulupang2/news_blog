### 리눅스 htop/top 설명
htiop/top는 리눅스에서 CPU 사용량이나 프로세스 목록을 보기 위한 명령어야. 

#### uptime
우선 htop의 최상단 왼쪽에 있는 uptime를 봐. 
이건 시스템이 실행된 시간을 나타내는데, `uptime` 명령어를 사용해도同じ 값을 볼 수 있지.
```bash
$ uptime
12:17:58 up 111 days, 31 min, 1 user, load average: 0.00, 0.01, 0.05
```
그럼 `uptime`는 어떻게 시스템이 실행된 시간을 알 수 있을까?
그건 `/proc/uptime` 파일에서 값을 읽어 오는 거임. 
```bash
$ cat /proc/uptime
9592411.58 9566042.33
```
이 파일엔 시스템이 실행된 총 시간과 시스템이闲してる tổng 시간이 써있는 거임. 

#### load average
`uptime` 명령어를 사용했을 때 load average도 같이 나와. 
이 값은 시스템의 평균 부하를 나타내는데, `/proc/loadavg` 파일에서 읽어 오는 값이야. 
```bash
$ cat /proc/loadavg
0.00 0.01 0.03 1/120 1500
```
이 파일엔 시스템의 1분, 5분, 15분 평균 부하가 써있고, 현재 실행중인 프로세스 개수와 총 프로세스 개수도 같이 나와. 

#### 프로세스
htop의 오른쪽 상단에 프로세스 관련된 정보가 나와. 
여기엔 전체 프로세스 개수와 실행중인 프로세스 개수가 나와. 
이 값을 Tasks로 표시하는데, Task는 프로세스의 다른 이름으로 리눅스 커널에서 프로세스를 Task라고 부르는거래. 
htop에서 프로세스 상세 정보를 볼 수 있는 건 `/proc/<pid>`에서 값을 읽어 오는 거임. 
```bash
$ ls /proc/12503
attr  cgroup  clear_refs  cmdline  comm  cpu  cpuset  cwd  environ  exe  fd  fdinfo  gid_map  io  limits  map_files  maps  mem  mountinfo  mounts  net  numa_maps  oom_adj  oom_score  oom_score_adj  pagemap  personality  projid_map  root  sched  schedstat  sessionid  setgroups  smaps  stack  stat  statm  status  syscall
```
이 값들은 프로세스 상태, 메모리 사용량, 실행 경로 등 프로세스와 관련된 정보를 나타내. 

요약하자면 htop/top는 리눅스 시스템의 CPU 사용량과 프로세스 목록을 확인하는 명령어야. 
uptime과 load average 값을 통해 시스템의 실행 시간과 부하를 확인할 수 있고, 프로세스 관련된 정보를 `/proc/<pid>`에서 확인할 수 있지.
