### 해쉬맵/셋 라이브러리 'hopscotch-map' 소개
   hopscotch-map 라이브러리는 C++로 만든 속도甲 해쉬맵/셋 라이브러리야. 
   이 라이브러리는 tsl::hopscotch_map, tsl::hopscotch_set, tsl::hopscotch_pg_map, tsl::hopscotch_pg_set라는 네 가지 클래스를 제공하고 있어. 
   각각의 클래스는 성능과 메모리 사용량에 있어서 std::unordered_map/set보다 더 나은 performans를 보장해주고, 구글에서 제공하는 google::dense_hash_map 라이브러리와도 유사점을 가지고 있어.
   
### 성능 비교
   해쉬맵/셋 라이브러리의 성능比較 결과에 따르면, tsl::hopscotch_map이 std::unordered_map보다 더 빠르다고 나와 있음. 이건 캐시 친화적인 데이터 구조를 사용해서 성능을 개선한 결과라고 할 수 있따.
   
### 사용법
   ```cpp
   #include <tsl/hopscotch_map.h>
   int main() {
       tsl::hopscotch_map<std::string, int> map = {"a", 1};
       map["b"] = 2;
       map.insert({"c", 3});
       map.erase("b");
       for (auto it = map.begin(); it != map.end(); ++it) {
           // ...
       }
   }
   ```
   
### 특징
   + 헤더 파일만 추가하면 사용할 수 있는 헤더-온리 라이브러리야.
   + 캐시 친화적인 데이터 구조를 사용해서 성능 개선.
   + 이동 전용 및 非기본 생성자 키/값 지원.
   + 이종 조회 지원.
   + 예약 필요 없는 감시자 값.
   + 해쉬 값 재계산 없이 빠른 조회와 삽입 지원.
   
### 성장 정책
   라이브러리では 성장 정책을 3가지 제공하고 있어.
   + power_of_two_growth_policy : 해쉬맵/셋의 사이즈를 2의 거듭제곱으로 제한.
   + prime_growth_policy : 해쉬맵/셋의 사이즈를 소수로 제한.
   + mod_growth_policy : 사용자 지정 성장 팩터 사용.
   ### 설치 방법
   ```bash
   git clone https://github.com/Tessil/hopscotch-map.git
   cd hopscotch-map
   mkdir build
   cd build
   cmake ..
   cmake --build .
   ./tsl_hopscotch_map_tests
   ```
