### 카스토르 소개
카스토르는 테라폼처럼 인공지능 에이전트를 관리할 수 있는 도구임. 현재 에이전트들은 프레임워크 안에서 임페러티브하게 정의되거나 플랫폼 UI에서 클릭해서 조합함. 하지만 벤더 중립적인 버전 관리가 가능한 코드는 없음. 카스토르는 이런 문제를 해결해 줌.

### 주요 특징
* HCL로 작성된 선언형 스펙(.agent, .tool, .prompt 파일)
* Go 툴체인으로 에이전트를 빌드하고 관리할 수 있음
* 호스팅된 플랫폼에서 에이전트를 長期資源으로 관리할 수 있음
* 상태, 차이,드리프트 탐지를 지원함

### 설치 및 사용법
* 홈브루로 설치할 수 있음: `brew tap weirdGuy/tap && brew install kastor`
* 스크립트로 설치할 수 있음: `curl -fsSL https://raw.githubusercontent.com/weirdGuy/kastor/main/scripts/install.sh | sh`
* Go 1.26 이상에서 설치할 수 있음: `go install github.com/weirdGuy/kastor/cmd/kastor@latest`

### 예제
* weather 에이전트 예제 있음: `go build ./cmd/kastor && ./kastor validate examples/weather/ && ./kastor build examples/weather/`
* 빌드 후에 LangGraph 프로젝트가 생성됨

### 기타
* 카스토르는 에이전트 런타임이나 프레임워크가 아님. 그냥 에이전트를 관리하는 도구임.
* 개발자는 SPEC.md를 참고하면 됨. CLAUDE.md는 일일 컨벤션을 기록함.
