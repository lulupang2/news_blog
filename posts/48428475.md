### 소개
-sem은 결국 git으로 코드 이해를 개선하는 신비한 툴이야. 기존 LSP랑은 달리 엔티티 중심으로 돌아가는듯.
-기존 LSP는 코드 변경 사항을 알려주지만, sem은 변경된 코드가 어떤 영향을 미치는지 보여줌.
-예를 들어 함수를 변경했을 때, 함수를 사용하는 다른 코드가 어떤 것인지 알 수 있게 해주는 거지.

### 주요 기능
- ** Sem Diff **: 엔티티 중심의 diff를 제공해서, 어떤 코드가 변경되었는지 쉽게 파악할 수 있어.
  -예를 들어 함수를 추가했다면, `⊕ function validateToken [added]`라고 표시해 주는 거지.
- ** Sem Blame **: 각 엔티티가 언제 변경되었는지 보여줌. 
  -예를 들어 함수가 언제 변경되었는지, 누가 변경했는지 알 수 있어.
- ** Sem Impact **: 코드 변경이 다른 어떤 코드에 영향을 미치는지 보여줌.
  -예를 들어 함수를 변경했을 때, 함수를 사용하는 다른 코드가 어떤 것인지 알 수 있게 해줘.
- ** Sem Log **: 엔티티의 변경 이력을 보여줌.
  -예를 들어 함수가 언제 변경되었는지, 어떻게 변경되었는지 알 수 있어.

### 설치 및 사용법
- `$ brew install sem-cli`
- `$ sem setup`
- `$ git diff`를 사용하면 sem diff가 자동으로 작동해.
