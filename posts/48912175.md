### HTMX와 고랭으로 웹 애플리케이션 만들기
웹 애플리케이션을 구조하기 어렵나요? 제 새 책은 고랭으로 실제 웹 애플리케이션을 만드는 과정을 처음부터 끝까지 안내합니다. 코드 구조화, 의존성 관리, 다이나믹 페이지 만들기, 사용자 인증 및 인가 등 다양한 주제를 다룹니다. 7월 말까지 30% 할인 중입니다! 
저는 웹 애플리케이션에 인터랙티브 요소를 추가할 때 HTMX를 많이 사용합니다. 왜냐하면ooth한 앱 같은 느낌을 주기 떄문입니다. 그리고 자바스크립트 코드를 많이 안해도 되기 때문이요. 그리고 서버에서 HTML을 렌더링 하는 게 안정적이고 안전하기 떄문입니다. 이 포스팅에서는 HTMX를 고랭과 함께 사용하는 방법을 소개합니다. 
#### 프로젝트 셋업
다음 명령어를 실행하여 프로젝트 기본 구조를 만들 수 있습니다.
```bash
$ go mod init example.com/htmx
$ mkdir -p assets/static/css assets/static/img assets/static/js assets/html/partials assets/html/pages cmd/web
$ touch assets/efs.go assets/html/base.tmpl assets/html/partials/images.tmpl assets/html/pages/home.tmpl cmd/web/main.go cmd/web/handlers.go cmd/web/html.go
```
이 명령어를 실행하면 다음과 같은 디렉토리 구조가 만들어집니다. 
```
.
├── assets
│   ├── efs.go
│   ├── html
│   │   ├── base.tmpl
│   │   ├── pages
│   │   │   └── home.tmpl
│   │   └── partials
│   │       └── images.tmpl
│   └── static
│       ├── css
│       ├── img
│       └── js
├── cmd
│   └── web
│       ├── handlers.go
│       ├── html.go
│       └── main.go
└── go.mod
```
#### HTMX 설치
HTMX를 설치하는 방법은 여러 가지가 있지만, 거의 언제나 HTMX를 직접 다운받아 정적 파일로 제공합니다. 왜냐하면 이렇게 하면 CDN을 사용하는 단점을 피할 수 있으니까요.
다음 명령어로 HTMX를 다운받을 수 있습니다.
```bash
$ wget -P assets/static/js https://cdn.jsdelivr.net/npm/htmx.org@1.8.4/dist/htmx.min.js
$ wget -P assets/static/css https://cdn.jsdelivr.net/npm/bamboo.css@2.10.0/dist/bamboo.min.css
$ wget -O assets/static/img/gopher.png https://raw.githubusercontent.com/egonelbre/gophers/refs/heads/master/sketch/misc/standing-left.png
```
#### HTML 템플릿
저는 대부분의 프로젝트에서 `assets/html` 디렉토리를 사용합니다. 이 디렉토리에는 다음과 같은 구조를 사용합니다. 
```
assets/html
├── base.tmpl
├── pages
│   └── home.tmpl
└── partials
    └── images.tmpl
```
`base.tmpl` 파일에는 공통 HTML 레이아웃이 들어갑니다. `pages` 디렉토리에는 개별 페이지의 컨텐츠가 들어가고, `partials` 디렉토리에는 재사용 가능한 HTML 조각이 들어갑니다. 
`base.tmpl` 파일에 다음 코드를 추가할 수 있습니다.
```html
{{define "base"}}
<!doctype html>
<html lang='en'>
<head>
    <meta charset='utf-8'>
    <title>{{template "page:title" .}}</title>
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <link rel="stylesheet" href="/static/css/bamboo.min.css">
    <script defer src="/static/js/htmx.min.js"></script>
</head>
<body>
    <h1><a href="/">Example website</a></h1>
    <main>
        {{template "page:content" .}}
    </main>
</body>
</html>
{{end}}
```
`home.tmpl` 파일에는 다음 코드를 추가할 수 있습니다.
```html
{{define "page:title"}}
Home
{{end}}
{{define "page:content"}}
<button hx-get="/gopher" hx-swap="outerHTML"> Wanna see a cute gopher? </button>
{{end}}
```
`images.tmpl` 파일에는 다음 코드를 추가할 수 있습니다.
```html
{{define "partial:image:gopher"}}
<img alt="Gopher" src="/static/img/gopher.png" width="{{.}}">
{{end}}
```
#### 정적 파일 임베딩
고랭 1.16에서부터 파일 임베딩을 사용할 수 있습니다. 이 기능을 사용하면 HTML 파일과 정적 파일을 런타임에 읽는 대신, 빌드 시에 고랭 바이너리에 직접 포함시킬 수 있습니다. 
`assets/efs.go` 파일에 다음 코드를 추가할 수 있습니다.
```go
package assets

import (
    "embed"
    "io/fs"
)

//go:embed "html" "static"
var files embed.FS

var (
    HTMLFiles = sub(files, "html")
    StaticFiles = sub(files, "static")
)

func sub(f embed.FS, dir string) fs.FS {
    sub, err := fs.Sub(f, dir)
    if err != nil {
        panic(err)
    }
    return sub
}
```
#### HTML 템플릿 렌더링
HTML 템플릿을 렌더링 하기 위해, `htmlRenderer` 타입을 만들 수 있습니다. 이 타입은 다음 기능을 수행합니다. 
*   공유 템플릿을 초기화 합니다. 
*   공유 템플릿을 복제하여 특정 템플릿을 렌더링합니다. 
*   렌더링된 템플릿을 HTTP 응답으로 보냅니다. 
`cmd/web/html.go` 파일에 다음 코드를 추가할 수 있습니다.
```go
package main

import (
    "bytes"
    "html/template"
    "io/fs"
    "net/http"
    "time"
)

type htmlRenderer struct {
    templateFS fs.FS
    sharedTemplates *template.Template
}

// The newHTMLRenderer function creates a new htmlRenderer containing a shared
// set of parsed templates with support for any custom template functions.
func newHTMLRenderer(templateFS fs.FS, sharedTemplateFiles ...string) (*htmlRenderer, error) {
    funcs := template.FuncMap{
        "now": time.Now,
        // Other custom template functions go here...
    }
    sharedTemplates, err := template.New("").Funcs(funcs).ParseFS(templateFS, sharedTemplateFiles...)
    if err != nil {
        return nil, err
    }
    r := &htmlRenderer{
        templateFS: templateFS,
        sharedTemplates: sharedTemplates,
    }
    return r, nil
}
```
