### 큐는 시스템 과부하를 해결하지 못한다 
システム 과부하를 해결하려고 큐를 사용하는 경우가 많아. 하지만 큐를 사용하는 것은 시스템 과부하를 해결하는 방법이 아님. 
계속해서 시스템이 과부하에 닥치게 된다면, 결국 큐는 더 많은 데이터를 담을 수 없게 되고, 시스템은 더 이상 처리할 수 없는 상태가 되버림. 
이러한 문제를 해결하기 위해서는 시스템의瓶頸을 찾아내고, back-pressure나 load-shedding을 통해 시스템의 처리량을 조절하는 방식을 사용해야 함. 
back-pressure와 load-shedding은 시스템의 처리량을 조절하는 방식으로, 시스템이 과부하에 닥치지 않도록 도와줌. 
back-pressure를 사용하면 시스템이 처리할 수 있는 데이터의 양을 조절할 수 있고, load-shedding을 사용하면 시스템이 처리할 수 없는 데이터를 버릴 수 있음. 
이러한 방식을 사용하면 시스템의 처리량을 조절할 수 있고, 시스템이 과부하에 닥치지 않도록 도와줄 수 있음. 
### 큐는 새로운 문제를 만든다 
큐를 사용하면 새로운 문제가 생길 수 있음. 큐는 시스템의 처리량을 증가시키지만, 시스템이 처리할 수 없는 데이터를 담을 수 있기 때문에, 결국 시스템이 과부하에 닥치게 될 수 있음. 
예를 들어, 큐를 사용해서 시스템의 처리량을 증가시켰지만, 결국 큐가 넘치게 되면 시스템이 과부하에 닥치게 될 수 있음. 
이러한 문제를 피하기 위해서는 시스템의瓶頸을 찾아내고, back-pressure나 load-shedding을 사용해서 시스템의 처리량을 조절해야 함. 
### 시스템의 처리량을 조절하는 방법 
시스템의 처리량을 조절하는 방법은 여러 가지가 있음. 
back-pressure와 load-shedding을 사용해서 시스템의 처리량을 조절할 수 있음. 
back-pressure를 사용하면 시스템이 처리할 수 있는 데이터의 양을 조절할 수 있고, load-shedding을 사용하면 시스템이 처리할 수 없는 데이터를 버릴 수 있음. 
이러한 방식을 사용하면 시스템의 처리량을 조절할 수 있고, 시스템이 과부하에 닥치지 않도록 도와줄 수 있음. 
또한, 시스템의瓶頸을 찾아내고, 병목 구간을 최적화해서 시스템의 처리량을 증가시킬 수 있음. 
이러한 방법을 사용하면 시스템의 처리량을 조절할 수 있고, 시스템이 과부하에 닥치지 않도록 도와줄 수 있음.
