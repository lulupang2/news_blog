# AI가 보안 취약점 두 가지 문화를 깨부순다!
   últimamente AI가 보안 쪽에 끼친 영향이 너무 크다고 생각함. 그중에서도 특히 보안 취약점 관련해서는 여러모로 일이 복잡해지는 중임. 
   일주일 전 Copy Fail 취약점이 발견됐고, 하현우 김이라는 사람은 그냥 보안 영향만 공유하고 바로 패치를 내놓음.这样 해서 리눅스 네트워킹 쪽에서 흔하게 쓰는 방식인 보안 영향만 공유하는 방식을 쓰는데, 그냥 패치만 공개하고 seriousness는 은은하게만 표현한 거지. 근데 다른 사람도 바꿈을 눈치 채고 보안 영향을 알리는 글을 또 올렸음. 
   그렇게 되니까 보안 취약점 두 가지 culture가 충돌하고 있네. 하나는 'coordinated disclosure' 문화인데, 보안 허점 발견하면 제작자한테만 먼저 말하고 고쳐질 때까지 기다리는 거임. 다른 하나는 '버그는 버그' 문화인데, 그냥 리눅스 커널에서 문제가 생겨도 그저 버그일 뿐이라는 거지. 바로 고칠판에 올리는 거임. 
   그런데 AI가 취약점 찾는 데 너무 잘해줌. 보통 사람들보다 훨씬 빨리 찾기 때문에, 이제는 그냥 고칠판에 올리는 게 더 문제가 되는 거임. 
   예를 들어, 9시간밖에 안 되는 시간에 다른 사람이 똑같은 ESP 취약점을 발견했음. 이렇게하면 embargo 기간이 길어질수록, 문제가 더 커지니까 그냥 짧게 유지하는 게 좋을듯? 
   AI는 디펜더도 빨리 만들 수 있게 해줘서, 짧은 embargo 기간도 충분히 가능해진 거지. 
   그냥 시험 삼아 AI 모델에 테스트 해 봤는데, Gemini 3.1 Pro, ChatGPT-Thinking 5.5, Claude Opus 4.7이 모두 해당 코드가 보안 패치라는 걸 바로 알아챈 거임. 
   요즘 보니 AI는 취약점 찾는 것뿐만 아니라, 디펜더도 빨리 만들 수 있게 해줌. 그러니까 앞으로는 보안 취약점이 발견되더라도 짧은 기간으로 임시방편을 사용하는게 더 나을지도 몰라.
