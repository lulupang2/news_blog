### AI 코딩은 품질 좋은 코드를 쓰는 거야, 빨리 쓰는 거 아니야!
   
   요즘은 AI를 이용해서 코드를 작성하는게 트렌드임. 그런데 문제는 대부분의 개발자들이 AI를 이용해서 코드를 schnell하게 작성하려고 하는데, 그럼 코드 퀄리티는 어떡함? 
   
   LLM이라는 건 대형언어 모델인데, 이 걸 이용해서 코드를 품질 있게 작성할 수 있다고 하는 글임. 작성자는 Mythos라는 모델을 이용해서 코드를 검토하고, 그 결과에 따라 코드를 개선하는 방법을 이야기하고 있당. 
   
   결론은 AI 코딩은 코드를 빠르게 작성하는 방법이 아니라는 거임. AI는 코드를 品質 있게 작성하는 도구일 뿐임. AI가 코드를 작성하는 과정에서 발생하는 버그나 이슈를 파악해서 코드를 개선하는 과정도 중요하다는 거임. 
   
   작성자는 LLM 모델을 이용해서 코드를 검토하고, 그 결과를 토대로 코드를 개선하는 방법을 소개하고 있당. 개발자들이 코드를 작성할 때 AI를 이용해서 품질 좋은 코드를 작성하는 방법을 제안하는 거임. 개빡치는 점은 개발자들이 AI를 이용해서 코드를 작성할 때, 코드의 품질을 고려하지 않고 빠르게 코드를 작성하려고 하는 경우가 많다는 거야. 
   
   그런데 AI 코딩은 코드를 품질 있게 작성하는 도구일 뿐이란 걸 잊지 말자. 코드를 작성할 때 AI를 이용해서 코드를 검토하고, 그 결과를 토대로 코드를 개선하는 방법을 이용해서 코드 퀄리티를 높이자!
