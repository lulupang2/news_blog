### 요약
파이썬에서 라이브러리를 작성할 때, 옵션이나 설정과 같은 데이터를 다루는 경우가 많음. 이런 데이터를 다루는 방법 중 하나가 비공개 데이터 유형(Opaque Types)임. 이 방법을 이용하면 데이터의 내부 구현을 숨기고, 사용자에게는 데이터를 조작할 수 있는 인터페이스만 제공할 수 있음.

### 비공개 데이터 유형(Opaque Types)란?
비공개 데이터 유형(Opaque Types)은 사용자에게 데이터의 내부 구현을 숨기고, 데이터를 조작할 수 있는 인터페이스만 제공하는 데이터 유형임. 이런 데이터 유형을 이용하면 데이터의 내부 구현을 변경하더라도, 사용자에게는 데이터의 인터페이스가 동일하기 때문에, 사용자 코드에 영향을 주지 않을 수 있음.

### ShippingOptions의 예
우리가 작성하는 라이브러리가 물건을 발송하는 경우, ShippingOptions라는 데이터가 필요할 수 있음. 이 데이터는 발송에 필요한 다양한 옵션을 포함하고 있을 수 있음. 하지만, 이러한 옵션을 모두 공개 인터페이스로 제공하면, 사용자 코드가 데이터의 내부 구현에 의존하게 되어, 데이터의 내부 구현을 변경하기 어려워짐.

### 비공개 데이터 유형(Opaque Types)을 이용한 해결 방법
비공개 데이터 유형(Opaque Types)을 이용하면, ShippingOptions 데이터의 내부 구현을 숨기고, 사용자에게는 데이터를 조작할 수 있는 인터페이스만 제공할 수 있음. 이러한 방법을 이용하면, 데이터의 내부 구현을 변경하더라도, 사용자에게는 데이터의 인터페이스가 동일하기 때문에, 사용자 코드에 영향을 주지 않을 수 있음.

### 구현 방법
비공개 데이터 유형(Opaque Types)을 이용하여 ShippingOptions를 구현하는 방법은 다음과 같음:

```python
from dataclasses import dataclass
from typing import NewType

@dataclass
class _RealShipOpts:
    _speed: str

ShippingOptions = NewType('ShippingOptions', _RealShipOpts)

def shipFast() -> ShippingOptions:
    return ShippingOptions(_RealShipOpts('fast'))

def shipNormal() -> ShippingOptions:
    return ShippingOptions(_RealShipOpts('normal'))

def shipSlow() -> ShippingOptions:
    return ShippingOptions(_RealShipOpts('slow'))
```

이 방법을 이용하면, ShippingOptions 데이터의 내부 구현을 숨기고, 사용자에게는 데이터를 조작할 수 있는 인터페이스만 제공할 수 있음. 이를 통해, 데이터의 내부 구현을 변경하더라도, 사용자 코드에 영향을 주지 않을 수 있음.
