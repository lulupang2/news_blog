## 개요

안녕, 코딩 좀 해본 형이야. 오늘 제목 보고 "이 영감탱이가 노망났나? 부동소수점을 ==로 비교한다고?" 하면서 들어온 애들 많지? 

솔직히 말할게. 이거 제목 낚시 맞다. 더 정확한 제목은 **"엡실론(epsilon) 써서 부동소수점 비교하는 것 좀 그만해라"** 정도가 되겠네. 

너네 코딩 처음 배울 때 귀에 딱지가 앉도록 들었을 거야. `bool approxEqual(float x, float y) { return abs(x - y) < 1e-4f; }` 같은 식으로 대충 오차 범위(epsilon) 두고 비교하라고. 근데 내가 15년 넘게 기하학, 그래픽스, 물리 시뮬레이션 같은 빡센 분야에서 굴러보니까 말이야, 저런 엡실론 방식이 진짜 정답인 경우는 한두 번밖에 없었어. 대부분은 그냥 `==`를 써도 되거나, 코드 구조를 고치는 게 훨씬 깔끔하거든.

## 1. 부동소수점은 블랙박스가 아니야

많은 뉴비들이 착안하는 게 뭐냐면, 부동소수점이 무슨 컴퓨터 신의 장난으로 나오는 랜덤한 숫자인 줄 알아. "대충 계산하다 보면 오차 나니까 적당히 비슷하면 같다고 치자"는 식이지.

근데 말이야, 부동소수점은 존나게 **결정론적(deterministic)**이야. CPU 플래그나 컴파일러 옵션이 같으면 결과는 항상 똑같아. 물론 모든 실수를 표현할 수는 없지. 메모리는 유한하니까. 하지만 IEEE 754 표준이라는 게 장난으로 있는 게 아니거든. 

산술 연산 결과가 실제 값에 가장 가까운 부동소수점으로 표현된다는 보장이 있어. 비록 `(0.2f + 0.3f) + 0.4f`랑 `0.2f + (0.3f + 0.4f)`가 결합법칙이 안 성립해서 결과가 쥐꼬리만큼 다를 순 있어도, 그 동작 자체는 아주 예측 가능하다는 거야.

## 2. 엡실론(Epsilon) 비교의 3대 문제점

내가 왜 엡실론 비교를 싫어하냐고? 이유가 세 가지나 돼.

1. **땜질 처방이야:** 근본적인 해결책이 아니라 임시방편일 뿐이야.
2. **디버깅 헬게이트의 시작:** 어떤 곳에선 오차를 1e-4로 잡고, 어떤 곳은 1e-6으로 잡으면 나중에 데이터 꼬였을 때 범인 찾기 진짜 힘들다. 보름달 뜬 날 밤에만 터지는 버그 겪어볼래?
3. **추이성(Transitivity) 파괴:** 이게 제일 빡치는 건데, 보통 알고리즘은 `A=B`고 `B=C`면 `A=C`라고 가정하거든? 근데 엡실론 비교는 이게 안 돼. 이거 때문에 정렬 알고리즘 같은 거 돌리다가 프로그램 뻗는 거 한두 번 본 게 아니다.

## 3. 사례 연구: 격자(Grid) 기반 이동

예를 들어보자. 턴제 게임에서 유닛을 이동시킨다고 쳐. 이동 애니메이션이 다 끝나고 유닛 좌표가 목표 지점 좌표랑 딱 맞으면 다음 턴을 넘겨주고 싶어.

```cpp
// 뉴비의 실수
void update() {
    if (selectedUnit.position != targetCell.center) return;
    // 다음 동작...
}
```

이러면 보간(interpolation) 계산하다가 생긴 미세한 오차 때문에 `==` 비교가 안 먹혀서 게임이 멈출 수 있지. 그럼 빡쳐서 엡실론을 넣겠지?

```cpp
// 땜질 처방
if (distance(selectedUnit.position, targetCell.center) > 1e-4) return;
```

이게 왜 구린지 알아? 1e-4라는 숫자는 그냥 니 뇌피셜이잖아. 애니메이션 방식을 바꾸면 저 숫자도 또 바꿔야 해. 

**진짜 고수는 어떻게 할까?** 데이터 모델이랑 표현(Presentation)을 분리하는 거야. 유닛이 이미 목표 지점에 도착했다는 '상태'는 내부 로직에서 관리하고, 화면에 그려주는 보간 값은 그냥 보여주기용으로만 쓰는 거지. 그러면 엡실론 따위 필요 없이 깔끔하게 논리가 돌아가.

## 요약하자면

부동소수점 비교할 때 무지성으로 엡실론 때려 박지 마. 
1. 왜 비교를 하는지 생각해라.
2. 데이터 구조를 고쳐서 해결할 수 없는지 봐라.
3. 정말 필요하다면, 그 오차 범위가 수학적으로 타당한지 계산해라.

컴퓨터는 거짓말 안 해. 사람이 설계를 대충 하는 게 문제지. 알았어? 가서 코드 다시 짜와.
