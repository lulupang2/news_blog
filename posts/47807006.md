## 1. 개요: 엔스로픽의 조용한 '질소포장'

안녕 형들, 오늘도 흥미로운 소식 하나 들고 왔어. 앤스로픽(Anthropic)이 최근에 **클로드(Claude) 4.7** 내놓으면서 가이드를 뿌렸거든? 거기 보면 새로운 토크나이저가 이전 4.6 모델보다 토큰을 **1.0에서 1.35배** 정도 더 쓸 거라고 적혀 있어.

근데 어떤 용자가 이걸 직접 측정해 봤더니 결과가 골 때려. 실측해보니까 **최대 1.47배**까지 나오네? 가격(Sticker price)은 그대로라는데, 토큰을 더 많이 쓰면 결국 우리 지갑에서 나가는 생돈은 늘어난다는 소리잖아. 이거 완전 과자 봉지에 질소 채워 파는 거랑 뭐가 다르냐? ㅋㅋㅋ

## 2. 실측 데이터: 내 코드가 왜 이렇게 비싸?

필자가 실제 `Claude Code` 사용자가 보내는 데이터들로 테스트를 해봤대. 결과표 한번 봐봐.

### 2.1 실제 사용 환경 (Real-world Claude Code content)
| 콘텐츠 타입 | 4.6 토큰 | 4.7 토큰 | 증가 비율 |
| :--- | :--- | :--- | :--- |
| CLAUDE.md (설정 파일) | 1,399 | 2,021 | **1.445x** |
| 일반적인 프롬프트 | 1,122 | 1,541 | **1.373x** |
| 기술 문서 (Markdown) | 1,209 | 1,654 | **1.368x** |
| Git 로그 | 910 | 1,223 | **1.344x** |
| 파이썬 스택 트레이스 | 1,736 | 2,170 | **1.250x** |

**요약:** 전체적으로 가중치를 둬서 계산해보면 **평균 1.325배** 정도 토큰이 더 들어가. 특히 코드나 기술 문서는 타격이 더 크지.

### 2.2 언어별 특징
- **영어/코드:** 1.20x ~ 1.47x 증가. (단어랑 코드 패턴을 더 잘게 쪼갬)
- **한중일(CJK):** 1.01x 정도로 거의 차이 없음. (동양권 형들은 그나마 다행인가?)
- **스페인어:** 1.35x 증가.

## 3. 왜 이렇게 토큰을 쪼개놓은 거야?

앤스로픽 형들 주장은 이거야. **"토큰을 더 작게 쪼개면 모델이 지시사항을 더 문자 그대로(Literal) 잘 따른다!"**

쉽게 말해서, 단어를 뭉텅이로 보는 게 아니라 글자 하나하나 빡세게 집중해서 보니까 실수를 덜 한다는 거지. 실제로 `IFEval`이라는 벤치마크(엄격한 지시사항 이행 테스트)를 돌려봤더니, 4.7이 4.6보다 **약 5%p 정도 성능 향상**이 있었대. 

근데 말이야, 성능 5% 올리자고 토큰을 30% 더 쓰는 게 가성비가 맞나 싶네? ㄹㅇ...

## 4. 그래서 내 지갑에 생기는 일 (빌링 시뮬레이션)

버그 수정하려고 `Claude Code`를 한 80턴 정도 굴린다고 쳐보자. 캐싱(Prompt Caching) 다 쓴다고 가정하고 계산기를 두드려보면:

- **Claude 4.6 세션 총액:** 약 **$6.65**
- **Claude 4.7 세션 총액:** 약 **$7.86 ~ $8.76**

**결론:** 똑같은 작업을 해도 **약 20~30% 정도 돈이 더 나가.** 만약 유료 플랜(Max) 쓰는 형들이라면 '5시간당 메시지 제한' 쿼터가 훨씬 빨리 녹아내리겠지? 

## 5. 세 줄 요약

1. **클로드 4.7 토크나이저 장난질(?)로 토큰 사용량이 1.3배~1.45배 늘어남.**
2. **이유는 "말을 더 잘 듣게 하려고"라는데, 성능 향상 대비 비용 증가가 뼈아픔.**
3. **캐싱을 써도 전체 비용은 20% 이상 오르니까, API나 유료 플랜 쓰는 형들은 예산 관리 잘해라.**

가성비 따지는 개발자라면 4.6이랑 4.7 사이에서 고민 좀 되겠는데? 암튼 다들 즐코딩하고 지갑 조심해!
