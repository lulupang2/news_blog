### Honker란 무엇임?
Honker는 SQLite랑 비슷한 NOTIFY/LISTEN 기능을 구현해서 Pub/Sub, Task Queue 이런 기능들을 쓸 수 있게 해주는 라이브러리임. Rust로 만들었고, Python, Node.js, Go, Ruby, Bun, Elixir랑 같은 언어들에서 쓸 수 있는 バ인딩이 포함되어 있음. 

### 어떻게 동작하나요?
Honker는 SQLite의 WAL 파일에 이벤트를 발생시키는 방식으로 동작함.这样하면 폴링 없이도 이벤트를 발생시킬 수 있음. SQLite의 WAL 파일에 이벤트를 발생시키는 방식으로 동작하는 거랑, 폴링 없이도 이벤트를 발생시킬 수 있는 점이 핵심인 거 같음. 

### 핵심 기능
- **Pub/Sub**: Honker는 Pub/Sub 모델을 지원함. publish랑 subscribe 이런 함수들로 이벤트를 발생시키고 구독할 수 있음. 
- **Task Queue**: Honker는 Task Queue 모델도 지원함. enqueue랑 dequeue 이런 함수들로 작업들을 추가하고 제거할 수 있음. 
- **Notify**: Honker는 Notify 기능도 지원함. notify 함수로 이벤트를 발생시킬 수 있음. 

### 사용 예
```python
import honker

db = honker.open('app.db')
emails = db.queue('emails')

# 엔큐
with db.transaction() as tx:
    tx.execute('INSERT INTO orders (user_id) VALUES (?)', [42])
    emails.enqueue({'to': 'alice@example.com'}, tx=tx)

# デキュー
async for job in emails.claim('worker-1'):
    try:
        send(job.payload)
        job.ack()
    except Exception as e:
        job.retry(delay_s=60, error=str(e))
```

Honker는 Rust로 구현되어 있고, SQLite의 로드 가능한 扩展으로 작동하는 거임.这样하면 C++랑 Rust만으로도 SQLite랑 동일한 인터페이스를 제공할 수 있음.
