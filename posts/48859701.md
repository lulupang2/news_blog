# LLM이 쓰는 코드도 사람이 유지보수해야 해
LLM이 자동으로 코드를 작성해줘서 좋긴 하지만, 코드 품질은 여전히 내가 책임져야 한다는 걸 깊이 느꼈음. 
LLM이 자동으로 코드를 작성해주면 그만이다 보니, 코드를 작성하면서 지켜야 할 원칙들이란 걸 잊고 있었어. 
예를 들면, 유저의 isActive 같은 속성이 true인지, hasPermission 같은 퍼미션이 있는지, isSuspended 같은 속성이 false인지 등등의 조건들을 체크해야 하는 코드가 여러군데서 필요했어.
그때마다 LLM에게 설명을 해줬고, LLM은 코드를 작성해줬지. 근데 코드가 조금씩 다른 변수 이름이나 로직을 가지고 있어서, 복붙해서 약간 수정해서 쓰고 있었음.
 그런데 이러니까 코드 품질이 떨어지고, 유지보수하기 어려워짐. 
그리고 LLM은 우리가 작성한 코드를 보고, 그 코드에 기반해서 새로운 코드를 생성해줘. 
그러니까 우리가 작성한 코드가 좋은 코드라면, LLM도 좋은 코드를 생성해줌. 하지만 우리가 작성한 코드가 나쁘다면, LLM도 나쁜 코드를 생성해줘. 
LLM이 코드를 생성해주면, 그 코드를 다시 nosotros가 유지보수해야 한다는 사실을 깨달았어. 
즉, LLM이 생성해준 코드도 사람이 유지보수해야 한다는 거임. 
그럼 어떻게 코드를 작성해야 합니까? 
첫째, 코드를 작성할 때는 다른 사람이 유지보수하기 쉽도록 코드를 작성해야 함. 
둘째, 코드를 작성할 때는 LLM이 생성해준 코드도 사람이 유지보수해야 한다는 것을 기억해야 함. 
셋째, 코드 품질을 높이기 위해서는, LLM이 생성해준 코드도 우리가 직접 코드를 작성하는 것과 똑같이 코드 리뷰를 하여야 함. 
这样하면, 코드 품질을 높이고, 유지보수하기 쉬운 코드를 작성할 수 있을 거임. 
개빡치는 점은, LLM이 자동으로 코드를 작성해주니까, 코드 품질에 대해서 신경을 쓰지 않게 된다는 거임. 
그렇다면, LLM을 쓰지 않는 것이 나을까? 
아님, LLM을 쓰는데, 코드 품질에 대해서 신경을 써서, 좋은 코드를 작성해서 LLM에게 학습시켜야 합니까? 
이것은 개발자們이 직접 판단하여야 할 문제임. 
암튼, LLM이 자동으로 코드를 작성해주니까 kode 품질에 대해서 신경을 쓰지 말고, 그냥 코드를 빨리 작성해버리면 안됨. 
코드 품질에 대해서 항상 신경을 쓸-Line 해야함. 
그럼, 코드를 작성하면서, LLM이 생성해준 코드도 우리가 유지보수해야 한다는 것을 항상 기억해야 함. 
그리고, 좋은 코드를 작성해서, LLM에게 좋은 코드를 생성해줄 수 있도록 학습시켜야 함. 
그러면, 좋은 코드를 작성할 수 있고, 유지보수하기 쉬운 코드를 작성할 수 있을 거임.
