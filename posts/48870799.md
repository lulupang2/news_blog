# 요약
     * 요약하자면 코드 최적화 이야기인데 최근에 퀵 정렬 최적화 구현하면서 Modern Compiler 에서 loop 를 optimize 하는데 branch-free instruction 쓸 수 잇는거 발견함 
     * 그래서 퀵 정렬 구현 예시 코드 들고 옴 
     * Clang 쓸 때pecially최적화 잘 되는데 C++ std::sort 보다 3배 정도 빠름 
     * 그래서 sorting network 로 loop unrolling 을 사용해서 최적화한 코드를 다시_rewrite_한 것도 추가함 
     * 근데 C++ std::sort 는 1.33초 걸리는데 이건 4.39초 걸림 
     * 요즘 컴퓨팅 파워가 엄청나서 걍 std::sort 써도 되지않나 싶은데 음.. 
     # 디테일 
     * 최근의 컴파일러(특히 Clang)는 빠르고 branch-free instruction을 사용하여 루프를 최적화하지만, 올바른 프로그래밍 스타일을 사용해야 합니다. 
     * 퀵 정렬의 최적화된 구현 예시 code 들고 옴 
     * 퀵 정렬의 일부인 partition 함수에서 loop unrolling 및 sorting network를 사용하여 코드를 최적화 
     * C++ std::sort 함수와 비교하여 성능을 테스트 
     * MacOS/M1에서 Clangcompiler로 빌드하고 -O3플래그 사용했을 때 테스트 결과 std::sort 보다 빠름
