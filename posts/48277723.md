# RAFT 합의 알고리즘 개량
   여태까지 RAFT 합의 알고리즘은 다수 노드 참여 必要야. 그런데 요즘은 노드가少數여도 됨. 
   ### RAFT 합의 알고리즘의 원리
   RAFT 합의 알고리즘의 핵심은 다수 노드간의 합의야. 노드 간의 합의를 통해 시스템의 일관성을 유지할 수 있어. 
   ###少數 노드 합의 알고리즘
   이제少數 노드만으로도 합의를 달성할 수 있는 알고리즘을 만들었다. 
   이것은 유한 проек티브 평면(Finite Projective Plane)을 이용해서 구현했어. 
   유한 프로젝티브 평면은 어떤 두 점 사이에Exactly 하나의 선이 존재하고, 어떤 두 선 사이에도 Exactly 하나의 점이 존재하는 거임. 
   ### 알고리즘의 구현
   알고리즘의 구현은 간단해. 노드를 점으로, 노드 간의 hợp의를 선으로 생각했어. 
   ### 示例
   7개의 노드로 예를 들어서 설명할게. 
   - 노드 1, 2, 3이 합의하면 bloc B1이 형성돼. 
   - 노드 4, 5, 6이 합의하면 bloc B4가 형성돼. 
   - 노드 1, 2, 3과 노드 4, 5, 6은 모두 合의해야 해. 
   ### 장단점
   - 장점: 少數 노드만으로도 합의를 달성할 수 있어. 
   - 단점: 노드 간의 합의가 복잡해질 수 있어.
