# Introduction
LLM이 코드를 개빠르게 만들긴 하지만 정확하게 뭘 만들고 싶은지 경계를 설정해 줘야함. DSL과 도메인 추상화가 이런 경계 설정에 큰 도움을 주는 것 같은디.
 Tickloom 이라는 DSL 예시를 봤는데 이거挺 쓸만하더라. DSL을 사용해서 LLM이 코드를 생성할 때 도움되는 점들이 몇몇 있어.
 DSL이 spole LLM과 잘 맞는 이유
 * DSL은 narrow한 도메인에 특화된 언어이기 때문에 LLM이 이해하기 쉽다 이말임.
 * DSL은 deterministic validator를 제공하기 때문에 LLM이 생성한 코드를 자동으로 검증할 수 있음.
 Tickloom_example
 Tickloom은 분산 시스템의 동작을 표현하기 위한 DSL이야. 이 DSL을 사용해서 LLM이 코드를 생성하는 것을 봐볼께.
 * DSL을 사용해서 Powerpoint презентацию 만들기
예를 들어distributed system을 설명하기 위해 sequence 다이어그램을 많이 사용하잖아. 그런데 sequence 다이어그램 하나만 보여주면 이해하기 힘들잖아. 그래서 sequence 다이어그램을 step별로 보여주는 툴을 만들었다이거든. LLM을 사용해서 YAML 명세를 적으면 Powerpoint презентацию를 자동으로 생성해준당.
 DSL이 spole LLM 쓰는법
 1. DSL을 설계하고, validator를 만든다.
 2. LLM을 사용해서 DSL로 코드를 생성한다.
 3. validator를 사용해서 생성된 코드를 검증한다.
