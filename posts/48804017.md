> **🔗 원문 보러가기:** [Why low-latency Java still requires discipline?](https://chronicle.software/insights/blogs/why-low-latency-java-still-requires-discipline)

자바는 높은 성능을 제공하지만, 낮은 대기 시간을 달성하려면 엄청난 노력이 필요하다. 
   여기에는 코드 최적화, 적절한 데이터 구조 선택, 멀티 쓰레딩 등이 포함된다. 
   그리고 모두가 다루기 힘든 기술적 문제는 아니었다. 
   개발자가 코드를 작성할 때 항상 낮은 대기 시간을 고려해야 하는 것은 당연한 일이지만, 
   이는 쉽지 않은 작업이다. 
   특히 대규모 시스템을 개발할 때는 더더욱如此이다. 
   그러면 자바에서 낮은 대기 시간을 달성하기 위한 방법은 무엇일까? 
   일단 가장 중요한 것은 데이터 구조를 올바르게 선택하는 것이다. 
   적절한 데이터 구조를 선택하면 코드의 성능을 크게 개선할 수 있다. 
   하지만 이는 쉽지 않은 작업이다. 
   개발자가 코드를 작성할 때 항상 올바른 데이터 구조를 선택해야 하는 것은 당연한 일이지만, 
   이는 쉽지 않은 작업이다. 
   특히 대규모 시스템을 개발할 때는 더더욱 그렇다. 
   그리고 또 하나의 중요한 점은 멀티 쓰레딩이다. 
   멀티 쓰레딩을 사용하면 코드의 성능을 크게 개선할 수 있다. 
   하지만 이는 쉽지 않은 작업이다. 
   개발자가 코드를 작성할 때 항상 멀티 쓰레딩을 고려해야 하는 것은 당연한 일이지만, 
   이는 쉽지 않은 작업이다. 
   특히 대규모 시스템을 개발할 때는 더더욱 그렇다. 
   결국 자바에서 낮은 대기 시간을 달성하기 위해서는 엄청난 노력이 필요하다. 
   하지만 이는 가능하다. 
   개발자가 코드를 작성할 때 항상 낮은 대기 시간을 고려하고, 
   데이터 구조와 멀티 쓰레딩을 올바르게 사용한다면, 
   높은 성능을 달성할 수 있다. 
   그러니 모두가 자바에서 낮은 대기 시간을 달성하기 위해 노력해 보길 바란다.
