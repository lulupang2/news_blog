# 만들기
개발자로서 코드를 작성하는 것은非常한 만족감을 준다. 하지만 AI로 코드를 생성하거나, 다른人が 나의 요청으로 코드를 작성한다고 해서 내가 만들었다고 생각하지 않는다. 그래서 슬픈 일이긴 한데, 만들기에는 이러한 점들이 포함되는 것이긴함.
## AI 개발자들의 분열
AI 개발자들은 '크래프트의 손실', '수동 코딩의 손실', '저레벨 문제 해결의 손실' 등이 발생한다고 생각한다. 하지만 이러한 것은 사실이고, 개발자们에는 다양한 생각이 있을 수 있다.
## 내 관점
나는 80년대 마이크로컴퓨터時代에 시작해서, 현재까지 20년 이상의 업무 경험과, Hewlett-Packard, 스타트업, Activision 등에서 일했다. 또한 CS 강사로도 활동하고 있으며, 내 성향은 65%가 AI-UTOPIA/도وم 스케일에 있다. 나는 때때로 Claude Code 사용자이기도 하며, 손으로 코드를 작성하기도 한다.
## 내 최근 hoạt動
우주 전함과 외계인으로 가득 찬 SF 소설을 쓰고 있으며, 다음과 같은 코드를 작성했다:
```rust
fn try_move(&mut self, dx: i32, dy: i32) {
    // ...
}
```
이 코드는 내가 직접 작성한 것이라서, 매우 자랑스럽다. 하지만 Claude와 같은 AI를 사용하여 코드를 생성하면, 내가 만들었다고 생각하지 않는다.
## 만들기에는อะไร가 있을까
개발자로서, 코드를 작성하는 것은非常한 성취감을 준다. 하지만 AI로 코드를 생성하거나, 다른人が 나의 요청으로 코드를 작성한다고 해서 내가 만들었다고 생각하지 않는다. 이에는 여러 가지 이유가 있지만,主要한 이유는 '만들기'에는 내가 직접 코드를 작성하여, समसヤ를 해결하는 것이 포함되어 있다.
## 다른 사람들의 관점
다른 사람들은, 내가 만들었다고 생각하는 코드를 AI로 생성하거나, 다른 사람이 작성한다고 해서, 내가 만들었다고 생각할 수 있다. 하지만, 이러한 관점은 나와 다르다.
## 결론
개발자로서, 코드를 작성하는 것은非常한 만족감을 준다. 하지만 AI로 코드를 생성하거나, 다른人が 나의 요청으로 코드를 작성한다고 해서 내가 만들었다고 생각하지 않는다. 나는, 내가 직접 코드를 작성하여, 문제를 해결하는 것이 포함되어 있는, '만들기'를 좋아한다.
