> **🔗 원문 보러가기:** [Dotcl: Common Lisp Implementation on .NET](https://github.com/dotcl/dotcl)


Dotcl은 .NET 프레임워크에서 돌아가는 리스프 구현체야. 리스프 소스코드를 CIL로 컴파일해서 JIT에서 실행시켜줘. 그러면 같은 리스프 이미지로 윈도우, 맥, 리눅스에서 돌아가게 해줘. ANSI 리스프 표준에도 거의 맞춰놨고, ansi-test 수트에서도 검증까지 했어.

dotcl은 몇 가지 장점이 있는데, 
 embeddable 하다는 점이 하나야. C#이나 F#, VB.NET 프로젝트에서 쓸 수 있어. 리스프 코드를評価하고, 호출하고, 돌아가게 해줘. 
.net 코드도 리스프로 작성할 수 있어. dotnet: 패키지를 쓰면 .NET 타입에 직접 접근할 수 있고, (dotnet:new "System.Text.StringBuilder"), (dotnet:invoke sb "Append" "x"), (dotnet:static "System.Math" "Sin" 1.0) 이런식으로 사용할 수 있다고 해.
dotnet:define-class 로 .NET 타입을 리스프에서 상속할 수도 있어. 컴파일러가 진짜 .NET 클래스를 만들어주니까, MAUI, ASP.NET Core, MonoGame 같은 프레임워크에서도 그냥 일반적인 서브클래스처럼扱할 수 있어.

그리고 크로스 플랫폼에서도 쓸 수 있어. NuGet 패키지도 리스프에서 쓸 수 있고, Quicklisp 라이브러리도 대부분 쓸 수 있어. asdf, alexandria 이런 라이브러리도 그냥 로드할 수 있어.

사용하는 방법은 간단해.一度부트스트랩하고 나면.self-hosting도 가능하다고 해. DOTCL_LISP=dotcl로 컴파일러를 다시 빌드할 수 있어.

필요한건 .NET SDK 10+랑 Roswell thôi. macOS, Ubuntu, Debian, Windows에서도 설치 가능하대.Samples도 있는데, .NET MAUI 앱, ASP.NET Core 컨트롤러, MonoGame 게임, MCP 서버 이런거에 리스프를 사용하는 예시가 있다고 해.

아키텍처는 컴파일러랑 런타임으로 나눠져 있어. 컴파일러는 리스프 소스코드를 CIL로 컴파일하고, 런타임은 오브젝트 표현, 리더, CIL 어셈블러, 표준 라이브러리 함수 이런거를 담당해. 부트스트랩은 Roswell SBCL로 컴파일러를 컴파일해서 .NET 런타임에 로드하고, 그 다음부터는 self-hosting 가능하다고 해.
