# 데노 데스크탑이 뭐냐면
     
     데노 프로젝트를 데스크탑 앱으로 만들어서 배포할 수 있게 해주는 툴 임.
     
     데노 프로젝트란거는 단순한 타입스크립트 파일 하나에서부터 넥스트.js 앱까지 모두 포함이 돼.
     
     결과물은 각 플랫폼 별로 코드, 데노 런타임, 웹 렌더링 엔진을 하나로 묶은 배포용 바이너리임.
     
     # 데노 데스크탑은 왜 필요해?
     
     웹 기술이 가장 잘 알려진 UI 툴킷이라고 하는데, 웹 스택 기반 데스크탑 앱들은 이점을 활용하면서 각기 장단점들이 있어.
     
     일단 웹 기술은 장점이 많지만, 이걸 기반으로 하는 데스크탑 앱 툴킷들은Electron, Tauri, Electrobun 등이 있는데 각자 단점들이 있어.
     
     큰 바이너리 크기, 플랫폼 지원이 부족하거나, 자바스크립트 생태계가 부족해서 update도 안되고, 프레임워크도 안되고..
     
     # 데노 데스크탑의 특징은?
     
     일단 기본적으로는 작고, 노드 호환성이 있어.
     
     기본 웹뷰 백엔드는 운영체제의 웹뷰를 사용해서 바이너리 사이즈가 작고, 데노의 노드 호환层을 통해서 npm 생태계도 쓸 수 있드라.
     
     크로미움 백엔드를 쓰고 싶으면 옵션으로 선택할 수도 있고.
     
     자동으로 프레임워크를 감지해주기도 하고,
     
     기존 웹 프로젝트를 데스크탑으로移行시키려고 할 때 코드를 수정할 필요가 없고,
     
     프로세스 간 통신이 아니라 in-process Binding을 지원해서 IPC 대신에 사용할 수 있음.
     
     맥, 윈도, 리눅스 모두를 하나의 머신에서跨编译할 수도 있고, 백엔드는 로컬에서 빌드하는 게 아니라 필요한 때 다운로드해서 사용해.
     
     업데이트도 자동으로 바이너리 차이만을 전송해서 자동 롤백 이런 건 모두 지원해.
     
     # 데노 데스크탑 사용해보기
     
     일단 데노를最新 버전으로 업데이트 해주고,
     
     다음과 같은 코드를 만들어.
     ```typescript
     Deno.serve(() => new Response('<h1>HELLO DESKTOP</h1>', { headers: { 'content-type': 'text/html' } })
     );
     ```
     
     >_deno desktop main.ts
     
     # macOS / Linux
     >_./main
     # Windows
     >_.main.exe
