### 코드 라인 수의 허상을 깨달아라

요즘.ai 관련 기업들이 ai로 생성한 코드 라인 수를 자랑하는 모습을 보면 웃음밖에 안 나온다. 코드 라인이 많다고 무슨 소용일까? 오히려 코드가 간결하고 유지보수가 쉬우면 더 좋지 않을까? 
그런데도 코드 라인 수를 자랑하는 이유는 어쩔까? 그냥 숫자만 보면 기분이 좋을 뿐인 것 같다. 

### 숫자는 못속여

코드 라인 수를 기준으로 개발자의 실력을 평가하는 건 너무나 유치한 발상이다. 개발자는 코드를 짜는 사람이 아니다. 개발자는 코드로 고객의 문제를 해결해주는 사람이다. 코드 라인을 늘리는 건 코드를 지우는 것보다 쉬울 뿐이다. 

### ai도 마찬가지

ai로 코드를 생성하는 건 코드 라인을 늘리는 것과 다를 바 없다. ai가 코드를 생성한다고 해서 고객의 문제가 해결되는 건 아니지 않는가? 오히려 코드를 읽고 이해하는 데 더 많은 시간이 걸릴 수도 있징. 
그렇다고 ai를 사용하면 안 된다는 소린 아니다. 하지만 ai를 사용한다고 해서 코드 라인 수를 자랑하는 건 어리석은 짓이다. 

### 결론

코드 라린 수를 자랑하는 건 어리석은 짓이다. 우리는 코드를 짜는 것을 넘어서 고객의 문제를 해결하는 사람들이다. 코드 라인이 많다고 무슨 소용일까? 오히려 코드가 간결하고 유지보수가 쉬우면 더 좋지 않을까? 
ai를 사용한다고 해서 코드 라인 수를 자랑하는 건 그저 숫자를 위한 숫자일 뿐이다. 우리는 ai를 사용하여 고객의 문제를 더 잘 해결하는 방법을 찾아야 한다. kode
