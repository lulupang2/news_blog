### 와사비 웹어셈블리 훈수충들을 위한 웹어셈블리 최신 기술 정보
요즘 웹어셈블리에서 할 수 있는 게 너무 많아졌지? 이제는 tar 압축파일을 파일시스템으로 마운트하는 것도 가능하だって? ㅋㄷㅋㄷ

웹어셈블리에서 tar 압축파일을 파일시스템으로 마운트하기 위해서는 tar-vfs-index라는 패키지를 사용하면 된대. 이 패키지는 tar 압축파일을 읽고, 파일의 위치와 크기를 담은 JSON 파일을 만들어주는데, 이 파일을 이용해서 파일시스템으로 마운트할 수 있대.

tar-vfs-index를 사용하는 방법은 간단해. 먼저 npm을 이용해서 패키지를 설치하고, tar-vfs-index 명령어를 실행하면 돼.
```bash
npm install tar-vfs-index
npx tar-vfs-index archive.tar.gz
```
이 명령어는 archive.tar.gz 파일을 읽고, 파일의 위치와 크기를 담은 JSON 파일을 만들어줘. 이 파일을 이용해서 파일시스템으로 마운트할 수 있지.

파일시스템으로 마운트하는 방법은 간단해. 먼저, tar-vfs-index 패키지를 이용해서 JSON 파일을 만들고, 이 파일을 이용해서 파일시스템으로 마운트하면 된대.
```javascript
const metadata = await fetch('archive.tar.gz.json').then(res => res.json());
const blob = await fetch('archive.tar.gz').then(res => res.blob());
FS.mount(WORKERFS, { packages: [{ metadata, blob }] }, '/pkg');
```
이 코드는 archive.tar.gz.json 파일을 읽고, archive.tar.gz 파일을 읽은 뒤 파일시스템으로 마운트해 줘. 이제는 tar 압축파일을 풀지 않고도 파일을 읽을 수 있지.

이 기술은 웹어셈블리에서 사용할 수 있어. 특히, 웹어셈블리에서 파일시스템을 사용해야 하는 경우에 유용할 것 같아. 웹어셈블리 훈수충들을 위한 꿀팁이라고 생각해 ㅋㄷㅋㄷ

### 결론
