클로저에 대해서 알아보다가 한 달째 쓴 리뷰 같은 거임 ㅋㅋㅋ
처음에는 Lisp 언어들 중에 클로저가 제일 쓰기 편하다고 생각했다.
Common Lisp 보다 클로저가 더 좋은 점은 
- 클로저가 더 일관성이 있는데, 예를 들어 seq라는 추상화 덕분에 클로저에서는 리스트, 배열, 해시맵 등에 대해서 동일한 함수를 사용할 수 있음.
- Scheme 보다는 클로저가 더 많은 기능을 제공하는데, 클로저는 hosted 언어로 JVM 위에서 돌아가기 때문에 자바의 생태계를 그대로 사용할 수 있음.
클로저가 좋은 점
- Clojure는 Common Lisp보다 더 일관된 언어结构을 가지고 있다고 생각함. 
예를 들어 seq라는 추상화 덕분에 리스트, 배열, 해시맵 등에 대해서 동일한 함수를 사용할 수 있음.
eq, eql, equal, equalp 같은 함수들 대신에 단순하게 =를 사용할 수 있음.
- Ergonomic data structures: 리스트, 벡터, 해시맵, 세트 등이 모두 일급 컬렉션으로 사용할 수 있음.
클로저가 나쁜 점
- 너무 많은 문법을 가지고 있다고 생각 있는데, 예를 들어 () [] {} #{}등의 다양한 컬렉션 문법이 있다는 점.
- 클로저를 사용하다보면 자바를 모르면 안되는데, 자바를 몰라서 힘들다는점 ㅋㅋ
하지만 클로저는 그래도 사용하기 편하고 재밌고, 프로젝트babashka 덕분에 속도도 빠르다는 장점이 있음.
클로저는 아직까지는 사용할 예정임. 프로젝트 이üler에 클로저로 참여해서 코드를 작성하고 있는데 재밌당 ㄹㅇ
