# 엘라스틱서치로 에이전트 기억 저장소 만들기
   엘라스틱서치로 에이전트 기억 저장소를 만들었다는 뉴스에요. 
   이 에이전트 기억 저장소는 0.89 리콜률을 달성했댔음. 
   개쩐다고 생각하네염 ㅋ 
   
   # 에이전트 기억이 뭐냐면 
   에이전트가 사용자와 대화하면서 쌓아가는 기억을 저장하는 곳이지. 
   이 기억은 에이전트가 사용자에게 더 अच्छ은 서비스를 제공하기 위해 필요하댈세. 
   
   # 구조는 어쩌게?
   이 에이전트 기억 저장소는 세 가지 종류의 기억을 저장하는데 
   - 에피소드 기억 (사용자와의 대화 이력)
   - 의미 기억 (사용자에 대한 정보)
   - 절차 기억 (사용자와의 대화에서 얻은 지식)
   이렇게 세 가지 기억을 저장하는데, 
   각 기억은 별도의 인덱스에 저장되어서 
   에이전트가 쉽게 접근할 수 있도록 설계되었댈세.
   
   # 어떻게 동작하냐면 
   사용자가 질문을 하면, 
   에이전트는 이 에이전트 기억 저장소를 뒤져서 관련된 정보를 찾고 
   관련된 정보를 찾아서 사용자에게 답변을 해준댈세. 
   이过程에서 0.89 리콜률을 달성해서 꽤 좋은 성능을 보인댈세. 
   개쩐다고 생각하네염 ㅋㅋ 
   
   # 마치며 
   이 에이전트 기억 저장소는 엘라스틱서치를 기반으로 작동하기 때문에 
   확장성과 유연성이 뛰어나고, 
   쉽게 구축하고 관리할 수 있댈세. 
   앞으로 더 많은 곳에서 사용될 수도 있을 것 같음. 
   ㄹㅇ 기대된다.
