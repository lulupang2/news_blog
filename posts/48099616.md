# 라이브러리 소개
개빡센 자바 레코드를 네이티브 메모리로 매핑하는 라이브러리 タイプドメモリ 등장 ㅋㅋㅋㅋ

   이 라이브러리는 자바 레코드를 네이티브 메모리로 매핑하기 위해 만들어졌는데, 자바의 FFM(외국어 함수 및 메모리) API를利用해서 구현됐MethodImplt

   
# 장점
이 라이브러리를 사용하면, 자바 레코드를 네이티브 메모리로 매핑할 때, 수동으로 레이아웃과 오프셋을 관리하지 않아도 돼 ㅋㅋㅋㅋ

   이 라이브러리는 타입안전한 추상을 제공하면서도, 네이티브 호환성을 유지할 수 있게 해줌. 그리고, 데이터 지향적인 프로그래밍과 네이티브 인터럽트를 지원해주는 건 기본임 ㅋ

   
# 사용예시
```java
import module com.mamba.typedmemory; 
record Point(float x, float y) {} 
void main() { 
    try (Arena arena = Arena.ofConfined()) { 
        Mem<Point> points = Mem.of(Point.class, arena, 10); 
        points.set(0, new Point(5, 3)); 
        Point point = points.get(0); 
        IO.println(point); 
    } 
}
``` 
# 라이브러리 특징
* 자바 레코드를 네이티브 메모리로 매핑 
* 메모리 할당 및 생명주기 관리 
* 네이티브 인터럽트 지원 
* 데이터 지향적인 프로그래밍 지원 
* 성능 최적화 

   
# 라이브러리 설치 방법
.maven 프로젝트라면, `pom.xml` 파일에 아래의 의존성을 추가하면 됨. 
```xml
<dependency> 
    <groupId>io.github.mambastudio</groupId> 
    <artifactId>typedmemory</artifactId> 
    <version>0.1.0</version> 
</dependency>
```
