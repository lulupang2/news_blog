### AMD 오토업데이트에 구멍이 뻥 났다는데 AMD가 왜 고치지 않냐고요 ㅋㅋㅋㅋ

새로 산 게임용 컴퓨터에서 AMD 오토업데이트 소프트웨어 때문에 콘솔 창이 계속 튀어나와서 기분이 존나 나빴음ㅋㅋㅋ
그래서 그 소프트웨어 디컴파일해서 어떤 방식으로 작동하는지 보고 싶었는데, 의외로 Remote Code Execution(RCE) 취약점을 발견함ㅇㅇ

AMD 오토업데이트가 사용하는 更新 URL이 app.config 파일에 저장되어 있어서 보안상 문제가 있다는 생각은 했지만,HTTPS 사용 중이라 괜찮다고 생각했음.
근데 그 XML URL을 웹 브라우저에서 열어보니,_executable 다운로드 URL이 다 HTTP로 돼있었음 ㅋㅋㅋㅋ
이럼 중간자 공격(MITM) 하면 원하는 코드를 돌릴 수 있겠네? 싶었는데, AMD측은 뭐라고 할까?

 AMD가 코드를 검증하는 과정은 없는 건가? 싶어서 코드를 확인해봤더니, 코드 검증 과정은 없는 거임 ㅋㅋㅋㅋ
그냥 다운로드 받은 파일을 바로 실행하는 거임ㅇㅇ
이건 너무 심한데, AMD에 리포트를 해야겠다고 생각했음.
근데 AMD側 버그 번티 프로그램에서는 중간자 공격은 범위 외라고 닫아버림 ㅋㅋㅋㅋ

 UPDATE! 
Hacker News에 포스트를 올렸더니 AMD측에서 다시 연락와서 확인할테니 기다리라고 함.
원래대로면 그냥 넘어가야 했는데, AMD측에서 처리를 안 해줘서 불편한 당사자가 직접 고치라고 만들면됨 ㅋㅋㅋㅋ
 AMD측에서 CVE 발급하고, 고침.
근데 보안 연구자 이름은 알려줄 생각이 없음 ㅋㅋㅋㅋ

### 언제까지 기달려야 하나요?

 AMD측에서 어떻게 할건지 물어봤더니, 다른 툴에도 영향이 있다면서 더 시간을 요청함.
그냥 요청만 하고 기간은 안 알려줌.
근데 보통 90일이 표준이니까, 90일 뒤에 다시 묻고 있음 ㅋㅋㅋㅋ
AMD측에서는 더 길게 기달리라고 하길래, 124일이 지나서야 업데이트를 하겠다고 함.
124일 동안 AMD측에서 연락 없길래, 다시 연락하니까 업데이트를 할 거라고 하던데, 업데이트 내용물을 안 알려줌 ㅋㅋㅋㅋ
업데이트 내용물은 리포트 작성자만 알고 있음.

### 업데이트는 어떤식으로 된 거죠?

 AMD측에서 AutoUpdate 기능을 제거하고 Ryzen Master에 옮겨서 HTTPS로 통신하게 바꿨음.
근데 업데이트한 뒤에 코드가 잘못돼서 AutoUpdater가 작동을 안 함 ㅋㅋㅋㅋ
그냥 업데이트가 안 되어서 Catch-22 같은 상황이 되었음.
업데이트 하려면 업데이트를 해야 하는데, 업데이트가 안 되니까 안 되는 상황임 ㅋㅋㅋㅋ
암튼 AMD 쓰는 사람들이면 그냥 쓰지 말고 다 지우고 새로 깔아야함.
 AMD측에서 업데이트를 했는데, 그냥 CRC-32 체크를 해서 업로드 파일 검증하던데, 얘네들은 보안에 관심이 없는 거 같은데 말임? ㅋㅋㅋㅋ
