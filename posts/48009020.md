# 모네로의 POW: RANDOMX
   모네로의 POW는 RandomX 라고 불림. 비트코인과 다르게 단순한 해시 함수를 반복하는 것이 아니라, 가상 머신에서 랜덜한 프로그램을 실행하고 메모리를 많이 사용하는 방식임.
  (RandomX는 일반 CPU에 친화적인 방식으로 설계됨. 반면 비트코인의 SHA-256은 특별한 칩을 만들기 좋음)

   ## 간단 요약
   모네로에서 블록을 만들기 위해 다음과 같은 과정을 따름:
   - 후보 블록 헤더와 난스 값을 해시하여 시드 값을 생성
   - 가상 머신에서 랜덤한 프로그램을 실행
   - 최종 결과를 해시하여 네트워크 목표 값보다 낮으면 블록이 유효함

   ##왜 모네로가 단순 해시 함수를 사용하지 않는가?
   해시 함수가 정해져 있으면 하드웨어 디자이너들이 그 함수를 최적화 해서 빠르고 저렴하게 만드는 것이容易하다는 거야. 
   (비트코인에서 그랬음. SHA-256 ASIC 등장) 
   모네로는 böyle 하지 않고 RANDOMX를 도입해서CPU 친화적인 POW를 만들었어.

   ## RANDOMX의核心 아이디어
   RANDOMX는 CPU의 다양한 기능을 사용해서 POW를 구성. 
   - CPU 캐시
   - 정수 연산
   - 부동소수점 연산
   - 분기 처리
   RANDOMX는 이러한 기능들을 사용해서 가상 머신에서 랜덤한 프로그램을 실행.

   ## RANDOMX의详细 과정
   1. **Cache 생성**: 키 값을 Argon2d 함수로 해시하여 캐시를 생성.
   2. **Dataset 생성**: 캐시를 사용해서 대용량 Dataset을 생성.
   3. **Scratchpad 초기화**: 블록 입력 값을 해시하여 Scratchpad을 초기화.
   4. **랜덤 프로그램 생성**: Scratchpad을 사용해서 랜덤한 프로그램을 생성.
   5. **프로그램 실행**: 가상 머신에서 프로그램을 실행.
   6. **최종 결과 해시**: 프로그램 실행 결과를 해시하여 최종 결과를 얻음.

   # 결론
   모네로의 RAMDOMX는 CPU 친화적인 방식으로 설계된 POW 알고리즘이고, ASIC 저항성을 갖는다는 점에서 매우 흥미롭다. 
   RANDOMX의 디자인은 가상 머신에서 랜덤한 프로그램을 실행하고 메모리를 많이 사용하는 방식으로, 일반 CPU에 적합하고 특별한 칩을 만드는 것을 어렵게 만드는 방식인 거야. 
  RANDOMX는 비트코인에서 나타난 ASIC 저항성 문제를 해결하고, 동시에 일반 사용자가 참여하기 쉬운 POW 시스템을 제공하는 것을 목표로 함.
