# Restartable Sequences 살펴보기
Restartable Sequences는 리눅스 4.18 버전 이후부터 지원하는 기능으로, 기존에 스레드 세이프티를 유지하기 위해 사용했던 뮤텍스나 아토믹 오퍼레이션 없이도 스레드 세이프한 자료구조를 구현할 수 있어

##Restartable Sequences의 개념
Restartable Sequences는 커널과 사용자 공간 간에 공유된 메모리 영역을 통해 작동한다. 이 공유 메모리 영역은 rseq_cs 필드와 rseq_cs 필드의 값을 확인하여 스레드가 중단되거나 재시작 되어야 하는지를 판단하는 역할을 해

## Restartable Sequences의 사용법
Restartable Sequences를 사용하는 방법은 다음과 같음
1.  rseq_cs 필드를 설정하여 공유 메모리 영역을 지정
2.  커널이 rseq_cs 필드를 확인하여 스레드가 중단되거나 재시작 되어야 하는지 판단
3.  Restartable Sequences를 사용하여 스레드 세이프한 자료 구조 구현

##Restartable Sequences의 이점
Restartable Sequences는 기존의 스레드 세이프티 유지 방법에 비해 많은 이점이 있다
1.  뮤텍스나 아토믹 오퍼레이션 없이도 스레드 세이프한 자료 구조를 구현할 수 있음
2.  스레드 간의 상호 배제를 최소화하여 성능 개선

## Restartable Sequences의 예시
Restartable Sequences의 예시는 다음과 같음
```c
// rseq_cs 필드를 설정하여 공유 메모리 영역을 지정
rseq_cs = (void*)0x10000000;

// 커널이 rseq_cs 필드를 확인하여 스레드가 중단되거나 재시작 되어야 하는지 판단
if (rseq_cs != NULL) {
    // Restartable Sequences를 사용하여 스레드 세이프한 자료 구조 구현
    //...
}
```

## Restartable Sequences의 결론
Restartable Sequences는 리눅스 커널 4.18 버전 이후부터 지원하는 기능으로, 기존에 스레드 세이프티를 유지하기 위해 사용했던 뮤텍스나 아토믹 오퍼레이션 없이도 스레드 세이프한 자료구조를 구현할 수 있는 мощ한 도구임
Restartable Sequences를 사용하여 스레드 세이프한 자료구조를 구현하면, 스레드 간의 상호 배제를 최소화하여 성능 개선과 스레드 세이프티를 향상시킬 수 있음
