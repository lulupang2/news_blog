# 스피넬: 루비의 최적화된 네이티브 컴파일러
스피넬은 루비 소스 코드를獨立적인 네이티브 실행 파일로 컴파일해줌. 그리고 전체 프로그램 타입 추론을 수행하고 최적화된 C 코드를 생성해서, CRuby보다 훨씬 빠르게 동작함 ㅋㅋㅋ. 가장 중요한 점은 스스로를 컴파일할 수 있다는 거임. 즉, 컴파일러 백엔드는 루비로 작성되어 있고, 스스로를 네이티브 바이너리로 컴파일할 수 있단 거지.

## 작동 원리
루비 소스 코드가 들어오면, Prism이라는 라이브러리를 사용해서 소스 코드를 AST로 파싱하고, 그.ast 파일을 생성함. 그 다음에, AST 파일을 받아서 C 코드를 생성하고, 그 C 코드를 컴파일해서 네이티브 바이너리를 만듦. 그리고 그 네이티브 바이너리는 독립적인 실행 파일이 되어서,(runtime 의존성 없이) 바로 실행할 수 있음.

## 예시
```ruby
def fib(n)
  if n < 2
    n
  else
    fib(n - 1) + fib(n - 2)
  end
end

puts fib(34)
```
이 루비 코드를 스피넬로 컴파일하고 실행하면, 즉시 결과가 출력됨. CRuby로 실행하면, 훨씬 느릴텐데, 스피넬로 컴파일하면 최적화된 네이티브 코드로 실행되기 때문에, 훨씬 빠르게 동작함.

## 벤치마크 결과
스피넬은 CRuby보다 평균 11.6배 빠르다고 하는데,具体적인 벤치마크 결과는 아래와 같음.
* life (Conway's GoL): 20ms -> 1,733ms (86.7배 빠름)
* ackermann: 5ms -> 374ms (74.8배 빠름)
* mandelbrot: 25ms -> 1,453ms (58.1배 빠름)
* ... 등등

스피넬은 루비 개발자들에게 좋은 소식임. 이제 루بی로 작성한 코드를 최적화된 네이티브 코드로 컴파일할 수 있으니까, 성능에 민감한 부분은 스피넬로 컴파일해서 사용할 수 있을 것이고, 개발效率와 성능을 모두 잡을 수 있을듯 ㅋㅋㅋ
