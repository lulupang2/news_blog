### 내부 서비스용 TLS 인증서 발급하는 방법 정리
내부 서비스용 TLS 인증서 발급하는 방법에 대해선 다들 잘 알고 있지만, 막상 해보면 어려운 게 tls 인증서 발급임ㅇㅇ. 
그래서 오늘은 tls 인증서 발급하는 방법에 대해 정리해 볼 거임. 
내부 서비스용 tls 인증서를 발급받으려면 도메인을 하나 필요로 하는데, 이 도메인은 일반적으로 `.internal` 같은 걸 사용함. 
근데 `.internal` 도메인에 tls 인증서를 발급받으려면 인증서가 자체적으로 발급되는건 아니고, 수동으로 발급 받아야 하는 번거로움이 존재함. 
이런 번거로움을 해결하기 위해선 `split-horizon dns` 설정을 사용하면 됨. 
`split-horizon dns` 설정은 公開 dns에 내부 ip주소가 기록되지 않도록 하는 설정임. 
이 설정을 사용하면 내부 서비스에 tls 인증서를 발급받을 수 있고, 외부에서 내부 서비스에 접근 못하도록 할 수 있당.
 
### tls 인증서 발급 방법
tls 인증서를 발급받으려면 `acme.sh` 같은 툴을 사용해야 함. 
`acme.sh` 는 Lets Encrypt 나 ZeroSSL 같은 곳에서 tls 인증서를 발급받을 수 있게 도와주는 툴임. 
근데 이 툴을 사용하려면 `split-horizon dns` 설정을 해야함. 
`split-horizon dns` 설정을 하고 나면, 내부 서비스에 tls 인증서가 발급될 거임. 
이렇게 발급받은 인증서는 nginx 나 apache 같은 웹 서버에서 사용할 수 있음. 
그리고 인증서가 만료되기 전에 자동으로 갱신되게 할 수도 있어 편리함.

### 자동갱신
인증서를 자동갱신 하려면, `acme.sh` 에서 제공하는 `--cron` 옵션을 사용하면 됨. 
이 옵션을 사용하면, 인증서가 만료되기 전에 자동으로 갱신하게 됩니다.
 
### 예시
예를 들어, 내부 서비스 중 하나인 grafana 에 tls 인증서를 발급하려면, 
`acme.sh --issue -d grafana.tuxnet.dev --server letsencrypt --standalone` 
라는 명령어를 사용하면 됨. 
이 명령어는 grafana.tuxnet.dev 에 letsencrypt 인증서를 발급받게 함.

###まとめ 
요약하자면, tls 인증서 발급은 어렵지 않지만, `split-horizon dns` 설정이 필요하고, 자동갱신 하는 방법도 알아야 해서 조금 복잡할 수 있음. 
하지만, 오늘 정리한 방법을 따라하면, 쉽게 tls 인증서를 발급받을 수 있을거임. 
그리고, 자동갱신 설정까지 하게 되면, tls 인증서관련 더이상 고민할 일은 없을거임. 
앞으로 tls 인증서 관련된 글이 더 올라오면 좋을텐데, 기대하겠음. 
 tls 인증서는 중요하니까, 알아야 한다 싶은 내용이였음.
