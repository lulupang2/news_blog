## 액티고리!
   카테고리 이론에서 정말 중요한 개념 중 하나인 **액티고리**에 대해 알려줄게. 
   프로그래밍에서 액티고리는 **レン즈**나 **프리즘**などの **옵틱스**에서 핵심적인 역할을 해. 
  那么, 액티고리를 이해하기 위해서는먼저 **모노이달 카테고리**를 알아야 해.

   ### 모노이달 카테고리
   모노이달 카테고리는 **텐서 프로덕트**를 갖춘 카테고리야. 
   이 텐서 프로덕트는 **펑터**로 여길 수 있고, 우리는 이 프로덕트가 **어소시어티브**하고 **유니탈**이라는 가정 아래에서 시작할 거야. 
   어소시어티브란, **어소시에이터**가 존재한다는 뜻이야. 
   또한, **유니터**라는 개념도 필요해. 
   유니터는 **유니터스**로 표시하고, 두 개의 인버티블한 **유니터스**를 가질 수 있어.

   하스켈로 이것을 모델링할 수 있을까? 
   일단, **모노이달 카테고리**를parameter화하는 **ten**이라는 **바이펑터**를 정의해 볼게.
   ```haskell
   class (Bifunctor ten) => MonoidalCategory ten where
     ...
   ```
   그다음, **모노이달 카테고리**의 **유니터**는 **assosiated 타입**으로 정의할 수 있을 거야.
   ```haskell
   type Unit ten :: Type
   ```
   이제, **모노이달 카테고리**의 정의를 완성해보자.
   ```haskell
   class (Bifunctor ten, obj (Unit ten)) => MonoidalCategory (obj :: Type -> Constraint) ten where
     type Unit ten :: Type
     ...
   ```
   ## 액티고리
   **액티고리**는 **모노이달 카테고리**의 **액션**을 지원하는 카테고리야. 
   **액션**은 **펑터**로 여길 수 있고, **모노이달 카테고리**의 **텐서 프로덕트**와 **유니터**를 사용해서 정의할 수 있어.
   ```haskell
   class (MonoidalCategory obj ten, Bifunctor act) => Actegory obj ten act | act -> ten where
     ...
   ```
   ## 예시
   **모노이달 카테고리**와 **액티고리**의 예시를 들어보자.
   ```haskell
   instance MonoidalCategory Hask (,) where
     ...
   instance Actegory Hask (,) (,) where
     ...
   ```
   이것으로今天의 이야기를 마치겠어. 
   혹시라도 **카테고리 이론**에 관심이 있는 사람이라면, **액티고리**를 공부하는 것이 정말 좋은 선택일 거야.
