### SmallVector의 push_back 최적화
SmallVector는 LLVM에서 가장 많이 사용되는 컨테이너입니다. push_back는 가장 핫한 연산이죠. trivially-copyable specialization에서 빠른 경로를 최적화해야 합니다. 
```cpp
#include <llvm/ADT/SmallVector.h>
void f(llvm::SmallVectorImpl<int> &v, int x) { 
    v.push_back(x); 
}
```
clang -S --target=x86_64 -O2 -DNDEBUG로 컴파일해서 최적화된 코드를 확인해 보았습니다. 
최적화 전에 push_back의 코드가 다음과 같았습니다: 
```cpp
push rbp 
push rbx 
mov eax, [rdi + 8] 
cmp eax, [rdi + 12] 
jae .Lgrow
```
이 코드에서는 prologue에서 rbp와 rbx를 push하고, eax에 size를 로드하고 capacity와 비교합니다. 
최적화 후에는 다음과 같이 바뀌었습니다: 
```cpp
mov eax, [rdi + 8] 
cmp eax, [rdi + 12] 
jae growAndPushBack 
mov rcx, [rdi] 
mov [rcx + rax*4], esi 
inc dword ptr [rdi + 8] 
ret 
```
fast path의 instructions이 14에서 7으로 줄었습니다. 
이 최적화는 tail duplication을 사용하여 slow path의 코드를 분리하고, fast path의 코드를 최적화했습니다. 
```cpp
LLVM_ATTRIBUTE_NOINLINE 
void growAndPushBack(ValueParamT Elt) { 
    T Tmp = Elt; 
    this->grow(this->size() + 1); 
    std::memcpy(reinterpret_cast<void *>(this->end()), &Tmp, sizeof(T)); 
    this->set_size(this->size() + 1);
}
void push_back(ValueParamT Elt) { 
    if (LLVM_UNLIKELY(this->size() >= this->capacity())) 
        return growAndPushBack(Elt); 
    std::memcpy(reinterpret_cast<void *>(this->end()), &Elt, sizeof(T)); 
    this->set_size(this->size() + 1);
}
```
이 최적화로 인해 SmallVector의 performance가 개선되었습니다. 
### std::vector의 push_back
std::vector의 push_back도 확인해 보았습니다. 
```cpp
#include <vector>
void pb_int(std::vector<int> &v, int x) { 
    v.push_back(x); 
}
```
std::vector의 push_back는 fast path에서 frame을 생성하고, slow path에서 emplace_back을 호출합니다. 
```cpp
push rbx 
sub rsp, 48 
mov [rsp+12], esi 
mov rax, [rdi+8] 
lea rcx, [rsp+16] 
mov [rsp+24], rcx 
mov [rsp+32], rsi 
mov [rsp+40], rdi 
jae .Lslow
```
fast path에서 frame을 생성하고, slow path에서 emplace_back을 호출하는 코드입니다. 
이 코드는 slow path에서 emplace_back을 호출하고, fast path에서 frame을 생성하는 코드입니다. 
std::vector의 push_back는 SmallVector의 push_back보다 더 많은 instructions을 사용합니다. 
### absl::InlinedVector의 push_back
absl::InlinedVector의 push_back도 확인해 보았습니다. 
```cpp
absl::InlinedVector<int, 4>::push_back
```
absl::InlinedVector의 push_back는 fast path에서 frame을 생성하고, slow path에서 EmplaceBackSlow를 호출합니다. 
```cpp
push rax 
mov [rsp+4], esi 
mov rax, [rdi] 
mov edx, 4 
test al, 1 
je .L2
```
fast path에서 frame을 생성하고, slow path에서 EmplaceBackSlow를 호출하는 코드입니다. 
absl::InlinedVector의 push_back는 SmallVector의 push_back보다 더 많은 instructions을 사용합니다. 
### 결론
SmallVector의 push_back 최적화로 인해 performance가 개선되었습니다. 
std::vector와 absl::InlinedVector의 push_back는 더 많은 instructions을 사용합니다. 
SmallVector의 push_back 최적화는 performance 개선에 기여합니다.
