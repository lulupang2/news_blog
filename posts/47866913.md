### AI 코딩 도구들이 왜这么 많이 코드를 고쳐 븅신같이 만들까
   개빡치는 점은 AI 코딩 도구들이 코드를 고칠 때 자꾸 불필요한 코드까지 몽땅 고치고 말이야. 그냥 버그 하나 고치는데 함수 전체를 갈아엎잖아 ㅋㅋㅋㅋ
   이유를 찾아보기로했음. 
   ### 오버-에디팅問題
   첫째, **오버-에디팅**은 모델이 코드를 수정할 때 꼭 필요한 수정 외에도 불必要한 수정을 같이 하는 거임. 예를 들어, 버그 하나 고칠 때 함수 전체를 고치거나 변수 이름을 바꾸거나 하는 등의 행동임. 
   ### 오버-에디팅 문제의 심각성
   코드 리뷰는 이미 병목 현상인데, 모델이 코드를 너무 많이 고치면 코드 리뷰하기 개힘들어짐. 코드 리뷰어는 변경된 코드를 이해해야 하는데, 모델이 코드를 너무 많이 고치면 코드를 이해하기 어려워지고 코드 품질이 나빠질 가능성도 높아짐. 
   ### 오버-에디팅 측정 방법
   오버-에디팅을 측정하기 위해선 **토큰 레벨 레벤슈타인 거리**와 **인지된 복잡성**을 사용할 수 있음. 
   **토큰 레벨 레벤슈타인 거리**는 코드를 토큰으로 나누고, 원본 코드와 수정된 코드의 토큰 사이의 거리를 측정하는 거임. 
   **인지된 복잡성**은 코드를 이해하기 어려운 정도를 측정하는 것으로, 예를 들어 함수의 중첩 정도나 논리 연산자의 사용 여부 등을 측정함.
   ### 모델의 오버-에디팅 성향
   실험 결과, 최신 모델들 모두 오버-에디팅 성향을 보였음. 예를 들어, GPT-5.4 모델은 오버-에디팅 성향이 가장 높았음. 
   ### 프롬프팅의 효과
   모델에 explicit prompts를 주면 오버-에디팅 성향이 немного 줄어들음. 
   ### 이유모델 vs 비이유모델
   이유모델은 비이유모델에 비해 오버-에디팅 성향이 높았음. 
   결론은 AI 코딩 도구들이 아직 코드를 너무 많이 고치고 있다는 점임 ㅋㅋ
