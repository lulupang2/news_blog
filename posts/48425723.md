# zeroserve: 설정 없이도 쉽게 웹서버를 만들 수 있는 eBPF 스크립트 가능한 웹서버
zeroserve는 tarball로 된 웹사이트를 쉽게 설정 없이도 웹서버로 만들어 주는 거임. HTTP/2와 TLS 1.3을 지원해서安全하고, hot reload도 가능하고, 작은 응용프로그램의 메모리 점유율도 낮아. 

# eBPF 스크립팅
zeroserve에서 가장 재밌는 점은 eBPF 스크립팅이야. eBPF 프로그램을 넣으면 요청과 응답을 처리할 수 있고, 인증이나 레이트 리미팅도 가능해. 

## 요청과 응답 처리
요청과 응답을 처리하는 것은 매우 쉽고, 요청의 경로, 헤더, 본문을 쉽게 처리할 수 있어. 

## 인증과 레이트 리미팅
인증과 레이트 리미팅도 쉽게 처리할 수 있고, AWS SigV4도 지원해서 AWS 서비스와 쉽게 연동할 수 있어. 

# 성능비교
zeroserve는 nginx나 Caddy보다 빠르다는 점도 장점이야. 작은 파일과 큰 파일 모두에서 성능이 뛰어나서, 실제로 쓰임새가 많을 것 같아. 

## 작은 파일
작은 파일의 경우, zeroserve는 nginx보다 약 17% 빠르다는 결과가 나왔어. 

## 큰 파일
큰 파일의 경우도 zeroserve는 약 780MB/s의 속도로 nginx와 비슷한 성능을 보여줬어. 

# eBPF vs Lua
zeroserve의 eBPF 스크립팅은 LuaJIT보다도 빠르다는 점이 장점이야. 

## 헤더 추가
헤더 추가의 경우, zeroserve의 eBPF 스크립팅은 LuaJIT보다 약 50% 빠른 성능을 보여줬어. 

## JSON 응답
JSON 응답의 경우도 zeroserve의 eBPF 스크립팅은 LuaJIT보다 약 15% 빠른 성능을 보여줬어.
