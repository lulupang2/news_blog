### 리디스 8.8 나왔 븅신들아 ㅋㅋㅋㅋ
리디스 8.8 오픈소스가 나왔고 새로운 기능들과 성능 개선이 포함되어 있어. 
고lights에는 새로운 배열 자료구조, 윈도우 카운터 레이트 리미터, 스트림 메시지 nack, 해시 필드[subkey] 알림, 
-json 숫자 배열 저장 방식 제어, 하나의 시간 σει열 쿼리에서 복수의 집계함수 사용, 
-sorted set 유니온과 인터섹션 연산에 새로운 COUNT 집계함수가 포함되어 있음.

### 성능 개선 요약
리디스 8.8에는 다음과 같은 성능 개선들이 포함되어 있음:
- 문자열: MGET (파이프라이닝, I/O 스레드 사용)에서 최대 68%의 성능 개선
- 문자열: MGET (파이프라이닝, 싱글 스레드)에서 최대 50%의 성능 개선
- 해시: HGETALL에서 최대 25%의 성능 개선 (1천 이상 필드)
- 스트림: XREADGROUP에서 최대 83%의 성능 개선 (COUNT 100)
- 정렬 집합: ZADD, ZINCRBY, ZRANGEBYSCORE에서 최대 74%의 성능 개선
- 비트맵: 비트맵 연산에서 최대 28%의 성능 개선 (x86)
- 하이퍼로그로그: PFCOUNT에서 최대 18%의 성능 개선 (x86)
- 여러 가지: SCAN, HSCAN, SSCAN, ZSCAN에서 최대 40%의 성능 개선
- 영속성과 복제 (풀 싱크로나이즈)가 최대 60% 빠р지게 개선됨.

### 새로운 기능 요약
리디스 8.8에는 다음과 같은 새로운 기능들이 포함되어 있음:
- 배열: 새로운 전용 자료구조로 인덱스 주소 지정 가능한 문자열 값의 집합.
- 윈도우 카운터 레이트 리미터: 레이트 제한을 위한 새로운 메커니즘으로서 기존의 서버측 루아 스크립트와 클라이언트 로직이 필요없어짐.
- 스트림 메시지 nack: 消費者가 대기 중인 메시지를 명시적으로释放하여 다른 消費者가 사용할 수 있게 하는 기능.
- 해시 필드 알림: 해시 필드에 대한 알림을 받을 수 있는 기능.
- json 숫자 배열 저장 방식 제어: json 숫자 배열을 저장하는 방식을 사용자가 제어할 수 있는 기능.
- 하나의 시간 세리아 쿼리에서 복수의 집계함수 사용: 하나의 쿼리에서 여러 개의 집계 함수를 사용할 수 있는 기능.

### 배열 자료구조
 배열은 인덱스 주소 지정 가능한 문자열 값의 집합으로, 각 요소는 숫자 인덱스에 저장되고 매우 빠르게 접근할 수 있음. 
배열은 동적으로 크기가 조정되고, 요소를 삽입하거나 삭제할 수 있으며,compute-aware한 컨테이너로 새로운 사용 사례와 기존 사용 사례의 유연성과 효율성을 제공함.

배열은 다음과 같은 특성을 가짐:
- 동적 크기 조정: 배열의 크기는 동적으로 조정되고, 요소를 삽입하거나 삭제할 수 있음.
-稀疏성 지원: 배열은稀疏한 자료구조를 지원하므로, 요소를 삽입하거나 삭제할 때에도 메모리 사용량이 효율적으로 관리됨.
- 컴퓨팅 지원: 배열은 컴퓨팅 지원을 제공하므로, 서버 측에서 배열의 요소를 계산하고 집계할 수 있음.

배열은 다음과 같은 사용 사례에 적합함:
- 매우 빠른 인덱스 접근이 필요한 경우
- 슬라이딩 윈도우를 사용해야 하는 경우
- 서버 측 계산과 집계가 필요한 경우
- 요소 검색이 필요한 경우

배열은 다음과 같은 사용 사례에 부적합함:
- 푸시/팝 연산이 필요한 경우 (리스트 사용)
- 필드 이름 기반 접근이 필요한 경우 (해시 사용)
