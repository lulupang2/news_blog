### 원격 서버 리모트 데스크톱 만들기, SSH 그래픽 셸 등장 ㄷㄷ
원격 서버도 로컬 머신처럼 그래픽 셸로 사용할 수 있는 기술이 등장했음. 기존의 터미널 기반 애플리케이션과는 다르게 브라우저 기반 그래픽 셸을 제공함. 이 셸은 홈 스크린에 다양한 앱들을 제공하고, 각 앱은 작은 HTTP 서버로 동작하며 웹 사용자 인터페이스를 제공함. 

이 그래픽 앱들은 터미널 기반 앱들의 대안으로 작용할 수 있음. 이 앱들은 로컬에서 실행되거나 SSH를 통해 접근할 수 있고, 일반적으로 개인용으로 사용되며 다른 기기에서 접근할 수 없음. 대신에 유닉스 도메인 소켓 파일을 사용하며, 이는 포트와 유사하지만 파일 시스템에 Explicit한 사용자 권한을 갖도록 설계됨.

Outer Loop이라는 SSH 브라우저에서 이 기술을 적용해 보았고, Outer Shell이라는 오픈소스 프로젝트를 출시했음. 아래는 데모 비디오링크임. 
[Outer Shell 데모](링크)

Outer Shell과 그 작동 방식을 소개함. 그리고 이 기술의 가능성을 보여줌. 업데이트로 유튜브 미러링도 제공함. 

이 기술에 대한 문서도 제공하며, 브라우저 작동 방식, Outer Shell API, 네이티브 앱 등의 정보를 포함하고 있음. 
- 브라우저 작동 방식: https://outerloop.sh/
- Outer Shell API: https://outershell.org/
- 네이티브 앱 작동 방식: https://outerframe.org/

개인적인 생각으로 이 기술은 왜 이제야 등장했을까 생각해 봄. 브라우저에서 유닉스 소켓에 연결하는 기능은 매우 니치한 기능으로 여겨졌지만, SSH와 함께 사용하면 새로운 가능성이 생김. 기존의 리눅스 서버를 로컬 그래픽 운영체제로 취급하지 말고, 원격 기기에 대한 그래픽 셸로 사용할 수 있으면 더 좋을 것 같음. 
그리고 이제는 AI가 코드를 작성하는 데 도움을 줄 수 있기 때문에, 각 애플리케이션은 타겟 플랫폼에 맞는 코드베이스를 가질 수 있게 되었음. HTML과 자바스크립트를 사용하는 웹 애플리케이션은 읽기나 간단한 작업에 사용되고, 네이티브 플랫폼을 위한 애플리케이션은 실제 작업을 처리하는 데 사용될 수 있게 되었음.
