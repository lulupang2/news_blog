# 러스트 언어의 프로시저 매크로 만들기
 러스트 언어에서 매크로는 사실 함수랑 거의 같다고 보면 됨. 함수는 입력 값을 넣었을 때 출력 값을 줘. 매크로도 거의 똑같음. แต 차이점이 하나 있음. 매크로는 입력 값을 넣었을 때 출력 값을 주는 건 같지만, 매크로가 작동하는 곳이 달라. 함수는 프로그램이 실행될 때 작동하지만, 매크로는 프로그램이 컴파일되는 단계에서 작동해.
 
 ## 매크로의 종류
 매크로는 크게 두 가지로 나눌 수 있음. 선언적 매크로(Declarative Macro)랑, 프로시저 매크로(Procedural Macro)임. 선언적 매크로는 사실 매크로를 사용해 함수를 확장하는 법임. 예를 들면, `unwrap_or_break`이라는 매크로를 사용해서 `Option` 타입의 값을 감싸는 함수를 쉽게 만들 수 있음.
 
 ## 프로시저 매크로
 프로시저 매크로는 함수랑 거의 같다고 보면 됨. 하지만, 매크로가 작동하는 단계가 달라. 프로시저 매크로는 컴파일 단계에서 작동하고, 함수는 프로그램이 실행되는 단계에서 작동해. 프로시저 매크로는 토큰 스트림을 입력 값으로 받고, 토큰 스트림을 출력 값으로 줘. 토큰 스트림은 사실 코드의 일부분을 나타내는 것임. 예를 들면, `let x = 5;`라는 코드는 `let`, `x`, `=`, `5`, `;`라는 토큰으로 구성돼.
 
 # 프로시저 매크로 사용하기
 프로시저 매크로를 사용하려면, `proc-macro` 크레이트를 추가해야 해. 그 다음, `#[proc_macro]` 어트리뷰트를 사용해서 프로시저 매크로 함수를 정의할 수 있음. 예를 들면, 다음과 같은 코드를 사용할 수 있음:
 
 ```rust
 #[proc_macro]
 pub fn custom_proc_macro(input: TokenStream) -> TokenStream {
     eprintln!("{:?}", input);
     input
 }
 ```
 
 이 함수는 입력 값을 받아서, 입력 값을 출력하는 함수임. 이 함수는 컴파일 단계에서 작동하고, 토큰 스트림을 입력 값으로 받고, 토큰 스트림을 출력 값으로 줘.
 
 # 결론
 러스트 언어의 프로시저 매크로는 함수랑 거의 같다고 보면 됨. 하지만, 매크로가 작동하는 단계가 달라. 프로시저 매크로는 컴파일 단계에서 작동하고, 함수는 프로그램이 실행되는 단계에서 작동해. 프로시저 매크로는 토큰 스트림을 입력 값으로 받고, 토큰 스트림을 출력 값으로 줘. 프로시저 매크로를 사용하려면, `proc-macro` 크레이트를 추가해야 해. 그 다음, `#[proc_macro]` 어트리뷰트를 사용해서 프로시저 매크로 함수를 정의할 수 있음. 프로시저 매크로는 코드를 확장하는 데에 사용될 수 있음. 따라서, 프로시저 매크로를 사용해서 코드를 더 간결하게 만들 수 있고, 코드의 가독성을提高할 수 있음.
