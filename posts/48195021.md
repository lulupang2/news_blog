## 수퍼로그 등장 ㅋㅋㅋ
Hey 펨코 안녕, 우리는 니코랑 아르세니이. 수퍼로그라는 자기 설치도 하고 자기 수리도 하고 그런 관측 툴을 만들고 있음. 
우리가 만들 툴은 관측 툴 중에 가장 귀찮은 설정도 자동으로 해주고 에러 고치기도 자동으로 해주는 툴임. 
아래 유튜브 링크로 가면 간단한 데모를 볼 수 있음. [https://www.youtube.com/watch?v=xFhU9Mk247M](https://www.youtube.com/watch?v=xFhU9Mk247M)ㅋㅋㅋ

우리가 전회사에서 썼던 관측 툴들인데, Sentry, Datadog, Grafana, Dash0 이런 건 너무 설정하기 귀찮았음. 
우리가 쓰던 것도 제대로 동작 안했고, 자꾸 비용만 불어나는 거 봐서 그냥 만들기로 했음. 
특히 슬랙 채널에 계속 팅기는 알림 때문에 밤잠도 설쳐가며 일할 때가 많았음. 

우리가 해결하려는 문제는 서버 메모리 다 차서 죽는 거, 디스크 공간 다 차서 죽는 거 이런거임. 
그리고 AWS 메트릭스를 보면 같은 숫자인데三个 다른 값을 보여주는 것도 이해가 안감. 
대시보드 그래프인데 대부분이 비어 있거나 古い 자료만 있어서 수시로 클릭해서 보는 건 시간 낭비一样 cảm覺함.

## 수퍼로그는 어떻게 작동할까
우리의 목표는 그냥 설치하면 알아서 돌아가도록 만들기임. 
우리에 이런 문제를 해결하기위한 지식이 있어서 만들 수 있었음. 
우리는 리포를 스캔해서 자동으로 잘 구조화된 로그, 트레이스, 메트릭을 OpenTelemetry로 붙여줌. 
에러가 발생하면 자동으로 조사해서 PR을 만들어줌. 

수퍼로그는 다른 관측 툴과 달리 설정하기가 너무 쉬움. 
自動으로 설정하고, 자동으로 수리하는 툴임. 
에러가 발생하면 자동으로 조사해서 알림을 보내줌. 
그리고 자동으로 PR을 만들어줌. 

우리가 만들 툴은 이런 식으로 작동함. 
1. **설정 안해도됨**: 마법사가 자동으로 설정해줌. 
2. **자동으로 고침**: 에러 발생하면 자동으로 조사해서 해결해줌. 
3. **알림도 자동**: 에러 발생하면 자동으로 알림을 보내줌. 

수퍼로그는 이런 식으로 작동하고, 우리가 만든 툴을 사용하면 더 편해질 거임. 
다만, 아직早期라서 버그가 있을 수也是 있겠지만, 너네의 피드백이 필요함. 
수퍼로그를 사용해보고, 어떤 문제가 발생하면 알려줘. 
그리고, 너네가 지금 쓰고 있는 관측 툴은 어떤 거고, 어떤 문제가 있으면 알려줘. 
그리고, 수퍼로그의 '한 번의 머지 가능한 PR' 모델이 آیا 쓸만한지 알려줘. 
특히, 통합된 제품을 운영하는 사람이나, 자체 관측 툴을 만들었지만 실패한 사람, Sentry나 Datadog MCP를 사용해 본 사람의 의견도 궁금함. 
의견이나 피드백은 언제든지 환영임. 
수퍼로그를 사용해보고, 너네의 생각을 알려줘. [https://superlog.sh](https://superlog.sh)
