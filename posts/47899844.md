> **🔗 원문 보러가기:** [Show HN: A Karpathy-style LLM wiki your agents maintain (Markdown and Git)](https://github.com/nex-crm/wuphf)

### 개빡치는 점
해커스 뉴스에 떴다!! 카르파시 스타일 LLM 와이키에 에이전트가 알아서 관리해 준다는 뜻임 ㅋㅋ
### 소개
WUPHF 라는 이름으로slack for AI employees with a shared brain 라고 설명함 ㄷㄷ 
CEO, PM, 엔지니어, 디자이너, CMO, CRO 등 다 구분 안 하고 다 같이 보이는데 할 일 주고 받으며 일이 돌아감 ㅋ 
무슨 말인지 모르겠으면 [30초 티져 виде오](링크) 보셈 
### 시작하는 법
1. **Prerequisites**: 에이전트 CLI 설치 (기본값: 클라우드 코드)하고 tmux 가 있어야 함 
2. 터미널에서 `npx wuphf` 입력 
   - 자동으로 브라우저가 열리고 localhost:7891에 접근해서 office 에 접속할 수 있음 
   - 그 후 [README.md](링크)랑 [FORKING.md](링크) 잘 읽어보셈 
3. 혹시 깃헙에 로그인이 된 상태고 WUPHF 저장소를 별로 안 눌렀으면 저장소에 별 누를 때 물어보고 tự động으로 클릭해 줄 수도 있음 
4. 혹시 CEO에서 에이전트로 바꾸고 싶으면 `/focus` 명령어를 친다음에 할 일 배분 받을 수 있음 
### 메모리 
- 에이전트가 작업하면서 노트북에 기록한 내용을 작업이 완료되면Wiki에 자동으로 기록되지 않고 에이전트가 판단해서 Wiki로 승급시킴 
- Wiki 는 마크다운 형식이고 Git 으로 관리되는데 기본값임 (기존 사용자들은 Nex 나 GBraiin 백엔드를 그대로 쓸 수 있음) 
- Wiki 는 로컬 저장소에 저장되어 있다고해서 Api 키 없이 Git 명령어로直接 읽어올 수 있음 
   - `cat`, `grep`, `git log`, `git clone` 등 지원 
### 기타 커맨드 
- `wuphf init` : 초기 설정 
- `wuphf shred` : 세션 종료 
- `wuphf --1o1` : CEO 와 1:1 대화 
- `wuphf --1o1 cro` : 특정 에이전트와 1:1 대화 
### 텔레그램桥 
- `/connect` 명령어로 텔레그램 연동이 가능하고 메세지 서로 주고받을 수 있음 
### 오픈클로우橋 
- `/connect openclaw` 명령어로 오픈클로우 에이전트를 브릿지할 수 있음 
  - 에이전트랑 커뮤니케이션이 가능해지고 에이전트가 작업하고 있는 내용을 볼 수 있음 
### 외부 동작 
- WUPHF ships with two action providers. 
   - local-first 
   - Composio 
- `/config set action_provider one` 커맨드로 local-first 로 바꿀 수 있음
