### 모토롤라 MR2600 라우터 원격 코드 실행 취약점 발견 ㅋㅋㅋㅋ

요즘은 라우터 하나하나마다 코드 실행 취약점이 있는 것 같음 ㅋㅋㅋㅋ 내가 최근에 발견한 모토롤라 MR2600 라우터도 예외는 아니었음. 
이 라우터는 Wi-Fi 5 라우터이고 마지막 펌웨어 업데이트는 2024년에 했음. 

첫번째로 해결해야 했던 문제는 펌웨어를 얻는 것이었음. 모토롤라는 대부분의 라우터에 대한 펌웨어를 공개적으로 배포하지 않음. 대신에 OTA(에어) 업데이트 통해만 배포함. 
예외는 모토롤라 MR2600이었음. fwupload 엔드포인트라는 개념을 찾아야 했고, 펌웨어를 업로드하고 인증过程 없이 실행할 수 있는 것을 발견했음. 

업로드된 펌웨어는 /tmp/firmware.img 에 저장되는데, 이 파일은 인증 과정 없이 바로 실행할 수 있었음. 
내가 발견한 취약점은 fwupload.cgi 와 LoadFirmwareValidation API에서 발생함. 
fwupload.cgi는 업로드한 펌웨어를 검증하는 과정 없이 바로 /tmp 에 저장하고, LoadFirmwareValidation API는 업로드한 펌웨어를 검증하고, 인증 과정 없이 바로 실행하는 것을 가능하게 해줌.

이제 내가 발견한 취약점을 어떻게 이용할 수 있는지 알려줄게. 
1. fwupload.cgi 호출해서 악의적인 펌웨어 이미지 업로드
2. LoadFirmwareValidation API 호출해서 인증 과정 건너뛰기
3. 펌웨어 이미지 검증과정과 CRC32 체크 통과
4. 펌웨어 플래싱하고 라우터 리부팅

이 취약점은 최소한으로 LAN 에서 발생할 수는 있지만, 원격 관리가 활성화된 경우에는 원격에서도 발생할 수 있음. 
Shodan 에서 확인한 결과, 원격 관리가 활성화된 MR2600 라우터가 41대 이상 존재함. 

이 취약점은 모토롤라에 보고했지만, 모토롤라는 이 라우터가 이미 종료되었다고 하면서 취약점을 공개하기로 결정했음. 
나도 이번 기회에 모토롤라 MR2600 라우터에서 발견된 취약점을 공개하기로決定함.
