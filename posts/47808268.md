## 1. 개요

허허, 요새 젊은 개발자 친구들이 컨테이너가 최고인 줄 알고 살길래, 내가 진짜 '근본' 넘치는 물건 하나 들고 왔다. 이름은 **Smol machines (smolvm)**. 이름 그대로 아주 작고(smol) 빠른 가상머신인데, 이게 아주 요물이야. 0.2초 만에 켜지는 VM 본 적 있어? 못 봤으면 일단 들어와 봐.

## 2. 주요 특징 (이게 왜 지리는가)

- **미친 부팅 속도:** 콜드 스타트가 1초도 안 걸려. 보통 200ms 미만으로 컷한다. 눈 깜빡하면 이미 켜져 있는 수준이지.
- **하드웨어 수준의 격리:** 도커처럼 커널 공유하는 찝찝한 방식이 아니다. 하이퍼바이저 수준에서 하드웨어로 딱 격리해버려. 보안 하나는 확실하겠지?
- **메모리 다이어트의 달인:** virtio balloon이라는 걸 써서 호스트 메모리를 필요한 만큼만 야금야금 가져다 쓰고, 안 쓰면 바로 돌려줘. 8GB 할당해놔도 실제 쓰는 만큼만 먹는다는 소리야. 아주 기특하지?
- **휴대성 씹상타치:** VM 상태 그대로 `.smolmachine` 파일 하나로 딱 구워버릴 수 있어. 이거 들고 다른 컴퓨터 가서 실행하면 환경 설정이고 뭐고 그대로 돌아간다.

## 3. 이걸 어디다 써먹냐?

- **수상한 코드 테스트:** 인터넷 안 되는 샌드박스 딱 만들어서 굴려버리면, 아무리 악성코드라도 내 컴 본체는 못 건드려. 
- **환경 설정 지옥 탈출:** Python 버전 맞추느라 venv, conda 깔고 삽질할 필요 없어. 그냥 VM 하나로 구워서 줘버리면 끝이야.
- **SSH 키 보안:** 호스트의 SSH 에이전트를 VM 안으로 포워딩해주는데, VM 안에서는 실제 키를 절대 못 빼가게 하이퍼바이저가 감시해. 깃허브 클론 딸 때 보안 걱정 없겠지?

## 4. 기술 스택 (틀딱들이 좋아할 만한 근본)

- **언어:** Rust (역시 요즘 대세는 러스트지, 암 그렇고말고)
- **엔진:** libkrun 기반의 VMM을 사용함
- **지원 OS:** macOS (Apple Silicon 완벽 지원), Linux (KVM)

## 5. 기존 도커/VM이랑 비교해줌

| 특징 | **smolvm** | 컨테이너 (Docker) | QEMU/Colima |
| :--- | :--- | :--- | :--- |
| **격리 방식** | VM별 하드웨어 격리 | 네임스페이스 (커널 공유) | 별도 VM |
| **부팅 시간** | **<200ms (개빠름)** | ~100ms | 수 초 이상 |
| **휴대성** | .smolmachine 단일 파일 | 이미지 (데몬 필요) | 무겁고 복잡함 |
| **임베디드 SDK** | 지원함 | 지원 안 함 | 지원 안 함 |
