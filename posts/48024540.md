### 게마4속도업그레이드

지난주 게마4 공개하고 벌써 600만개가 넘게 다운로드 됐네 ㅋㅋㅋㅋ 
요즘 개발자들이 게마4로 뭘 만들고 있는지는 몰라도, 우리는 게마4의 효율성은 더 높여야겠다고 결심함. aujourd'hui, 게마4 모델 패밀리 用 멀티토큰예측 드래프터를 공개함 ㅋ 

이 드래프터들은专用한 스펙큐러티브 디코딩 아키텍처를 사용해서 输出품질,추론로직에는 아무런 영항도 없이 인퍼런스 속도를 최대 3배까지 올려줌 ㅋㅋㅋㅋ. 
게마4 26B 모델을 엔비디아 RTX PRO 6000 에서 테스트했는데, 스탠다드 인퍼런스보다 MTP 드래프터가 tokens-per-second 속도가 훨씬 빠르다고 함. 

 ### 왜 스펙큐러티브 디코딩일까?

현실적으로, 일반적인 LLM 인퍼런스는 메모리 대역폭에 제한되어서, latency bottleneck 가 생김. 
프로세서는 매번 토큰 하나를 생성하기 위해, VRAM에 있는 억억개의 파라미터들을 모두 compute 유닛으로 옮겨야 하는데, 이 과정에서 대부분의 시간이 걸리게 되고, 컴퓨팅 파워도 못 써서, latency도 높고 ㅋㅋㅋㅋ. 
게다가 소비자급 하드웨어에서는 이 문제가 더 심해짐. 

스펙큐러티브 디코딩은 토큰 생성과 토큰 검증을 분리함으로써,闲置중인 컴퓨팅 파워를 tận dụng해서 idle time을 줄일 수 있음. 
예를 들어, 31B 모델과 가벼운 드래프터를 페어링해서, target 모델 하나당 여러 개의 토큰을 예측할 수 있음. 
그리고 target 모델에서 검증할 때, 동시에 여러 토큰을 검증할 수 있어서, 속도가 훨씬 올라감. 

예를 들어 보자. 
'어셈블리 언어'라는 문장 뒤에, 다음 토큰은 '프로그래밍'일 확률이 거의 100%라면, 그 뒤에 이어질 토큰들을 미리 예측하고, 그 뒤에 target 모델에서 검증하는 방식임. 
이 과정에서, 실제로 드래프터에서 예측한 토큰들이 target 모델에서도 검증이 된다면, 인퍼런스 속도가 더 올라감. 

### 에지 디바이스에서부터 워크스테이션까지 더 빠른 AI

개발자들이 게마4 모델에 멀티토큰예측 드래프터를 사용하면, 다음과 같은 장점이 있음: 

*   반응속도 개선: 레이턴시가 크게 감소해서, 채팅, 음성앱, 에이전트 워크플로우 등에서 반응속도가 개선됨. 
*   로컬 개발 속도 개선: 개인 컴퓨터와 소비자급 GPU에서도, 26B MoE와 31B Dense 모델을 이전과 같은 정확도로 더 빠르게 사용할 수 있음. 
*   에지 디바이스 성능 개선: 에지 디바이스에서 E2B와 E4B 모델도 더 빠른 인퍼런스를 통해 배터리 효율을 높일 수 있음. 
*   품질 저하 없음: Primary Gemma 4 모델에서 최종 검증을 하므로, 결과물의 품질은 이전과 같음. 

요즘처럼 개발 속도가 중요해지는 세상에서, 이런 속도 개선은 정말 큰 장점이라고 볼 수 있음. 
게마4에 멀티토큰예캐시트 드래프터를 사용해서, 개발자들은 이전과 같은 정확도로 더 빠른 인퍼런스를 사용할 수 있게 된거임. 
개인 컴퓨터, 에지 디바이스 등에서 게마4 모델을 사용할 때, 이제 이전보다 더 빠른 속도로 개발할 수 있게 된 것이네.
