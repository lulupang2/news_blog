## 개요
   이 포สต는 하스켈(Koan)이다. 배경과 동기를 다루겠지만, 여기서 목표는 소규모이자 드문 기술을 공유하는 것이며, 완벽한 블로그 글이다. 간단히 말하자면, 타입 체크된 비어있지 않은 문자열 생성자를 구현했고, 이를 통해 TemplateHaskell의 호출을 대체하여, 대량의 데이터가 많은 패키지의 빌드 시간을 약 10% 개선했다.
   ```haskell
before = $$(NonEmptyText.make "hello")
after = NonEmptyText.make "hello"
invalidBefore = $$(NonEmptyText.make "")  // 컴파일 에러
invalidAfter = NonEmptyText.make ""  // 타입 에러: 비어있지 않은 문자열이 필요합니다
   ```
   ### 비어있지 않은 문자열
   비어있지 않은 문자열은 Bellroy의 소프트웨어 설계에서 핵심적인 목표다. 따라서, 많이 사용되는 `NonEmptyText` 타입을 정의했고, 이 타입은 문자열이至少 한 문자 이상으로 구성된다는 것을 의미한다. 
   ### 타입 체크
   이 기술은 지난 15년 동안 GHC 기능이 수렴하는 결과물이다. 특히, 9.10 버전에서 도입된 RequiredTypeArguments를 사용하면, 타입 수준에서 문자열 리터럴을 함수에 전달할 수 있다. 
   ```haskell
type family IsNonEmptySymbol symbol :: Constraint where
   IsNonEmptySymbol "" = Unsatisfiable (Text "비어있지 않은 문자열이 필요합니다")
   IsNonEmptySymbol _ = (() :: Constraint)
   ```
   위의 `IsNonEmptySymbol`은 타입 수준에서 비어있지 않은 문자열을 확인하는amily이다. 
   ### 타입 체크된 생성자
   이 타입 체커를 사용하여, 비어있는 문자열을 생성하는 것을 방지할 수 있다. 
   ```haskell
make :: forall symbol -> (IsNonEmptySymbol symbol) => NonEmptyText
make symbol = NonEmptyText (fromString (symbolVal (Proxy :: Proxy symbol)))
   ```
   위의 생성자는 비어있지 않은 문자열을 만들기 위해 `IsNonEmptySymbol` 타입을 사용한다.
   ## 타입 수준 파싱
   이 기술을 사용하여, DynamoDB 테이블 이름을 타입 수준에서 파싱할 수 있다. 
   ```haskell
newtype TableName = TableName Text deriving (Show)
   ```
   여기서 `TableName`은 DynamoDB 테이블 이름을 위한 새로운 타입이다.
   ## 유효성 검사
   `IsValidTableName`은 DynamoDB 테이블 이름의 유효성을 확인하는 타입 클래스이다.
   ```haskell
class (KnownSymbol a) => IsValidTableName a
   ```
   이 타입 클래스는 `KnownSymbol` 클래스를 사용하여, 테이블 이름의 유효성을 확인한다.
   ### 유효성 검사 구현
   위의 `IsValidTableName__` 함수는 실제 유효성 검사를 수행하는 함수이다. 
   ```haskell
type family IsValidTableName__ text :: Bool
   ```
   위의 함수는 `text`가 유효한 테이블 이름인지 확인한다.
   ```haskell
valid, invalid, invalid2 :: TableName
valid = make "hello-bellroy123"
invalid = make "no"  // 컴파일 에러
invalid2 = make "tablename!!"  // 컴파일 에러
   ```
   위의 예제는 유효한 테이블 이름과 유효하지 않은 테이블 이름을 생성한다.
