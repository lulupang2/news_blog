# 레드 프로그래밍 언어, 정적 연결 지원 업데이트 ㅋㅋㅋㅋ

      

     레드는 요즘 코딩 에이전트의 혁명에 빠져있고, 우리는 그 중심에 있음 ㄹㅇ

      레드란 무엇일까? 그냥 코딩 언어일 뿐이긴 한데, 에이전트랑 잘配合되도록 업데이트중임

      

      이번 업데이트는 C로 작성된 라이브러리의 정적 연결을 지원하게끔 바꿨당

      이게 뭐가 좋냐하면, 이제 프로그래머들이 단 하나의 실행 파일만 만들면 됨 ㅋㅋ

      모든 의존성이 다 포함되어 있기 때문에, 따로 설치할게 없어서 매우 편함

      

      ### 어떻게 작동하냐하면

      레드 시스템에서 `miniz` 라이브러리를 컴파일하고 정적 연결하는 방법을 보럼

      

      ```red/system

      #import ["miniz" cdecl [

        compress: "mz_compress" [

          dst [byte-ptr!]

          dst-len [int-ptr!]

          src [byte-ptr!]

          src-len [integer!]

          return: [integer!]

        ]

        uncompress: "mz_uncompress" [

          dst [byte-ptr!]

          dst-len [int-ptr!]

          src [byte-ptr!]

          src-len [integer!]

          return: [integer!]

        ]

      ] ]

      ```

      

      그런 다음, `-s` 플래그를 추가해서 정적 연결을 사용하면됨

      `red -r -s demo.reds`를 실행하면 `miniz` 라이브러리가 실행 파일에 포함됨

      

      이 업데이트는 모든 레드 시스템의 대상에서 작동함

      윈도우, 리눅스, macOS 등 모두 지원하고, 크로스 컴파일도 가능

      

      그래서 레드 언어가 실제로 어떻게 사용되는지 보여주는 예제도 준비했음

      체리 트래커라는 사운드 트래커 스타일의 모드 플레이어를 만들었당

      이 프로젝트는 실제로 레드 언어와 에이전트의 khả력을 테스트하고 보여주기 위해 만들어짐

      체리 트래커는 레드/레드 시스템으로 작성되었고, 드로우 기반의 UI를 가지고 있음

      

      레드 언어와 에이전트의 능력을 결합해서 정말 멋진 걸 만들어 낼 수 있을 거라 생각함

     체리 트래커를试해 보고, 레드 언어의 능력을 직접 경험해 보기 바랍니다 ㅋㅋ
