## Vite+ 베타公開
VoidZero는 클라우드플레어에 합류했다는 소식과 함께 Vite+ 베타를 공개했다. Vite+는 웹 프로젝트에 필요한 런타임, 패키지 관리자, 프론트엔드 툴들을 하나로 통합한 툴체인이라고 할 수 있따. 이제 새로운 프로젝트를 시작할 때 `vp create`를 사용하거나 기존 프로젝트에선 `vp migrate`를 사용해서 쉽게採用할 수 있다.

### 왜 Vite+일까?
Vite+는 Vite가 보여준 빠르고 편한 개발 환경을 전체 개발 워크플로우에 적용한もの다. 이제 개발자들은 프로젝트마다 다른 툴과 명령어를 기억할 필요 없이 일관된 워크플로우를 사용할 수 있다.

#### 기능들
- `vp dev`: 핫모듈 리플레이스먼트와 함께 개발 서버를 시작한다.
- `vp check`: 코드를 포맷팅하고 린트를 돌린다.
- `vp test`: 유닛 테스트를 실행한다.
- `vp build`: 프로덕션 환경에 맞게 빌드를 한다.
- `vp pack`: 라이브러리를 번들링한다.
- `vp run`: 任意의 npm 스크립트나任务를 실행한다.

이 툴체인은 팀과 코드베이스가 커질수록 더 큰 혜택을 제공한다.
- 툴 버전을 일관性 있게 관리할 수 있다.
- 설정을 쉽게 공유할 수 있다.
- 새로운 기여자가 설정하기가 더 쉽다.
- CI에서 개발자들이 로컬에서 사용하는 명령어를 사용할 수 있다.

### Vite+는 Vite 생태계를 대체하지 않는다
Vite 플러그인도 여전히 사용할 수 있고, 프로젝트마다 원하는 패키지 관리자를 사용할 수 있다. Vite+는 이러한 툴들을 하나로統一하는 통합 계층을 제공한다.

## 베타公開 이후의 변경점들
Vite+ 베타公開 이후 많은 변경점들이 있었다.
- 캐싱을 개선했다.
- 마이그레이션을 개선했다.
- 기업용 기능을 추가했다.
- 크로스 플랫폼 지원을 강화했다.
- 180개 이상의 버그를 수정했다.

이 모든 것은 오픈 소스 프로젝트로 진행 중이며, 국제적인 커뮤니티의 참여로 진행 중이다.

## 실제 사용 사례
Vite+는 이미 1,300개 이상의 공개 리포지토리에서採用되고 있으며, 이를 통해 다양한 프로젝트에서 사용되고 있다.

## 로드맵
Vite+ 1.0을 향한 로드맵은 다음과 같다.
- 원격 캐싱을 추가할 계획이다.
- GitLab CI/CD용 설정 툴을 추가할 계획이다.
- 다양한 프레임워크와 플러그인 지원을 개선할 계획이다.
- 배포 채널을 추가할 계획이다.

## 시작하기
다음 명령어를 사용해서 Vite+를 설치할 수 있다.
```bash
sh curl -fsSL https://vite.plus | bash
```
그리고 새로운 프로젝트를 생성할 수 있다.
```bash
vp create
```
또는 기존 프로젝트에 Vite+를 적용할 수 있다.
```bash
vp migrate
```
기존 프로젝트에採用할 때는 마이그레이션 가이드를 읽어보는 것을 추천한다.
