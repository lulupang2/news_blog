### 요약
엑스코드 안열고 맥, 아이폰 앱 개발하는 법 ㅋㅋㅋㅋ

### 엑스코드 설치
먼저 엑스코드를 설치해야 함. 이유는 간단하고, 빌드에 필요한 툴이 엑스코드 안에 있다고 함. 그런데 설치하고 나서도 gui 안 들어가고 cli로만 가능하다는 점이 좋음 ㅋㅋ 

설치 후에 `xcode-select -p`로 현재 선택된 툴체인이 đúng인지 확인하고, 안되면 `sudo xcode-select -s /Applications/Xcode.app/Contents/Developer`로 설정 바꿔줌. 

### 엑스코드ген 설치
그다음에 엑스코드 프로젝트 관리를自动화 해주는 툴을 설치해야 하는데, 이름이 엑스코드젠임. 이건 홈브류로 쉽게 설치 가능함. 

엑스코드 프로젝트는 그냥 폴더인데, мак오에스가 파일인 것처럼 보이게 해주고, 안에 프로젝트 설정이나 파일 참조 같은 것들이 다 들어 있음. 그런데 이게 문제가 됨. 왜냐면 엑스코드가 항상 바꿔서 git 저장소에 관리하기 힘들어질 수록 있기 때문임. 

엑스코드젠은 프로젝트 설정을 yaml 파일로 만들고, 프로젝트 폴더를 자동으로 생성해 주고, 이 yaml 파일만 관리하면 되기 때문에 좋음. 

### 초기 설정
엑스코드 초기 설정은 gui로 해야 하는데, 그 다음부터는 cli로만 가능하다고 함. 초기 설정으로 아래와 같은 것들이 있음. 
- 엑스코드 라이선스 동의하고, 추가 컴포넌트 설치(명령어: `xcodebuild -license accept`)
- 애플 개발자 계정 추가 및 로그인 (설정 -> 계정 -> + -> 애플 아이디로 로그인) 
- 앱을_ship하기 위해서 개발자 id 애플리케이션 인증서 발급 받아야 함. (설정 -> 계정 -> 애플 아이디 관리 -> 인증서 -> + -> 개발자 id 애플리케이션) 

그리고 노타라이제이션 프로필을 생성해야 하는데, 이때 앱 이름으로 프로필 이름을 설정하고, 앱 관련 비밀번호 생성해야 함. 

이거 말고는 다 CLI로 할 수 있게 설정이 되어 있기 때문에, 개발자는 이 초기 설정 이후에는 gui를 안 들어가도 됨. 

### 배포
배포는 자동화 스크립트로 가능하다고 함. 일단 배포 스크립트에는 아래와 같은 단계들이 포함되어 있다고 함. 
- 프로젝트 재생성(xcodegen 명령어)
- 릴리즈 빌드
- 개발자 아이디로 서명
- 노타라이제이션
- 설치 

그리고 자동화가 안되는 경우에는 LLM(클라우드 코딩 툴) 도움을 받을 수 있다고 함. 

이런저런 설정이나 단계가 복잡하고 힘든 부분은 LLM으로 넘기고, 개발자들은 그냥 개발만 신경 써도 된다고 함.
