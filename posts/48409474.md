# 리눅스/유닉스 lost+found 폴더는 뭔가요?
리눅스/유닉스 운영체제에서 `/lost+found/` 폴더를 본 적이 있잖아요? 
이 폴더는 갑자기 시스템이 멈췄거나 파일 시스템에 문제가 생겼을 때.fsck라는 명령어를 사용해서 파일 시스템을 수리할 때 사용함. 
lost+found 폴더 안에는 원래 존재했지만 이름이나 위치가 사라진 파일들이 들어감. 이 파일들은 원래 삭제할 예정이었던 파일들일 수도 있고, 파일 시스템에 문제가 생겨서 사라진 파일들일 수도 있음.

### lost+found 폴더와 fsck
fsck 명령어는 파일 시스템을 수리하는 명령어로, 시스템이 멈춘 후에 자동으로 작동하기도 함. 
이때 fsck가 파일 시스템에서 문제가 있는 파일들을 찾으면 lost+found 폴더에 넣어줌. 
이 파일들은 이름이나 위치가 없기 때문에 lost+found 폴더에 넣어진 것들임.

### lost+found 폴더의 용도
lost+found 폴더는 파일 시스템이 손상되었을 때 파일을 회복하는 역할을 함. 
lost+found 폴더 안에 있는 파일들은 원래 존재했지만 이름이나 위치가 사라진 파일들이 들어가는데, 이 파일들은 시스템이 멈춘 후에 fsck 명령어를 사용해서 파일 시스템을 수리할 때 나타남.

### lost+found 폴더를 사용하는 방법
lost+found 폴더는 각 파티션에 하나씩 만들어지는 폴더로, 이 폴더 안에 있는 파일들은 파일 시스템의 손상으로 인해서 사라진文件들임. 
lost+found 폴더 안에 있는 파일들을 복구하기 위해서는 원래의 위치로 이동시키는 방법을 사용할 수 있음. 
예를 들어, lost+found 폴더 안에 `file`이라는 파일이 있으면, 이 파일을 원래의 위치로 이동시킬 수 있음.

### lost+found 폴더의 권한 문제
lost+found 폴더는 파일 시스템의 일부로, 이 폴더 안에 있는 파일들은 파일 시스템의 권한을 따름. 
따라서 lost+found 폴더 안에 있는 파일들을 실행하거나 삭제하기 위해서는 파일 시스템의 권한을 변경해야 할 수 있음. 
일반적으로 lost+found 폴더 안에 있는 파일들은 시스템 관리자의 권한으로 실행되거나 삭제되기 때문에, 일반 사용자가 lost+found 폴더 안에 있는 파일들을 처리하기 위해서는 시스템 관리자의 권한이 필요할 수 있음.

lost+found 폴더 안에 있는 파일들은 원래 존재했지만 이름이나 위치가 사라진 파일들이 들어가는데, 이 파일들은 시스템이 멈춘 후에 fsck 명령어를 사용해서 파일 시스템을 수리할 때 나타남.
