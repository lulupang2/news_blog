### Forge: 가드레일과 컨텍스트 관리로 모델 성능 극대화
Forge는 자기ホスト형 LLM 툴 콜링과 멀티스텝 에이전트 워크플로우를 위한 파이썬 프레임워크야. 쉽게 말해 모델을 더 똑똑하게 만들어 주는 툴 같은 거임. 
 llama-server, Ollama, Llamafile, Anthropic 같은 백엔드를 지원하니까 선택의 폭이 넓어. 
Forge를 이용하면_workflow runner, slot worker, 가드레일 미들웨어_ 등을 사용할 수 있어서 모델 성능을 극대화할 수 잇음. 
그리고 Proxy 서버도 지원하니까 모델을 더 쉽게 관리할 수 잇다는데, 이건 디벨로퍼들한테 엄청 좋은 소식일 듯?

###Forge를 어떻게 사용하면 되나요?
Forge는 파이썬 3.12 이상에서 사용할 수 잇으며, pip를 통해 설치할 수 있음.
```bash
pip install forge-guardrails
```
그리고 Forge를 사용하기 위해서는 백엔드를 설정해야하는데, llama-server, Ollama, Llamafile, Anthropic 중에서 선택할 수 잇음.
 llama-server를 사용하는 경우에는 다음 명령어를 통해 설치할 수 있음:
```bash
pip install -e ".[dev]"
git clone https://github.com/ggml-org/llama.cpp.git
cd llama.cpp
mkdir build
cd build
cmake ..
cmake --build .
```
그리고 Forge를 사용하기 위해서는 다음과 같은 코드를 작성하면 됨:
```python
import asyncio
from pydantic import BaseModel, Field
from forge import (
    Workflow,
    ToolDef,
    ToolSpec,
    WorkflowRunner,
    OllamaClient,
    ContextManager,
    TieredCompact,
)

# ...

async def main():
    client = OllamaClient(model="ministral-3:8b-instruct-2512-q4_K_M", recommended_sampling=True)
    ctx = ContextManager(strategy=TieredCompact(keep_recent=2), budget_tokens=8192)
    runner = WorkflowRunner(client=client, context_manager=ctx)
    await runner.run(workflow, "What's the weather in Paris?")
asyncio.run(main())
```
