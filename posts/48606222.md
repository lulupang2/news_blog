### SSH 터널링 이란?
SSH 터널링은SSH 프로토콜을 사용하여 안전하게 데이터를 전송하는 기술이야. 쉽게 설명하면, 내가 원하는 서비스나 서버에 접근할 수 있게 해주는 일종의 '똑소리'라고 생각할 수 있지 ㅇㅇ.

### 로컬 포트 포워딩
로컬 포트 포워딩은 로컬 머신에서 리모트 서버의 특정 포트에 접근할 수 있게 해줘. 예를 들어, 리모트 서버의 8080 포트에 접근하고 싶지만, 내 로컬 머신에서만 접근할 수 있다면, 로컬 포트 포워딩을 사용하면 됨 ㄹㅇ.
```bash
ssh -L [로컬_주소:]로컬_포트:리모트_주소:리모트_포트 [사용자@]SSH_대상_주소
```
로컬 포트 포워딩을 사용해서 내 로컬 머신에서 리모트 서버의 특정 포트에 접근할 수 있어. 진짜 쉽고 좋은 기능이야 ㅋㅋㅋㅋ.

### 로컬 포트 포워딩 예시
예를 들어, 리모트 서버의 80 포트에 접근하고 싶지만, 내 로컬 머신에서만 접근할 수 있다면, 다음 명령어를 사용할 수 있어:
```bash
ssh -f -N -L 8080:localhost:80 사용자@리모트_서버_주소
```
그럼, 내 로컬 머신의 8080 포트로 리모트 서버의 80 포트에 접근할 수 있어.真的便捷呢～.

### 리모트 포트 포워딩
리모트 포트 포워딩은 리모트 서버에서 내 로컬 머신의 특정 포트에 접근할 수 있게 해줘. 예를 들어, 내 로컬 머신의 8080 포트에 접근하고 싶지만, 리모트 서버에서만 접근할 수 있다면, 리모트 포트 포워딩을 사용하면 됨 ㅋㅋ.
```bash
ssh -R [리모트_주소:]리모트_포트:로컬_주소:로컬_포트 [사용자@]SSH_대상_주소
```
리모트 포트 포워딩을 사용해서 리모트 서버에서 내 로컬 머신의 특정 포트에 접근할 수 있어. 진짜 편리한 기능이야 ㅇㅈ.

### 리모트 포트 포워딩 예시
예를 들어, 내 로컬 머신의 80 포트에 접근하고 싶지만, 리모트 서버에서만 접근할 수 있다면, 다음 명령어를 사용할 수 있어:
```bash
ssh -f -N -R 0.0.0.0:8080:localhost:80 사용자@리모트_서버_주소
```
그럼, 리모트 서버의 8080 포트로 내 로컬 머신의 80 포트에 접근할 수 있어.真的太方便了～ ㅋㅋㅋㅋ.
