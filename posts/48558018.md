### Bash로 HTTP 요청 날리기
   Bash에서 컬없이 HTTP 요청을 날릴 수 있는 방법이 있더라. `/dev/TCP`를 이용하면 됨.
   예를 들어, 도커 내부 네트워크에서 컨테이너 간 통신을 확인해야 하는 상황이였음. 
   서비스에 GET 요청을 날려서 응답이 오는지 확인하고 싶었는데, 컬이나 wget이 없음. 
   다행히 Bash 자체에서 TCP 소켓을 열 수 있어서 HTTP 요청을 직접 날릴 수 있었음.
   ```
   exec 3<>/dev/tcp/서비스_이름/포트
   printf 'GET /health HTTP/1.1
Host: 서비스_이름
Connection: close

' >&3
   cat <&3
   ```
   이렇게 하면 응답 전체가 출력됨. 
   만약 헤더를 추가하고 싶으면, 요청의 끝 부분에 공백 줄을 넣기 전에 헤더를 넣으면 됨.
   ```
   exec 3<>/dev/tcp/서비스_이름/포트
   printf 'GET /v1/models HTTP/1.1
Host: 서비스_이름
Authorization: Bearer %s
Connection: close

' "API키" >&3
   cat <&3
   ```
   `/dev/tcp`는 실제 디바이스 파일이 아니고, Bash 내부에서 처리하는 리디렉션임.
   Bash에서 DNS.lookup과 연결을 처리해주고, `exec 3<>`를 이용해서 소켓에 파일 디스크립터를 부여함. 
   이 기능은 컬처럼 완전한 HTTP 클라이언트가 아님. HTTP를 제대로 파싱하지도 못해. 
   리다이렉션도 못하고, 응답을 조각조각 나눠받는 것도 못해.壓縮도 못하고, TLS도 못해.
   컬은 이런 모든 걸默默히 해주는구나. 
   `Connection: close` 헤더를 넣어줘야지, 서버에서 응답 후 커넥션을 닫아줌.
   그러지 않으면 서버에서 커넥션을 닫지 않으니까, `cat <&3`에서 영영 응답을 기다리다가 죽음.
   `timeout` 명령을 이용해서 커넥션이 영영 안 닫히는 경우를 대비하면됨.
   이 기능은 TLS를 지원하지 않음. `/dev/tcp`로 열린 소켓은 평문 HTTP만 가능.
   HTTPS는 `openssl s_client`를 이용해야 함.
   이 기능은 Bash만 지원함. POSIX에 없어서, Dash나 Zsh에서는 사용할 수 없음.
   Bash를 직접 호출해서 써야 함.
   Bash가 `--enable-net-redirections` 옵션으로 빌드되어야만 사용할 수 있음.
