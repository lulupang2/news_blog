# 슬럼버: 터미널 기반 HTTP 클라이언트
슬럼버는 리눅스 개발자들을 위한 터미널 기반 HTTP 클라이언트야. REST나 다른 HTTP 클라이언트랑 편하게 작동할 수 있도록 설계됨. 이 친구는 두 가지 모드를 지원하는데, 하나는 터미널용 인터페이스(TUI)고, 다른 하나는 명령行 인터페이스(CLI)임. TUI는 사용성이 개굿고, 바로바로 요청 보내고 응답 보기 좋은 인터페이스야. CLI는 간단한 요청 보내거나 스크립트 짤 때 유용하다는 점이 다르지. 슬럼버의 목표는 사용하기 쉬우며, 설정하기 간단하고, 공유하기 좋은 거임. 이렇게 하면 YAML 파일로 요청 컬렉션을 관리할 수 있단다. 

## 핵심 기능
- 터미널에서 HTTP 요청 보내기: 바로 요청을 보내고, 응답을 볼 수 있지. 
- YAML 파일로 설정 관리: 요청 컬렉션을 YAML 파일로 관리하면 설정하기도 편하고, 공유하기도 쉬어. 
- TUI와 CLI 지원: 개발자들마다 쓰는 스타일이 달라, 두 가지 모드를 모두 지원해 줌.
