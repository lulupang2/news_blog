# 개빡치는 스크린샷 자동 업데이트 기능 만들었음 ㅋㅋㅋ

진짜로 내가 젤리에서 만든 가장 신기한 기능인데 아마도 아무도 알아채지 못할걸؟ 어쨌든 웹애플리케이션의 도움말 센터나 문서 사이트를 관리해 본 사람이라면 스크린샷의 고통을 알거야. 도움말 기사를 작성하고, 문서하고 하는거야. 그때마다 스크린샷을 찍어서 편집하고 업로드하고 그런거야. 그런데 UI를 조금만 수정해도 스크린샷이 다 잘못돼버리잖아? ㅋㅋㅋㅋ

그러길래 나도 만들었어. 젤리의 도움말 센터에 스크린샷 자동 캡처 기능 만들었음. 마크다운에 조금이라도 트위스트가 있다면 자동으로 스크린샷을 캡처해준대. 

### 어떻게 동작하는 거냐하면

마크다운에 이런 주석을 추가해주면된다함. `<!-- SCREENSHOT: acme-tools/inbox | element | selector=#inbox-brand-new-section -->` 

이거는 젤리에게 이런 명령이래. 인박스 페이지에서 이 선택자에 맞는 엘리먼트를 찾으라고. 그런데 이게 끝이 아니었음. 스크린샷 시스템이 자동으로 동작하기 때문에 UI를 수정하고 빌드를 돌리면 스크린샷이 자동으로 업데이트 돼. ㅋㅋㅋㅋ

### 이게 어떻게 동작하는 거냐 하면

이 시스템은 레일즈 애플리케이션의 빌드 시스템에 통합되어있음. 마크다운 파일들에서 스크린샷 주석들을 찾아서 자동으로 캡처해주는 거지. 

### 어떻게 해 볼 수 있냐하면

 젤리 앱에서 `rails manual:build` 라고 친 후 엔터를 치면 자동으로 모든 스크린샷을 캡처하고 도움말 페이지를 빌드해 줌. 이제 UI를 수정하고 빌드를 돌리면 스크린샷도 자동으로 업데이트 돼. ㅋㅋㅋㅋ

### 개빡치는 점

이제 스크린샷을 업데이트할 때마다 수작업을 할 필요가 없다고!! 자동으로 업데이트 되는 거 보소 ㅋㅋㅋㅋ
