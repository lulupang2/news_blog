> **🔗 원문 보러가기:** [Caddy compatibility for zeroserve: 3x throughput and 70% lower latency](https://su3.io/posts/zeroserve-caddy-compat)

이번에는 zeroserve라는 HTTPS 서버에 캐디 호환성이 추가됨. [zeroserve](https://github.com/losfair/zeroserve)는 유저스페이스에서 eBPF 스크립트를 실행하는 고성능 HTTPS 서버야. 이제 캐디파일로 zeroserve 서버를 구성할 수 있게 됨. zeroserve는 캐디 파일을 JIT 컴파일해서 eBPF로, 그리고 네이티브 머신 코드로 컴파일해. 이렇게 하면 io_uring 이벤트 루프에서 실행할 수 있어. 
   
   zeroserve 성능은 다음과 같음:
   - zeroserve-clang: 38,948 req/s, 1.45ms, 3.91ms, 30.9 MiB
   - zeroserve-tcc: 36,653 req/s, 1.67ms, 4.00ms, 34.2 MiB
   - 캐디: 12,529 req/s, 4.74ms, 13.11ms, 67.4 MiB
   - nginx: 37,424 req/s, 1.57ms, 4.24ms, 25.7 MiB
   
   이 성능은 HTTPS 리버스 프록시, 2 쓰레드, AMD Ryzen 7 3700X에서 테스트한 결과임. zeroserve는 튜링 완전형 eBPF야. 그래서 캐디 파일에서 커스텀 코드를 호출할 수 있어. 예를 들어, S3 호환 버킷에 AWS SigV4 인증으로 리버스 프록시할 수 있음. 
   
   zeroserve 사용하는 방법은 다음과 같음:
   ```bash
   curl -fL -o zeroserve https://github.com/losfair/zeroserve/releases/download/v0.2.11/zeroserve-$(uname -m)-linux
   chmod +x zeroserve
   ./zeroserve --caddy /etc/caddy/Caddyfile
   curl http://127.0.0.1:8080
   ```
   
   개빡치는 점은 zeroserve가 아직 초기 버전이라서 불안정한 점이 있음. 그러나 성능 개선이 많아서期待되는 기술임. zeroserve를 사용해서 성능 개선할 수 있을지 기다려 볼 것임. ㄹㅇ
