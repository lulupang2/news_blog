## 개요
Theseus 라는건 윈도우 프로그램을 웹 어셈블리(wasm)로 번역해주는 툴인데, 개발자가 웹 버전으로 만드는 과정을 공유함.

## win32를 wasm으로
Theseus 라는 건 원래 win32/x86 에뮬레이터임. 근데 이제 웹 어셈블리 출력도 지원하는 것 같음. 그러니까 exe 파일을 웹에서 돌아가게 할 수 있는 것임.

## Blocking vs Non-Blocking
윈도우 프로그램에서 블록킹이 발생하면 웹 브라우저에서 돌아가게 할 때 문제가 생김. 왜냐면 메인 스레드가 블록킹되면 브라우저 이벤트 루프가 돌아가지 않으니까. 그래서 개발자는 워커 스레드를 사용해서 블록킹을 피하는 방법을 사용했음.

## 워커 스레드와 메인 스레드
웹 워커를 사용해서 에뮬레이터를 돌아가게 하는 건데, 워커 스레드에서 블록킹이 발생하면 메인 스레드에서 워커 스레드를 깨우는 식으로 동작함._shared 메모리를 사용해서 워커 스레드와 메인 스레드가 데이터를 주고받음.

## 문제점
인데, TypeScript를 사용해야 해서 러스트 코드와 브라우저 코드를 분리해야 했음. 또, 워커 스레드에서 DOM에 접근할 수 없어서 TypeScript를 사용해야 했음.

## 결론
요약하자면,_win32 게임이나 프로그램을 웹 브라우저에서 돌아가게 하는 방법은 아직까지 완벽하지 않지만, 러스트와 웹 어셈블리, 워커 스레드를 사용해서 개선하는 방향은 좋은 것 같음.
