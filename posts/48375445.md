### Branchless Quicksort登場!

퀵소트는 우리가 가장 자주 쓰는 정렬 알고리즘 중 하나임. 하지만 최근에 나온 Branchless Quicksort라는 알고리즘이 퀵소트를 개량해서 속도 개선한 거 같음.
이 알고리즘은 C와 C++로 구현됨. 그래서 C/C++용 API로도 제공됨.

#### 성능 테스트 결과

성능 테스트 결과는 아래와 같음. 
*   애플 M1에서 실행한 결과:
    *   std::sort: 1.33초
    *   pdqsort: 1.33초
    *   Branchless Quicksort: 0.97초
*   AMD 라이젠에서 실행한 결과:
    *   std::sort: 5.56초
    *   pdqsort: 2.81초
    *   Branchless Quicksort: 2.06초

테스트 결과 Branchless Quicksort가 std::sort와 pdqsort보다 훨씬 빠르게 나옴. 
#### Branchless 프로그래밍이 뭐길래?

모던 CPU에서는 브랜치 미스피디케이션(branch misprediction)을 피하는 게 프로그램을 속도 개선하는 데重要함.
예컨대 아래 두 코드 비교해ボ면 알 수 있음:
```c
// 일반적인 퀵소트
for (int i = 0; i < 1000; i++) {
    if (numbers[i] < 500) {
        small_numbers[smlen] = numbers[i];
        smlen += 1;
    }
}

// 브랜치 없는 퀵소트
for (int i = 0; i < 1000; i++) {
    small_numbers[smlen] = numbers[i];
    smlen += (numbers[i] < 500);
}
```
브랜치 없는 버전이 훨씬 빠르게 동작하는 거시다.

#### 블라퀴소트 사용법

C++버전에선 그냥 헤더 파일 include만 하면 돼:
```cpp
#include "blqs.h"
double data[SIZE];
blqs::sort(data, data + SIZE);
```
C 버전은 조금 귀찮지만 대충 마크로 정의해주면 되긴 하네:
```c
#define BLQS_CMP(a, b) ((a) < (b))
#define BLQS_TYPE double
#include "blqsort.h"
double data[SIZE];
blqsort(data, SIZE);
```
#### 커스텀 데이터 구조체 정렬

이게 가장 중요한 사항인듯. 디씨인사이드 같은 커뮤니티에선 정렬할 데이터가 단순 숫자뿐만 아니라 다양한 구조體를 포함하는 경우가 많기 때문임.
예를 들어 아래와 같이 커스텀 구조체를 정렬하고 싶다고 해보면:
```c
struct entry {
    int id;
    int value;
};

struct entry data[SIZE];
```
이럴 때는 아래처럼 비교 함수를 정의해서 사용하면 됨. 
```cpp
#define BLQS_CMP(a, b) (((a).id) < ((b).id))
#define BLQS_TYPE struct entry
#include "blqsort.h"
blqsort(data, SIZE);
```
C++버젼은 비교 함수를 구조체 내에 멤버 함수로 정의해서 사용할 수 있으니 더 간단하당:
```cpp
struct entry {
    int id;
    int value;
    bool operator<(const entry& other) const {
        return id < other.id;
    }
};

struct entry data[SIZE];
blqs::sort(data, data + SIZE);
```
이런 커스텀 데이터 구조체를 정렬하는 데도 Branchless Quicksort가 훨씬 빠르게 나오는 중:
*   애플 M1: std::sort (3.46초), pdqsort (3.46초), Branchless Quicksort (0.96초)
*   AMD 라이젠: std::sort (4.75초), pdqsort (4.72초), Branchless Quicksort (2.20초)

 Branchless Quicksort는 정말 빠르내. 이제 정렬할 때 이 알고리즘 쓰는 게 좋을듯.
