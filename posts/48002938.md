# 터미널 접근성 문제
터미널에 ilgili 개발자들은 대부분 ' 그래 터미널에 글자만 있으면 자동으로 접근성이 좋은거 아님?' 이라고 생각하는데 실제로는 그러지 않음. 
터미널에서 동적인 값(예를 들어, 로딩바, 타임스탬프 등)이 업데이트가 되면 화면 리더가 이 값을 계속 읽게되는데, 이게 사용자한테는 '현재 입력칸에 들어있는 값'을 읽는 것보다 우선순위가 더 높은듯.
그리고 터미널 인터페이스들의 업데이트가 자주 일어나면, 화면 리더가 이 값을 계속 읽어주면서 사용자가 입력하고자 하는 값을 읽을 수 없게 됨.
또한 최신 터미널에서 많이 사용하는 UI 프레임워크(예: Ink, Bubble Tea)들이 화면을 canvas로 생각하는 트렌드가 있는데, 이 경우에는 터미널에서 제공하는 스크롤링 기능을 사용하지 않고, 직접 텍스트를 업데이트한다는점에서 접근성이 구림.
요새 개발하는 터미널 프로그램들은 거의 다 그래픽 인터페이스 처럼 동적으로 레이아웃이 변하니까, 접근성이 구린거임.
예를들자면, gemini-cli라는 프로그램은 단순聊天 인터페이스 처럼 보이지만, 내부적으로는 Ink 프레임워크를 사용하고 있어서, 화면 리더를 쓰면, 계속 업데이트 되는 값을 읽어주면서 사용자가 입력값을 제대로 듣지 못하게 됨.
이 문제는 개발자가 '접근성'에관한 이해가 부족해서 생기는 문제임.
터미널에서의 접근성을 높이기 위해서는 개발자가 터미널의 특성을 이해하고, 접근성을 고려해서 개발해야함.
터미널에서 제공하는 기능들을 잘 사용하고, 화면 리더가 제대로 값을 읽을 수 있게 개발을 해야함.
그리고 접근성에 대한 문제를 제기하면, 개발자는 이를 묵살하거나, 해결하지 않음.
그러니까, 우리는 개발자들에게 접근성에 대한 문제를 계속 제기하고, 개발자들이 접근성을 고려해서 개발하게 해야함.
