# Tendril 소개
Tendril이라는 프로젝트는 인공지능이 스스로 도구를 만들고 등록하고 다시 사용하는 것을 가능하게 해줌.

## 어떻게 작동하는가?
사용자가 Tendril에게 어떤 일을 해달라고 요청하면, Tendril은 먼저 해당 요청을 수행할 수 있는 도구가 있는지 확인함.
만약 도구가 있다면, 그 도구를 사용하고, 없다면 새로운 도구를 작성하고 등록한 후에 사용함.

예를 들어, 사용자가 '해커뉴스 상위 기사를 가져와줘'라고 요청하면, Tendril은 기사 가져오는 도구가 있는지 확인하고, 없다면 해당 도구를 작성한 후에 가져옴.

## 핵심 구성 요소
Tendril의 핵심은 Strands SDK와 Tauri를 사용해서 만들어짐.
사용자는 Tauri를 통해 사용자 인터페이스를 제공하고, Strands SDK를 통해 Agent를 생성함.
Agent는 Bedrock 모델과 4개의 기본 도구를 가지고 있어.

## Agent Loop
Agent Loop는 Tendril의 핵심적인 부분으로, 사용자 요청을 처리하고 도구를 생성하고 등록하는 역할을 함.
Agent Loop는 사용자 요청을 받은 후에, 도구를 생성하고 등록하고, 도구를 사용해 요청을 처리함.

## 기술 스택
*   Tauri 2.x (Rust)
*   Frontend: React 18 + TailwindCSS v4
*   Agent: TypeScript (Node.js SEA binary)
*   Agent framework: @strands-agents/sdk
*   Inference: AWS Bedrock (Claude via Strands BedrockModel)
*   Code sandbox: Deno (bundled, subprocess with permission flags)
*   Protocol: JSON-RPC 2.0 / NDJSON over stdio

## 프로젝트 구조
Tendril 프로젝트는 두 개의 주요 디렉토리로 구성됨.
*   `tendril-agent`: Agent 관련 소스 코드가 들어있는 디렉토리임.
*   `tendril-ui`: Tauri를 사용한 데스크톱 애플리케이션 관련 소스 코드가 들어있는 디렉토리임.

## 시작하기
Tendril을 시작하려면 다음 명령어를 실행하면 됨.
```bash
git clone https://github.com/serverless-dna/tendril.git
cd tendril
make dev
```
이 명령어는 Tendril 프로젝트를 복제하고, 의존성 패키지를 설치하고, Agent를 빌드한 후에 Tauri 개발 모드로 실행함.

## 설정
Tendril의 설정은 `~/.tendril/config.json` 파일에 저장됨.
```json
{
  "workspace": "/Users/you/tendril-workspace",
  "model": {
    "provider": "bedrock",
    "modelId": "us.anthropic.claude-sonnet-4-5-20250514",
    "region": "us-east-1",
    "profile": "your-aws-profile"
  },
  "sandbox": {
    "denoPath": "deno",
    "timeoutMs": 45000,
    "allowedDomains": []
  },
  "agent": {
    "maxTurns": 100
  }
}
```
이 설정 파일에는 작업 공간 경로, 모델 제공자, 모델 ID, 지역, 프로파일과 같은 정보가 들어있음.

## 결론
Tendril은 인공지능이 스스로 도구를 만들고 등록하고 다시 사용하는 것을 가능하게 해주는 신세계임.
사용자가 Tendril에게 어떤 일을 해달라고 요청하면, Tendril은 해당 요청을 수행할 수 있는 도구가 있는지 확인하고, 없다면 새로운 도구를 작성하고 등록한 후에 사용함.
Tendril은 Strands SDK와 Tauri를 사용해서 만들어졌고, 사용자 인터페이스는 Tauri를 통해 제공되고, Agent는 Strands SDK를 통해 생성됨.
