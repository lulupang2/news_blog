오픈 소스 모델로 갈아타는 거 아깝지 않은 이유가 등장했어. 
최근에는 리눅스를 쓸 때 Compatibility 문제가 많았어. 예를 들어 워드 문서나 파워포인트를 제대로 렌더링하지 못하는 경우나 전문 용도의 파일을 쉽게 열 수 없는 경우도 많았지. 
그리고 소프트웨어 생태계도 지금보다 못했어. 오픈소스로 만든 프로젝트가 많았지만 다듬어지지 않은 상태だった거지. 
나도大学 때까지 윈도우를 쓰다가 연구를 졸업하고 나서야 리눅스로 갈아탔어. 
하지만 지금은 거의 그런 문제가 없어졌음. 대부분의 생산성 소프트웨어가 웹 앱으로 제공되고 리눅스도 성숙했고 오픈 소스 소프트웨어도 좋아졌어. 
아마도 CAD 같은 특정 응용 소프트웨어만 제외하고는 리눅스와 오픈 소스 소프트웨어를 쓰는 게 전에 비해 많이 나아졌다고 생각해. 
이제는 오픈모델을 쓰는 건 페널티가 적어졌어. 오픈 소스 모델도 이제는 거의 쓸만해졌고 대부분의 오픈 소스 모델이 MIT 라이선스로 제공되고 있어. 
최근에는 Claude와 GPT 같은 거대 모델들이 항상 Leaderboard에서 1등을 하는데 요즘에는 오픈 소스 모델도 거의 그 수준에 가깝게 왔어. 
물론 Compatibility도 아직 좀 부족한데요. 예를 들어 Claude 같은 경우에는 코드가 잘 동작하고 API도友好해서 사용하기 쉽지만 오픈 모델은 이런 점에서 부족한 편이야. 
오픈 모델을 쓰는 방법은 회사에서 제공하는 경우와 서드파티에서 제공하는 경우인데 둘 다 좀 문제가 있어. 
예를 들어 데이터 공유 같은 문제가 있는데 나는 클라이언트의 기밀 정보를 이런 모델에 넘기고 싶지 않아. 
그러면 직접 모델을 돌리는 방법도 있는데 이 경우에도 비용이 많이들고 복잡해서 실용적이지가 않아. 
요즘에는 이런 문제가 해결되는 듯한데요. Claude 같은 모델이 ID 인증을 도입하면서 사용자가 직접 모델을 실행하는 비율이 높아지고 있어. 
이제는 오픈 소스 모델로 갈아타도 페널티가 적을 것 같아. 나는 이미 오픈 소스 모델을 로컬이나 클라우드에서 실행할 수 있는 환경을 마련했고 대부분의 오픈 소스 모델도 리더보드의 TOP 모델과几乎 같은 수준에 왔어. 
물론 단기적으로는 생산성에 약간의 하락이있을 수 있겠지만 इसस 전에比하면 리눅스를 쓰던 때가 훨씬 낫다니까요. 
요즘에는 오픈 소스 모델도 쓰기 좋은 환경이 형성되어 가는 중인 거 같아. 
이제는 오픈 소스 모델을 쓰는 게 이전보다는 훨씬 나은 선택지가 된 거 같아. 
ㄹㅇ 이런 환경이 지속된다면 앞으로는 오픈 소스 소프트웨어의時代일지도 몰라.
