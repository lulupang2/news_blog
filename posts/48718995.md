# 와일드한XP 지원 프로젝트 
      Principia 개발자들이 만화처럼 Windows XP 지원 프로젝트를 진행 중임. 
      Principia는 원래 2014년에 출시되었을 때 윈도우 XP에서 돌아갈 수 있었음. 
      하지만 요즘에는 빌드툴이나 의존성이 바뀌어서 XP에서 돌아가지는 않음. 
      그래서 개발자들은 직접 빌드툴을 만들어서 XP에서 돌아갈 수 있는 Principia를 만들어보겠다고 함. 
      ## 툴체인 만들기 
      개발자들은 GCC, binutils, mingw-w64를 직접 빌드해야했음. 
      하지만 기존의 mingw-w64 툴체인은 XP에서 돌아가지 않음. 
      그래서 개발자들은 자신만의 툴체인을 만들기로 결정함. 
      ## 툴체인 빌드 
      개발자들은 Dockerfile을 사용해서 툴체인을 빌드할 수 있었음. 
     하지만 GMP라이브러리가最新 GCC와 함께 컴파일되지 않아 패치해야했음. 
      이후로 빌드는 순조롭게 진행되었음. 
      ## 디펜던시 빌드 
      Principia에는 여러 디펜던시가 있음. 
      이 중에 curl은 XP를 지원하지 않음. 
      그래서 개발자들은 구버전의 curl을 사용해야했음. 
      ## Principia 빌드 
      모든 디펜던시를 빌드한 후, Principia를 빌드할 수 있었음. 
      하지만 컴파일 에러가 발생했음. 
      이 에러는 Win32 API 호출 규약(__stdcall)과 관련이 있었음. 
      ## 결론 
      개발자들은 만화처럼 XP 지원 프로젝트를 진행 중임. 
      아직은 많은 문제가 있지만, 개발자들의 노력 덕분에 Principia가 XP에서 돌아갈 수 있다면 정말 감동일 것임.
