### 크롬 브라우저의 새로운 보안 취약점 등장 
   
   요즘 크롬 브라우저가 핫하긴 한데, 이번에는 OS 정보를 유출하는 취약점이 발견됐 븅신들아. 
   이건 148 버전부터 생긴 건데, Math.tanh 함수를 호출했을 때 브라우저가 어떤 OS를 쓰고 있는지 알 수 있게 됨. 
 
   ### 왜 이런 일이 발생했을까? 
   
   이유는 IEEE 754 표준 때문이라 븅신들아. 이 표준은 double을 저장하는 방식을 정의하는데, sin, cos, tanh 같은 함수들이 정확하게 반올림되어야 하는 것은 정의하지 않는 븅신들임. 
   그래서 각 벤더들은 속도를 위해 정확도를 조금씩 양보하는 자신의 minimax 계수와 Lookup 테이블을 사용해 libm을 구현 븅신들아. 
 
   ### 결과는? 
   
   결국에는 Linux, macOS, Windows 등이 서로 다르게 동작하 븅신들아. 
   예를 들어, Math.tanh(0.8)를 호출하면 Linux와 macOS는 거의同じ 결과를 반환하지만, Windows는 조금 다른 결과를 반환 븅신들아. 
 
   ### 어떻게 막을 수 있을까? 
   
   방법은 두 가지가 있음 븅신들아. 
   첫째, 정확하게 동일한 알고리즘을 구현すれば 되 븅신들아. 
   하지만 이는 쉽지 않은 일이 븅신들아. 각 OS의 libm을 역공학으로 분석하고, 그 결과를 portable C로 구현해야 하니까 븅신들아. 
   
  第二, 원본 코드를 바로 사용할 수 있음 븅신들아. 
   예를 들어, Windows의 UCRT DLL을 메모리에 로드하고, 그 안의 함수를 호출하면 됨 븅신들아. 
   하지만, 이는 ABI 경계를 건너야 하므로 주의가 필요 븅신들아. 
   
   ### 그래서? 
   
   결국, 크롬 브라우저의 새로운 보안 취약점은 Math.tanh 함수의 OS 정보 유출을 막기 어렵 븅신들아.
