# Claude Code 핵심 노트: 이렇게 쓰면 초고수되는 코드 작성법임 ㅋㅋ

BuildingBetterSubscribeSign in
Claude Code의SOURCE CODE를 읽어봤음. 그래서 공식 문서에 없는 설정값들을 공개함. 훅 필드들은 명령어를 재작성할 수 있고, 에이전트 메모리는 영속적임. 그리고 영어로 된 규칙들로 자동 모드를 설정할 수 있음. 셀프 업그레이드되는 드림 루프도 가능하고, 모든 예제는 복사-붙여넣기 직후 바로 동작함. 

André FigueiraApr 01, 2026

공식 문서에는 잘 안 나와 있는 Claude Code의 기능들이 좀 있음. YOLO Classifier는 실제 변수명임. 이건 yoloClassifier.ts 파일에 있지. 설정 파일에 plain English로 적는 환경 변수 설정들로AUTO-MODE PERMISSION SYSTEM을 설정할 수 있음. 예를 들면 "이 서버는 스테이징 서버이고, 파괴적인 작동은 허용됨" 이런 식으로 설정할 수 있음. 하지만 이게 문서에 없는 기능임.

Claude Code에 이런 기능들이 많이 숨겨져 있음. 이것들은 모두 지금 바로 작동하고, 모든 예제는 프로젝트에 붙여넣기 직후 바로 사용할 수 있음. 버전 관리에 관해서는, 이发现은 @anthropic-ai/claude-code@2.1.87 버전에 있는 기능임. 문서화되지 않은 기능들은 릴리즈 버전 사이에서 변경될 수 있으므로, 이것을 현재 버전에 대한 스냅샷으로 간주하셈. EXPERIMENTAL로 명명된 필드는 Anthropic 엔지니어们에 의해 명시적으로 불안정하다고 표시됨.

## 시작하기

 快速 참고: 설정 파일은 어디에 있는지 알려줌
- 설정 파일: ~/.claude/settings.json (개인용) 또는 .claude/settings.json (프로젝트용, 깃으로 공유 가능)
- 스킬 파일: ~/.claude/skills/<스킬명>/SKILL.md (개인용) 또는 .claude/skills/<스킬명>/SKILL.md (프로젝트용)
- 에이전트 파일: ~/.claude/agents/<에이전트명>.md (개인용) 또는 .claude/agents/<에이전트명>.md (프로젝트용)
- 훅 스크립트 파일: ~/.claude/hooks/ (이 conventions를 따르면 됨). 스크립트 파일은 chmod +x로 실행 가능해야 함.
프로젝트 파일들은 깃으로 공유할 수 있고, 개인 파일들은 개인용임.

## 훅 스크립트

 훅 스크립트들이 돌아가는 방식을 설명함. 훅 스크립트들은 JSON을 출력할 수 있고, 이벤트에 따라 Claude의 행동을 수정할 수 있음. 이건 문서에 안 나와 있지만, Claude Code의 기능임.

### PreToolUse 훅 스크립트

PreToolUse 훅 스크립트는 tool_input을rewrite할 수 있음. permissionDecision도 설정할 수 있음. 이 설정값들은 문서에 안 나와 있지만, 소스 코드에서 찾아볼 수 있음.

### SessionStart 훅 스크립트

SessionStart 훅 스크립트는 watchPaths, initialUserMessage, additionalContext를 설정할 수 있음. 이 설정값들은 문서에 안 나와 있지만, 소스 코드에서 찾아볼 수 있음.

### PostToolUse 훅 스크립트

PostToolUse 훅 스크립트는 updatedMCPToolOutput, additionalContext를 설정할 수 있음. 이 설정값들은 문서에 안 나와 있지만, 소스 코드에서 찾아볼 수 있음.

### PermissionRequest 훅 스크립트

PermissionRequest 훅 스크립트는 decision, updatedInput, updatedPermissions를 설정할 수 있음. 이 설정값들은 문서에 안 나와 있지만, 소스 코드에서 찾아볼 수 있음.

## 스킬

스킬은 Claude Code의 확장 기능임. 스킬은 model, effort, hooks 等의 필드를 가지고 있을 수 있음. 이 필드들은 문서에 안 나와 있지만, 소스 코드에서 찾아볼 수 있음.

### model 필드

model 필드는 사용할 모델을 설정할 수 있음. haiku 또는 opus를 설정할 수 있음.

### effort 필드

effort 필드는 모델의 노력을 설정할 수 있음. low, medium, high, max를 설정할 수 있음.

### hooks 필드

hooks 필드는 스킬에 대한 훅 스크립트를 설정할 수 있음. 훅 스크립트는 스킬이 활성화될 때 실행되며, 스킬이 비활성화되면 실행되지 않음.

## 에이전트

에이전트는 Claude Code의 확장 기능임. 에이전트는 custom agent를 설정할 수 있음.

### agent 필드

agent 필드는 사용할 에이전트를 설정할 수 있음.

### shell 필드

shell 필드는 사용할 쉘을 설정할 수 있음.
