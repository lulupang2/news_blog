### AI의 무한 가능성

요즘 회사에서 AI 도입하는 거 보니깐 참 웃ギ는 점이 많음. 회사에서는 GitHub Copilot, ChatGPT Enterprise, Claude, Gemini, Cursor 등 다양한 AI 툴을 도입했지만, 실제로 업무에 반영되는지는 미지수임. 일부 팀에서는 AI를 활용하여 효율성을 높이는 반면, 다른 팀에서는 AI가 도입되자마자 사망하는 경우도 많음. 

이러한 상황은 Ethan Mollick이라는 전문가가 언급한 'AI의 제한된 학습' 때문인 것 같음. Mollick은 'Making AI Work: Leadership, Lab, and Crowd'에서 개인의 생산성 향상은 자동으로 조직의 생산성 향상으로 이어지지 않는다는 점을 강조함. 

예를 들어 개발자들은 GitHub Copilot을 사용하여 코드를 자동으로 완성할 수 있지만, 이것이 조직 전체의 생산성 향상으로 이어지지는 않음. 또 한편으로는 AI의 효율성에 너무 익숙해져서 AI 없이 일을 할 수 없는 상황이 발생할 수 있음. 

### 비즈니스 프로세스와 AI의 충돌

현재 많은 회사들이 AI를 도입하고 있지만, 실제로 비즈니스 프로세스에 반영되기에는 많은 어려움이 있음. 일부에서는 AI를 활용하여 자동화된 업무 프로세스를 구축하고 있지만, 대부분의 경우에는 기존의 비즈니스 프로세스를 그대로 유지하고 있음. 

문제는 기존의 비즈니스 프로세스가 AI의 속도에 따라 설계되지 않았기 때문에, AI의 효율성을 제대로发揮하지 못하는 것임. 예를 들어 Scrum과 같은 애자일 개발 프로세스는 human iteration이 비싼 경우에 적합한 반면, AI를 활용하면 human iteration의 비용이 크게ลด어들음. 

### 새로운 비즈니스 프로세스 설계의 필요성

AI의 도입으로 인하여 새로운 비즈니스 프로세스를 설계할 필요성이 대두되고 있음. 기존의 비즈니스 프로세스를 AI의 속도에 맞게 개선하여야함. 

예를 들어 Agent Operations, Loop Intelligence, Agent Capabilities와 같은 새로운 개념을 도입하여야함. Agent Operations는 AI툴의 관리와 제어를 위한 프로세스를 의미하고, Loop Intelligence는 AI를 활용하여 업무 프로세스를 개선하는 프로세스를 의미함. Agent Capabilities는 AI의 능력을 극대화하여 활용하는 프로세스를 의미함. 

### 결론

결국 AI의 도입은 새로운 비즈니스 프로세스의 설계를 요구함. 기존의 비즈니스 프로세스를 AI의 속도에 맞게 개선하여야만 AI의 효율성을 제대로 발휘할 수 있음. 새로운 비즈니스 프로세스를設計하기 위하여서는 Agent Operations, Loop Intelligence, Agent Capabilities와 같은 새로운 개념을 도입하여야함.
