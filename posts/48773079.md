# 판독 루아 필터
판독에서 지원하는 필터는 판독 문서를 파싱하고 쓸 때 AST를 조작할 수 있음. 기존 필터는 JSON으로 AST를 받아서 조작한 뒤 다시 JSON으로 내보냄. 하지만 이런 방식은 JSON 쓰고 읽는 데 시간이 걸리고, 필터가 돌아가는 환경에 따라 다름. 
판독 2.0 버전부터는 루아 필터를 쓰면 외부 의존성을 전혀 안 쓰고 필터를 작성할 수 있음. 루아 인터프리터와 루아 판독 필터 라이브러리가 판독 실행 파일에 내장되어 있음. 
예를 들어, 강조된 글자를 작은 대문자로 바꾸는 루아 필터는 이런 거임: 
```lua
return { Strong = function (elem) return pandoc.SmallCaps(elem.content) end, }
```
이거 실행하려면 루아 파일에 저장하고 판독에 --lua-filter 옵션으로 넘겨주면 됨.
성능으로 치면 JSON 필터보다 루아 필터가 월등히 빠름. 판독 수동서를 HTML로 변환하는 데 걸리는 시간을 비교해봤음.
| 명령어 | 시간 |
| --- | --- |
| 판독 | 1.01초 |
| 필터 적용 판독 | 1.36초 |
| 파이썬 필터 적용 판독 | 1.40초 |
| 루아 필터 적용 판독 | 1.03초 |
루아 필터 문법
루아 필터는 문서의 각 요소에 대해 적용될 함수를 정의하는 테이블임. 각 함수의 이름은 요소의 이름과 같고, 그 함수는 해당 요소에 작용함. 필터는 각 문서 요소에 대해 호출되며, 그 때 요소의 이름에 해당하는 함수가 호출됨. 
요소의 이름에 해당하는 함수가 없으면 Inline이나 Block 함수가 호출됨. 
각 함수의 반환값은 nil, 해당 요소와 같은 타입의 요소, 해당 요소와 같은 타입의 요소 리스트 중 하나임. 반환값이 nil이면 요소는 바뀌지 않음. 
요소 리스트에 작용하는 필터 함수도 정의할 수 있음. 이 함수는 Inlines나 Blocks라는 이름을 가졌음. 
루아에서 판독 버전이나 파서/쓰기 옵션 등을 가져올 수 있음. 판독은 루아에 여러 전역 변수를 제공함. FORMAT은 판독 쓰기기에서 쓰이는 형식임. PANDOC_READER_OPTIONS, PANDOC_WRITER_OPTIONS는 파서/쓰기에 넘겨진 옵션들임.
판독에서 루아 필터 쓰면 개빡치게 편利하고 성능 개선됨. 그냥 쓰세요 ㄹㅇ
