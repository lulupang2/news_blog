### 마퀴 인터페이스에 대한 반전의법適用
애플의 마퀴 인터페이스 원리를 바꾸면 새로운 인터페이스가 나타날까? 이 기사에서는 그에 관한 실험을 해보는 건데, 96년에 나온 기사임.
기존 마퀴 인터페이스는 사용자 친화적이지만, 그만큼 한계가 있어. 예를 들어, 파일을 삭제할 때 쓰레기통에 집어넣는 방식이 직관적이지만, 실제로 삭제하는 과정은 복잡함. 그러니까 사용자가 파일을 삭제할 때, 단순히 파일을 쓰레기통으로 이동시키면 된다고 생각하지만, 실제로는 여러 단계의 처리가 필요해.
그리고 사용자가 이런 복잡한 과정을 이해하지 못하면, 인터페이스가 복잡하고 어렵게 느껴질 수밖에 없음. 그러니까 인터페이스를 디자인할 때는 사용자 친화적이면서도, 사용자의 목적을 명확하게 이해할 수 있게 만들어야 하는 거야.

#### 마퀴 인터페이스의 원리
마퀴 인터페이스의 원리는 다음과 같음:
*   사용자 친화적이어야 함
*   직관적이어야 함
*   일관성이 있어야 함
*   사용자의 행동을預測할 수 있어야 함
*   사용자의 목적을 이해할 수 있어야 함

이러한 원리를逆にすれば, 새로운 인터페이스가 나타날 수 있을 거라고 생각해. 기존 인터페이스의 한계를 극복하고, 사용자 친화적이면서도 사용자의 목적을 명확하게 이해할 수 있는 인터페이스를设计할 수 있을 거야.

#### 마퀴 인터페이스의逆버젼
이제 마퀴 인터페이스의 원리를逆にして 새로운 인터페이스를 디자인해 보자. 여기서는 몇 가지 예를 들어서 설명할 건데:
*   사용자와 컴퓨터가 공유하는 제어를 기반으로 하는 인터페이스
*   사용자의 행동을預測하는 代わりに, 사용자의 목적을 이해하는 인터페이스
*   일관성 대신, 사용자 친화적이면서도 사용자의 목적을 명확하게 이해할 수 있는 인터페이스

이러한 인터페이스는 기존 인터페이스의 한계를 극복하고, 사용자 친화적이면서도 사용자의 목적을 명확하게 이해할 수 있는 인터페이스를 제공할 수 있을 거야.

### 결론
마퀴 인터페이스의 원리를逆にして 새로운 인터페이스를 디자인하는 것은, 사용자 친화적이면서도 사용자의 목적을 명확하게 이해할 수 있는 인터페이스를 제공할 수 있는 방법이야. 기존 인터페이스의 한계를 극복하고, 새로운 인터페이스를 디자인하는 것은, 사용자 경험을 향상시키고, 사용자의 목적을 이해할 수 있는 방법이야. 그러니까 인터페이스를 디자인할 때는 사용자 친화적이면서도, 사용자의 목적을 명확하게 이해할 수 있게 만들어야 하고, 기존 인터페이스의 한계를 극복할 수 있는 방법을 찾아나서야 해.
