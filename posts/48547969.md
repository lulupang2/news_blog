# 아메리칸익스프레스의 새로운 결제 시스템 아키텍처
   아메리칸익스프레스의 핵심 결제 시스템은 전世界적으로 신용카드 회원과 파트너들이 사용하는 플랫폼임. 
   매일 수백만 건의 결제 요청이 처리되는데, 이 중에 하나라도 실패하면 큰일 南이야. 그래서 아메리칸익스프레스는 결제 실패를 최소화하는 새로운 아키텍처를 개발했어.
   
   ## 셀 기반 아키텍처(cell-based architecture)
   아메리칸익스프레스의 새로운 아키텍처는 '셀 기반 아키텍처'라고 불리는데, 이는 결제 시스템을 여러 개의 독립된 셀로 나누어 관리하는 것임. 
   각 셀은 독립적으로 작동할 수 있어, 하나의 셀이 고장 나면 다른 셀에는 영향을 미치지 않음. 
   이 아키텍처의 핵심은 셀을 독립적으로 관리하여 결제 실패의 영향을 최소화하는 거야.
   
   ### 셀의 특징
   각 셀은 다음과 같은 특징을 가짐:
   * 독립적으로 작동할 수 있어.
   * 자체의 마이크로 서비스와 데이터베이스를 가짐.
   * 하나의 실패 영역임. 즉, 하나의 셀이 고장 나면 다른 셀이 영향을 받지 않음.
   * 유지보수나故障에 대처하기 위해 셀을 제거할 수 있음.
   * 셀 사이의 동기적인 의존성이 없어.
   
   ## 데이터 및 처리 위치
   결제 요청을 처리하려면 데이터가 필요한데, 이 데이터는 정적 데이터와 동적 데이터로 나눌 수 있음. 
   정적 데이터는 통화 환율이나 상점 코드등이 있으며, 동적 데이터는 결제 요청이 처리될 때마다 변경되는 데이터임.
   
   ### 정적 데이터 복제
   정적 데이터는 각 셀에 미리 복제해 둠으로써, 결제 요청을 처리할 때마다 외부 시스템에 조회할 필요가 없어. 
   이런 방식으로 데이터를 지역적으로 관리하면, 결제 실패의 영향을 최소화할 수 있어.
   
   ### 동적 데이터 라우팅
   그러나 동적 데이터는 미리 복제해 둘 수 없기에, 각 셀이 독립적으로 작동할 수 있도록 동적 데이터를 라우팅해 줘야 해. 
   아메리칸익스프레스는 '지정된 라우터'를 사용해서, 결제 요청을 처리할 때마다 적절한 셀로 라우팅해 줌. 
   이렇게 함으로써, 결제 요청이 실패하는 경우를 최소화할 수 있어.
   
   ## 경계 강제
   아메리칸익스프레스의 새로운 아키텍처에서는, 각 셀이 독립적으로 작동할 수 있도록 경계를 강제함. 
   즉, 결제 요청이 하나의 셀로 들어오면, 다른 셀로 이동할 수 없다는 거지. 
   이렇게 하면, 결제 요청이 실패하는 경우를 최소화할 수 있어.
   
   # 결론
   아메리칸익스프레스의 새로운 결제 시스템 아키텍처는, 셀 기반 아키텍처와 데이터 및 처리 위치, 경계 강제 등을 통해 결제 실패를 최소화하는 데 성공했어. 
   이런 아키텍처는, 다른 회사에서도 참고할 수 있는 좋은 사례가 될 것 같아
