코드 복붙에 대한 이야기임.
코드를 중복해서 사용하는 건 유지 보수하기 좋지 않다고 생각함. 그래서 개발자들은 중복 코드를 추상화해서 사용함.
예를 들어, A라는 개발자가 코드를 중복적으로 사용한다고 가정해 봄. 그래서 B라는 개발자가 이 코드를 추상화해서 사용할 수 있도록 함수나 클래스로 만들어서 제공함. 
그 후 C라는 개발자가 또다시 이 추상화된 코드를 사용해야 하는데, 코드가 다소 다르긴 하지만 이 추상화된 코드를 그대로 사용해야 하는 입장임. 
C라는 개발자는 기존 추상화된 코드를 또다시 수정해서 사용하게 되는데, 이때 추상화된 코드에 또다시 조건문을 추가해서 사용하게 됨.
이런식으로 계속 진행하면 최종적으로 코드가 너무 복잡하고 어려워지게 되고, 이때 개발자들은 추상화 코드를 유지 보수해야 하는 입장임.
하지만 때로는 코드를 중복해서 사용하는 것보다 이런 추상화된 코드를 유지 보수하는 게 더 어렵고 불편할 수 있음.
그럼 어떻게 해야 할까? 그냥 코드를 중복해서 사용하고, 필요에 따라 코드를 추상화하여 사용하는 건 어때? ㅋㅋㅋ
코드를 중복해서 사용하는 건 결국 더 빠를 수 있고, 필요할 때 추상화하여 사용하면 된다고 생각함.
물론 이는 개발자가 코드를 작성하고 유지 보수하는 데 걸리는 시간과 노력을 고려해서 결정해야 할 부분임. 
결론은 코드를 중복해서 사용하는 걸 두려워하지 말고, 필요할 때 추상화하여 사용하는 게 더 효율적일 수 있다는 거임.
예를 들어, 개발자가 코드를 중복해서 사용하는 코드를 발견했다고 가정해 봄. 
그럼 이 코드를 추상화하여 사용할 수 있도록 함수나 클래스로 만들어서 제공할 수 있고, 이후에 또다시 코드를 중복해서 사용하는 코드가 발견되면, 
이때는 코드를 중복해서 사용하는 게 더 효율적일 수 있음.
이처럼 코드를 중복해서 사용하는 건 개발자에게서 자유로울 수 있는 선택肢이 될 수 있고, 
코드를 중복해서 사용하는 건 결국 코드를 유지 보수하기 더 빠를 수 있기 때문임.
요즘 코드를 작성하고 유지 보수하는 데에는 코드를 중복해서 사용하는 게 더 효율적일 수 있고, 
코드를 중복해서 사용하는 건 결국 더 빠를 수 있기 때문임.
코드를 작성하고 유지 보수하는 데에는 다양한 선택肢이 존재하고, 
코드를 중복해서 사용하는 건 개발자에게 중요한 기술임.
코드를 중복해서 사용하는 건 결국 코드를 더 빠르고 효율적으로 작성하고 유지 보수할 수 있도록 도와줄 수 있기 때문임.
코드를 중복해서 사용하는 건 개발자에게 중요한 선택肢이 될 수 있고, 
코드를 중복해서 사용하는 건 결국 더 빠르고 효율적으로 코드를 작성하고 유지 보수할 수 있도록 도와줄 수 있음.
