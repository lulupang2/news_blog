# 루프의 부흥
   코드 자동화 루프가 점점 더 보편화되는 지금, 소프트웨어 개발의 미래는 어떻게 될까? 
   요즘은 보통 코딩 에이전트 위에 뭔가 의미 있는 게 구축됨. 패턴은 모두 같음. 일종의 작업 큐에 일을 넣고, 기계가 일을 진행하고, 멈추고, 하니스가 작업이 끝났는지 말함. 
   아직 루프 안에 에이전트가 하나 있다고 생각해. 모델이 툴을 호출하고, 결과를 결합하고, 다른 툴을 호출하고, 파일을 읽고, 파일을 수정하고, 테스트를 실행하고, 결국 어떤 답을 내놓음. 
   이 루프는 우리가 지금까지 꽤 잘 알고 있었음. 다른 루프는 하니스 레벨 루프라고 부를 수 있음. 이 루프도 새로운 건 아니고, 클로드 코드 초기부터 사용했지만, 이 루프는 점점 더 중요한 역할을 함. 
   내가 아직 성공하지 못한 건, 내가 중요하게 여기는 코드에 이 방법을 적용하지 못한 것임. 내가 코드를 작성할 때, 코드를 이해하고 싶고, 내가 코드를 작성할 때, 코드가 어떤 일을 하는지 설명할 수 있어야 함. 
   코드의 이해가 중요하다고 생각해. 특히 루프를 사용해서 코드를 작성할 때, 코드가 어떻게 동작하는지 이해하는 게 중요함. 
   하지만, 지금의 모델은 코드를 너무 방어적으로, 너무 복잡하게, 너무 로컬한 방식으로 작성해. 예외를 많이 추가하고, 코드를 중복하고, 나쁨 아브스트랙션을 사용하고, 불확실한 디자인을 가리고, 그보다는 기계를 많이 사용해. 
   더 큰 문제는, 이 방식이 나아지지 않는다는 거임. 물론, 나의 입장에서는, 지금의 코드가 전보다 더 나빠졌다고 생각해.
   #
   # 루프가 효과적인 곳
   루프는 이미 일부 분야에서 효과적으로 작동하는 중임. 코드 포팅이 대표적인 예시임. 
   기계가 자동으로 코드를 포팅하거나, 성능을 최적화하는 등에는 루프가 매우 효과적임. 
   #
   # 소프트웨어의 미래
   하지만, 이런 루프를 사용해서 코드를 작성할 때, 코드의 이해하고 제어가 매우 어려워. 
   소프트웨어는 점점 더 복잡해지고, 자동화된 루프를 사용해서 코드를 작성하는 게 더 편해지지만, 코드의 품질은 떨어질 수도 있음. 
   #
   # 결론
   결론은, 루프는 이미 일부 분야에서 효과적으로 작동하는 중이고, 소프트웨어 개발의 미래는 어떻게 될지에 대해 많은 것을 생각하게 만듦. 
   루프를 사용해서 코드를 작성할 때, 코드의 이해하고 제어가 매우 중요하고, 소프트웨어 개발의 미래는 점점 더 자동화와 인공지능이 중요해질 거라는 결론임.
